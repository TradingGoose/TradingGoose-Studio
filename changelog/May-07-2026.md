### Design Decisions
- Records is now the operational history surface. Orders, log review, and stats share one page shell under `/workspace/[workspaceId]/records` instead of overloading the existing logs page with order history behavior.
- Order history is owned by workspace scope first. `orderHistoryTable.workspaceId`, route-level `checkWorkspaceAccess()`, and tool `_context.workspaceId` are required so order records are not discoverable by workflow ID alone.
- Workflow logs preserve deletion-tolerant metadata. `workflowExecutionLogs.workflowSummary`, nullable `workflowId`, and workspace IDs let log APIs, metrics, and webhooks keep rendering when the workflow row is missing.
- Webhook delivery config is snapshot-owned. `workflowLogWebhookDelivery.subscriptionSnapshot` is required delivery data; the background task does not reconstruct missing delivery config from live subscription rows.
- Order WHERE clauses do not hide join requirements. `buildOrderWhereCondition()` only searches base `orderHistoryTable` fields unless a caller passes `joinedSearchExpressions`, so routes own the joined `workflowExecutionLogs.workflowSummary` search expression beside their `LEFT JOIN`.
- The log search language is policy-driven. `LOGS_QUERY_POLICY` and `MONITOR_QUERY_POLICY` own supported fields and API mappings so UI search, monitor quick filters, `/api/logs`, and export do not grow separate qualifier implementations.
- Executor setup is intentionally stricter. The branch removes constructor overloads, missing accessible-block-map fallback, array first-item path fallback, and parallel suffix lookup fallbacks so invalid workflow state fails visibly during fresh development.
- Text workflow variables are `plain`. The removed `string` type is not backfilled because the project does not support legacy variable methods.

### Removed or Replaced Items
- The duplicated log type definitions in `apps/tradinggoose/stores/logs/filters/types.ts` were replaced with re-exports from `apps/tradinggoose/lib/logs/types.ts`. Future branches should not restore local `WorkflowLog`, `TraceSpan`, or `CostMetadata` copies in the store.
- Primary workspace navigation and shortcut behavior for log history moved from `/workspace/[workspaceId]/logs` to `/workspace/[workspaceId]/records`. Use `createWorkspaceNav()` and the Records route instead of reintroducing Logs as the primary history nav item.
- Workflow execution logs and snapshots are no longer owned only by non-null `workflowId`. Use workspace IDs plus `workflowSummary`; do not recreate code that assumes a workflow row must exist to display a historical log.
- Webhook delivery no longer reloads live subscription config to reconstruct delivery settings. Use `workflowLogWebhookDelivery.subscriptionSnapshot` created by `apps/tradinggoose/lib/logs/events.ts`.
- Order history recording no longer accepts workflow-only scope or the `_workflowId` side channel. Use `_context.workspaceId`, `_context.workflowId`, `_context.workflowLogId`, and `_context.submissionSource` through `executeTool()` and `tradingActionTool`.
- Broad `orderHistoryTable` JSON text search was replaced by explicit UUID and field expressions in `apps/tradinggoose/app/api/tools/trading/order-history/search/route.ts` and `apps/tradinggoose/app/api/orders/order-record-utils.ts`.
- The deprecated workflow variable type `'string'` was removed from `VariableType`, `VariableManager`, resolver formatting, and workflow-variable UI defaults. Use `'plain'` for text values.
- The overloaded `new Executor(workflow, states, env, input, variables)` path was replaced by `new Executor({ workflow, currentBlockStates, envVarValues, workflowInput, workflowVariables, contextExtensions })`.
- Input resolver compatibility fallbacks for missing accessible-block maps, implicit array first-element property access, and parallel iteration suffix lookup were removed. Use explicit indexes, generated accessible maps, and virtual block mappings.
- `parseIntervalDurationMs()` was removed from `apps/tradinggoose/app/workspace/[workspaceId]/monitor/components/shared/utils.ts` because active monitor code no longer uses it.
- `MonitorViewRowResponse`, `WorkflowTargetKey`, and several exported monitor config constant/type aliases in `components/view/view-config.ts` were folded into local implementation details. Reuse the exported monitor view/config types that remain public.

### Future Branch Guardrails
- Do not add new order APIs or tools that can read/write order history without `workspaceId` and `checkWorkspaceAccess()`.
- Do not pass workflow provenance through `_workflowId`; route it through `_context` and `ExecutionContextExtensions`.
- Do not add another log query parser or autocomplete model. Extend `QueryPolicy` and the parser/suggestion helpers in `apps/tradinggoose/lib/logs/*`.
- Do not recreate log types in Zustand stores or page components. Import from `apps/tradinggoose/lib/logs/types.ts`.
- Do not add hidden joined-table references inside `buildOrderWhereCondition()`. Put joined search expressions in the route or export query that owns the corresponding `LEFT JOIN`.
- Do not add runtime compatibility for missing webhook delivery snapshots. Invalid delivery rows should fail explicitly instead of deriving delivery config from a mutable subscription row.
- Do not restore the deprecated `string` variable type or add conversion aliases for it. Create text variables as `plain`.
- Do not restore executor compatibility fallback paths for missing workspace context, missing accessible maps, malformed parallel references, or implicit array property access.
- Do not manually edit generated files under `packages/db/migrations/*`; schema ownership remains in `packages/db/schema/workflows.ts`.
- Keep `/records` in workspace nav, workspace switching, and `Cmd/Ctrl+Shift+L` routing when changing operational history navigation.
- Keep order record serialization and CSV/list/detail behavior centralized in `apps/tradinggoose/app/api/orders/order-record-utils.ts`.
