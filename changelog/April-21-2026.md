### Design Decisions
- Personal PAYG remains a transition out of the default personal subscription row, not a separate pre-provisioned paid row. The branch intentionally uses advisory locks plus the `paygActivationAttemptId` metadata handshake in `apps/tradinggoose/app/api/billing/payg/activate/route.ts` so retries reconcile with Stripe instead of creating duplicate subscriptions.
- Personal Stripe customer ownership is centralized in `apps/tradinggoose/lib/billing/stripe-customers.ts`. Both the billing portal route and PAYG activation reuse that helper so customer creation, deleted-customer recovery, and idempotency-key selection stay consistent.
- Billing management for self-serve users now prefers Stripe Billing Portal over custom app-local cancel/restore flows. The replacement path is the shared `openBillingPortal()` helper plus the state machine in `apps/tradinggoose/global-navbar/settings-modal/components/subscription/payg-ui.ts`, not a bespoke dialog component.
- Onboarding allowance is treated as spend-down credit on the default personal path. `apps/tradinggoose/lib/billing/webhooks/subscription.ts` and `apps/tradinggoose/lib/billing/core/usage.ts` intentionally decrement `grantedOnboardingAllowanceUsd` when a personal user leaves that path, and only sync `customUsageLimit` automatically when it still equals the seeded default.
- Copilot turn status is now a transport-level stream contract. `turn_state` is emitted by the API routes and consumed by the stores so error turns and waiting-for-tools pauses do not depend on reconstructing state from tool-call timing alone.
- `/workspace/[workspaceId]/dashboard` is the canonical workspace URL now. The branch intentionally removes pathname-derived active-workflow logic and legacy `/w` redirect assumptions so dashboard widgets, invitation acceptance, and auth redirects all converge on one route.
- Monaco asset lookup is intentionally multi-root. `apps/tradinggoose/app/monaco-editor/esm/vs/[...assetPath]/route.ts` searches traced package roots and Bun install roots because this repo uses Bun and standalone tracing.
- The Zustand changes are deliberate consistency work, not a one-off optimization. Shared stores and selector consumers now use equality-aware `zustand/traditional` entrypoints so rerenders and hydration churn do not regress back to the plain hook behavior.

- Block prompt text and docs now standardize on exact reference grammar. `apps/tradinggoose/blocks/blocks/{api,function,webhook_request,condition,guardrails,mongodb,response,variables}.ts`, `apps/tradinggoose/tools/function/execute.ts`, and `apps/tradinggoose/lib/copilot/tools/server/blocks/block-mermaid-catalog.ts` all stop using invented `<block...>` examples and instead instruct callers to copy exact tags like `<agent.content>` or `<variable.riskLimit>`, while Function-block guidance now requires full Historical Data outputs plus object-shaped built-in indicator overrides. `apps/tradinggoose/lib/workflows/block-mermaid-contract.ts` and `apps/docs/content/docs/en/blocks/{loop,parallel}.mdx` update Loop/Parallel examples so incoming edges enter through container start handles and child outputs reconnect through container end handles.
- Billing overage math now treats included usage as explicit data instead of a price-derived fallback. `apps/tradinggoose/lib/admin/billing/tier-mutations.ts` now rejects any tier without `includedUsageLimitUsd`, `apps/tradinggoose/lib/billing/tiers.ts` makes `getTierUsageAllowanceUsd()` read only the included-usage field, and `apps/tradinggoose/lib/billing/threshold-billing.ts` now threshold-bills any active subscription with a real `stripeSubscriptionId`, even when the recurring price is zero, so PAYG tiers with included usage can still invoice overages correctly.
- The data-chart draw-tool picker no longer exposes `MarketDepth`. `apps/tradinggoose/widgets/widgets/data_chart/drawings/tool-types.ts` and `apps/tradinggoose/widgets/widgets/data_chart/components/draw-tool-icon-registry.ts` remove `MarketDepth` from the manual tool unions, labels, and icons, even though the underlying plugin code still exists in `apps/tradinggoose/widgets/widgets/data_chart/plugins/market-depth/*`.

### Design Decisions
- Public markdown is intentionally negotiated through the existing public pages instead of a second content store. `apps/tradinggoose/proxy.ts`, `apps/tradinggoose/app/api/markdown/route.ts`, and `apps/tradinggoose/lib/markdown/public-page-markdown.ts` keep HTML and markdown in one ownership path, with `MARKDOWN_BYPASS_HEADER` preventing recursive rewrites when converted pages fetch their own HTML.
- Homepage discovery is intentionally centralized in `apps/tradinggoose/lib/discovery/link-headers.ts`. That keeps the homepage `Link` headers, the API catalog payload, and the `/.well-known/api-catalog` response in sync instead of repeating those links in multiple routes.
- Copilot workflow editing now treats graph edits and block-config edits as separate tool families on purpose. The branch tightens `edit_workflow` vs `edit_workflow_block` descriptions because the previous overlap let the model retry full-document mutations for simple block property changes.
- Workflow output references now carry both `path` and `type` by design. `apps/tradinggoose/lib/copilot/tools/client/workflow/block-output-utils.ts` and `apps/tradinggoose/lib/copilot/tools/shared/schemas.ts` intentionally stop returning plain strings so later prompt/tool consumers can validate field compatibility and stop inventing placeholder prefixes.
- Workspace entity mentions and current contexts are intentionally driven by `apps/tradinggoose/widgets/widgets/copilot/workspace-entities.ts`. That one module now supplies mention menu options, submenu titles, context labels, pair-context resolution, and equality matching for workflows, skills, custom tools, indicators, and MCP servers.
- Included usage is now explicit billing configuration instead of a derived fallback from monthly price. `apps/tradinggoose/lib/billing/tiers.ts` and `apps/tradinggoose/lib/billing/threshold-billing.ts` take that position so zero-dollar PAYG tiers can still have real allowance and overage behavior without price-based guessing.
- The workflow editor header is intentionally narrower after this branch. `apps/tradinggoose/widgets/widgets/editor_workflow/components/control-bar/control-bar.tsx` keeps export, auto-layout, debug, deploy, and run, while `apps/tradinggoose/lib/copilot/inline-tool-call.tsx` prefers high-signal summary counts over noisy per-block chips.

### Future Branch Guardrails
- Do not add parallel public markdown or discovery routes outside `apps/tradinggoose/proxy.ts`, `apps/tradinggoose/app/api/markdown/route.ts`, and `apps/tradinggoose/lib/discovery/link-headers.ts`. Extend `isMarkdownRenderablePath()` and `renderPublicPageMarkdown()` instead.
- Do not reintroduce missing, relative, or inconsistent canonicals on public pages. Keep explicit canonicals in the landing/legal metadata files and keep the homepage sitemap entry aligned with the trailing-slash root used in `apps/tradinggoose/app/page.tsx`.
- Do not use `edit_workflow` for a single existing block `name`, `enabled`, or `subBlocks` change. The branch makes `edit_workflow_block` the default patch path for that case.
- Do not switch Copilot workflow references back to untyped string arrays or revive made-up `<block...>` placeholder examples. Future tools and prompts should keep using exact `outputs[].path`, `accessibleBlocks.outputs[].path`, and `<variable.name>` tags.
- Do not treat built-in indicators as editable workspace entities. Built-in indicators are read-only and identified by `runtimeId`; custom indicators are editable and identified by `entityId`.
- Do not duplicate workflow/skill/tool/indicator/MCP mention metadata outside `apps/tradinggoose/widgets/widgets/copilot/workspace-entities.ts`, and do not rebuild an inline mention system outside the split `apps/tradinggoose/widgets/widgets/copilot/components/user-input/` modules.
- Do not reintroduce the removed context-chip row in `copilot-message.tsx`; user-visible context highlighting now belongs inline in the message text.
- Do not rewire `TemplateModal` back into the workflow header or remove the `nodrag`/`nopan` stop-propagation behavior from `ActionBar` without a deliberate replacement interaction model. The current control bar intentionally stays focused on execution and deployment controls.
- Do not re-add `MarketDepth` to `tool-types.ts` or `draw-tool-icon-registry.ts` unless the product explicitly wants to expose that drawing tool again and the rest of the data-chart UX is updated with it.
- Do not restore Loop/Parallel docs or Mermaid examples that bypass Loop Start / Loop End or Parallel Start / Parallel End, or that claim a parallel container only supports one child block.
- Do not allow tiers without explicit `includedUsageLimitUsd`, and do not restore the monthly-price fallback in `getTierUsageAllowanceUsd()`. Threshold billing now depends on explicit included usage plus a real `stripeSubscriptionId`.
