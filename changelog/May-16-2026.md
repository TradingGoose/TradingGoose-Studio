### Design Decisions
- Yjs is now the source of truth for Copilot review reads and workflow mutation bases. The branch intentionally avoids database/API fallback reads because those can miss live review-session edits or overwrite unsaved Yjs state.
- Snapshot bootstrap is centralized in `readBootstrappedReviewTargetSnapshot()` so API routes, server-side Copilot context processing, and future Yjs readers use the same bridge lookup, bootstrap, expired-state, and persisted-state behavior.
- Workflow tools intentionally return the same entity envelope used by saved entity tools. `workflowId`, `workflowName`, and `workflowDocument` were parallel aliases that made downstream tool rendering and persisted result parsing inconsistent.
- `currentWorkflowState` remains an internal transport field, not a model-authored argument. The client reads the current Yjs workflow state immediately before calling the server mutation tool, and the server rejects missing or invalid state instead of reading from the database.
- Saved entity document reads intentionally use Yjs leases. List endpoints remain for discovery, but a saved entity's editable document must come from the same Yjs document that edit tools update.
- Workflow summaries now separate graph topology from sub-block text references. Copilot should use `workflowSummary.edges`, edge `scope`, and block `connections` rather than inferring graph connections from strings such as `<block.output>`.
- Resend audience membership now follows verified registration events instead of anonymous newsletter signups. The landing CTA no longer owns a public email capture API.

### Future Branch Guardrails
- Keep Copilot workflow and saved-entity document reads on the Yjs bootstrap/session path. If a later branch needs another read surface, make it consume `readBootstrappedReviewTargetSnapshot()` or the shared session lease rather than direct DB/API state.
- Preserve the generic entity result contract for workflow tools. Inputs may still use `workflowId` and `workflowDocument`, but results should remain `entityId`, `entityName`, and `entityDocument`.
- Keep `currentWorkflowState` generated by the client from the current Yjs document immediately before mutation. Do not let the model provide it, and do not let the server reconstruct it from normalized database tables.
- When workflow routes create, duplicate, update, or seed Yjs documents, pass the workflow name into `tryApplyWorkflowState()` so shared sessions and Copilot readable state keep `entityName`.
- Treat workflow summaries as topology contracts. Use explicit edges, `scope`, and connection counts; do not infer graph structure from sub-block references or hide edges whose endpoints are missing.
- Keep container edge handles explicit. Incoming outer edges to loop/parallel containers require `targetHandle: "target"`; child-to-container completion edges require the container end target handle.
- For saved entity tools, use list APIs only to discover entity ids and display names. Read and edit document fields through the Yjs entity session helpers.
- Keep mention source data entity-oriented. Avoid `Untitled Workflow` fallbacks that create ambiguous workflow/log mentions; omit incomplete rows instead.
- Do not add another newsletter contact path. Resend audience membership should follow verified registered users, while waitlist approval notifications should use `updateWaitlistStatuses()` and batched mailer calls.
- Keep `resolveResendServiceConfig()` as the canonical Resend runtime configuration reader for both transactional email and verified-user audience sync.
