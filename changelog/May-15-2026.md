### Design Decisions
- Subscription settings now derive their visible state through `getSubscriptionSurfaceState()` before rendering. The component no longer owns parallel plan-selection rules for free, paid, organization, custom organization, and team-member states.
- Public billing price text is formatted from hosted catalog data in `apps/tradinggoose/lib/billing/public-catalog.ts` so settings, plan cards, and admin billing summaries do not hand-format recurring price labels independently.
- Knowledge-base uploads intentionally use the authenticated server upload route, not batch presigned URLs. That keeps storage context, quota checks, upload validation, and knowledge-base ownership checks on the server path.
- Document processing moved onto `pendingExecution` because parsing and embedding generation are long-running work. The queue id includes the document id and request id so delete paths can remove queued document jobs without touching unrelated workflow executions.
- Search embeds queries with the same model used by the target knowledge base. Multi-KB vector search is allowed only when all requested knowledge bases share the same embedding model, preventing vector comparisons across incompatible embedding spaces.
- The PDF parser uses `unpdf` instead of `pdf-parse` because the branch makes `unpdf` the parser and Trigger.dev additional package dependency. Parser consumers should depend on the `FileParser` interface rather than importing a package parser directly.
- Webhook configuration is now trigger-owned. Trigger-mode blocks, selected trigger ids, `useTriggerConfigAggregation()`, `useWebhookManagement()`, and `TriggerSave` are the canonical path instead of a hidden legacy block and provider-specific webhook modal tree.
- Deploy modal trigger editing uses the same sub-block layout machinery as the editor. Required-field validation, advanced-mode filtering, deploy-ready state, and webhook save state are derived from trigger sub-block definitions rather than custom per-trigger JSX.
- Subflow definitions are centralized in `apps/tradinggoose/widgets/widgets/editor_workflow/components/subflows/config.ts` so loop and parallel metadata stays aligned across nodes, toolbar blocks, connection controls, and Copilot mention sources.
- Studio Mermaid container edges now require explicit start/end/target handles. The branch removes silent handle normalization so invalid serialized edges fail with boundary violations instead of being repaired implicitly.
