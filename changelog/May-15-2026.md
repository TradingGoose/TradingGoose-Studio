### Design Decisions
- Subscription plan display is now centered on the current public tier plus later public tiers by `displayOrder`, not a separate upgrade-only collection. That keeps the settings view stable for free users, paid users, and paid organization admins while still suppressing chooser UI for team members and custom sales-managed organization plans.
- A zero-dollar recurring tier is still a priced public tier. `formatBillingPriceLabel()`, `formatBillingPricePeriod()`, and the admin billing overview now distinguish `$0` from `null`, leaving `null` as the only custom/contact-sales signal.
- Trigger identity should be resolved, not persisted as a side effect. Singleton trigger blocks can derive their trigger id from `blockConfig.triggers.available`, while multi-trigger blocks still require explicit `selectedTriggerId`.
- Generic webhook setup belongs to trigger sub-block definitions, `TriggerSave`, `useWebhookManagement()`, and the deploy modal. The branch intentionally removes the parallel provider modal stack instead of keeping two webhook configuration paths.
- Workflow container edges use explicit handles as part of the canonical graph contract. The Mermaid parser can infer visible parentage, but the canonical edge metadata must still distinguish outer container input, start-to-child, and child-to-end edges.
