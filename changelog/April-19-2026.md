### Design Decisions
- Provider/runtime configuration and platform flags are intentionally separate contracts now. Use `packages/db/schema/system.ts` plus `apps/tradinggoose/lib/system-services/{catalog,service}.ts` for provider-specific settings or secrets, and use `apps/tradinggoose/lib/system-settings/service.ts` only for cross-platform flags such as registration or billing enablement.
- Stripe is explicitly deployment-owned again. The replacement `apps/tradinggoose/lib/system-services/stripe-runtime.ts` reads `STRIPE_SECRET_KEY` and `STRIPE_WEBHOOK_SECRET` from env, while `apps/tradinggoose/lib/billing/settings.ts` only decides whether billing is enabled at runtime. Future code should not move Stripe secrets back into DB-backed system settings.
- Copilot runtime tool exposure is generated from one registry. Extend `ToolIds`, `ToolArgSchemas`, and `TOOL_PROMPT_METADATA`, then let `getCopilotRuntimeToolManifest()` derive JSON Schema and semantic validators. The branch deliberately removed parallel hand-maintained tool lists and route-local schema islands.
- Workflow document mutation now assumes a canonical Mermaid document format instead of permissive ad hoc shapes. `tg-mermaid-v1`, `TG_WORKFLOW` / `TG_BLOCK` / `TG_EDGE` metadata prefixes, canonical block contracts, and sub-block validation are the expected integration surface for both Copilot edits and live-state application.
- Async execution now flows through a backlog that is keyed by billing scope instead of having separate direct async code paths per trigger type. `enqueuePendingExecution()` plus the drain loop are the expected coordination layer, and concurrency/rate-limit checks are applied around that queue rather than at scattered call sites.
- Better Auth billing plans must be build-safe. `apps/tradinggoose/lib/billing/plans.ts` returns an empty static config during `phase-production-build` and only resolves DB-backed plans at runtime, which keeps auth setup from failing during Next production compilation.

### Design Decisions
- Azure and Vercel are now first-class storage provider names. Future upload code should branch on `StorageProvider` or `getStorageProvider()` and should not reintroduce `blob` as a provider label.
- Vercel uploads must go through `apps/tradinggoose/app/api/files/vercel/client-upload/route.ts` and the signed token helpers in `apps/tradinggoose/lib/uploads/providers/vercel/{upload-token,download-token}.ts`. Do not rebuild a raw PUT flow for Vercel Blob.
- Azure multipart upload stays on the server-side multipart route, while Vercel Blob uses the signed client upload path. `apps/tradinggoose/app/api/files/multipart/route.ts` intentionally rejects Vercel.
- The file path helpers in `apps/tradinggoose/app/api/files/utils.ts` are now the canonical place to parse Azure/Vercel/S3 serve URLs and raw keys. Reuse them instead of adding new path parsing in route handlers.
- The billing multiplier split is intentional. Use `getTierWorkflowExecutionMultiplier()` for workflow log pricing and `getTierFunctionExecutionMultiplier()` for function runtime billing; do not revive the old `functionExecutionDurationMultiplier` name or the previous additive cost formula.
- Removing the hosted-mode gate from `apps/tradinggoose/proxy.ts` is deliberate. The proxy should not be used to 404 non-public routes in hosted environments anymore.

### Future Branch Guardrails
- Do not add new upload callers that assume a raw presigned PUT works for every provider. Vercel must keep using the signed client-upload route.
- Do not reintroduce `blob` path prefixes, `blob` storage provider names, or `USE_BLOB_STORAGE`.
- Do not bypass `apps/tradinggoose/lib/uploads/utils/validation.ts` when validating file size, MIME type, or image-only upload contexts.
- Do not restore hosted-mode route restrictions in `proxy.ts`; if public route policy changes, use a different explicit mechanism.
- Do not bring back the old billing multiplier names or the additive function-cost model. Keep workflow and function multipliers separate.
- Do not clear Copilot errors implicitly during store resets; preserve them unless the UI flow explicitly consumes them.

- When new admin billing fields are added, extend `apps/tradinggoose/app/admin/billing/tier-editor.tsx` and the shared billing snapshot/mutation types rather than recreating form logic in each page.
