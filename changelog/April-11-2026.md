# April-11-2026

## feat/admin-panel vs origin/staging

### Summary
- Adds a first-class system admin area for registration, billing, and system-managed integrations, with bootstrap ownership handled in `apps/tradinggoose/lib/admin/access.ts` and surfaced through `apps/tradinggoose/app/admin/*` plus `apps/tradinggoose/app/api/admin/*`.
- Replaces hardcoded signup and billing defaults with DB-backed system settings, billing tiers, workspace billing ownership, and runtime billing gating across `packages/db/schema/system.ts`, `apps/tradinggoose/lib/billing/*`, and the execution routes.
- Moves OAuth and SSO management onto explicit platform contracts: seeded system integration definitions, request-scoped system OAuth credential hydration, and issuer-discovery-only organization SSO registration.
- Cleans up adjacent platform ownership by centralizing public pricing copy, moving workspace navigation into the layout, splitting AI provider metadata from execution code, removing Robinhood support, and tightening release/image automation.

### Key Changes
- Registration is now controlled by system settings instead of assuming open signup. The durable contract lives in `packages/db/schema/system.ts` via `registrationModeEnum`, `systemSettings.registrationMode`, `waitlistStatusEnum`, and `waitlist`, while `apps/tradinggoose/lib/registration/shared.ts` and `apps/tradinggoose/lib/registration/service.ts` own the shared literals, CTA routing, waitlist mutations, invitation bypass, and eligibility checks.
- Signup, waitlist, and admin review all route through that registration contract. `apps/tradinggoose/app/api/registration/route.ts` exposes the current mode for rendering, `apps/tradinggoose/app/api/waitlist/route.ts` handles self-service waitlist submissions, `apps/tradinggoose/app/api/admin/registration/route.ts` manages mode changes and bulk waitlist approval or rejection, and the user/admin surfaces live in `apps/tradinggoose/app/(auth)/signup/*`, `apps/tradinggoose/app/(auth)/waitlist/*`, and `apps/tradinggoose/app/admin/registration/registration-admin.tsx`.
- Waitlist email delivery was folded into the same service layer instead of being handled ad hoc. `apps/tradinggoose/lib/registration/service.ts` sends confirmation and approval mail through `apps/tradinggoose/components/emails/render-email.ts`, which now renders `WaitlistConfirmationEmail` and `WaitlistApprovedEmail`.
- System administration is now a durable platform area. `apps/tradinggoose/lib/admin/access.ts` owns `claimFirstSystemAdmin()`, `getSystemAdminAccess()`, and `requireSystemAdminUserId()`, and `apps/tradinggoose/app/admin/layout.tsx` plus `apps/tradinggoose/app/admin/page.tsx` consistently treat "existing system admin" and "first authenticated bootstrap user" as the only supported admin entry conditions.
- Billing moved to DB-backed system tiers and settings. `packages/db/schema/system.ts` now defines `systemBillingSettings`, `systemBillingTier`, `systemIntegrationDefinition`, and `systemIntegrationSecret`, while `packages/db/schema/core.ts` adds `workspace.billingOwnerType`, `workspace.billingOwnerUserId`, `workspace.billingOwnerOrganizationId`, and `workspace.allowPersonalApiKeys`. The admin editor/query surface is built around `apps/tradinggoose/lib/admin/billing/snapshot.ts`, `apps/tradinggoose/lib/admin/billing/tier-mutations.ts`, `apps/tradinggoose/app/api/admin/billing/*`, and `apps/tradinggoose/app/admin/billing/*`.
- Workspace billing ownership now has a single canonical helper layer. `apps/tradinggoose/lib/workspaces/billing-owner.ts` owns `workspaceBillingOwnerSchema`, `serializeWorkspaceBillingOwner()`, `toWorkspaceApiRecord()`, `resolveWorkspaceBillingOwnerUpdate()`, and the guard rails that keep a billed workspace tied to valid admin access, and those helpers are wired into the workspace and organization APIs instead of duplicating billing-owner serialization or downgrade checks.
- Runtime billing now follows an explicit enablement contract instead of silently falling back to a default tier. `apps/tradinggoose/lib/billing/settings.ts` exposes `getResolvedBillingSettings()` and `isBillingEnabledForRuntime()`, `apps/tradinggoose/lib/billing/core/subscription.ts` owns `ensureDefaultUserSubscription()` and `backfillDefaultUserSubscriptions()`, and `apps/tradinggoose/lib/auth.ts` plus `apps/tradinggoose/app/api/admin/system-settings/route.ts` provision default user subscriptions when billing is turned on.
- Workflow, webhook, copilot, function, rate-limit, storage, and usage paths now resolve billing through shared workspace context helpers and respect the runtime billing flag. `apps/tradinggoose/lib/billing/workspace-billing.ts` centralizes `resolveWorkspaceBillingContext()`, `resolveWorkflowBillingContext()`, `toRateLimitBillingScope()`, and `getBillingContextResolutionMessage()`, while `apps/tradinggoose/app/api/workflows/[id]/execute/route.ts`, `apps/tradinggoose/background/schedule-execution.ts`, `apps/tradinggoose/lib/logs/execution/logger.ts`, `apps/tradinggoose/services/queue/RateLimiter.ts`, `apps/tradinggoose/lib/execution/concurrency-limit.ts`, `apps/tradinggoose/lib/billing/core/{billing,usage}.ts`, `apps/tradinggoose/lib/billing/calculations/usage-monitor.ts`, and `apps/tradinggoose/lib/billing/storage/limits.ts` now either block with an explicit billing-context message or short-circuit when billing is disabled.
- Public pricing, hosted marketing copy, and SEO metadata now share one public catalog. `apps/tradinggoose/lib/billing/catalog.ts` builds the server catalog, `apps/tradinggoose/lib/billing/public-catalog.ts` owns the public DTO and hosted-copy helpers, `apps/tradinggoose/app/api/billing/public-catalog/route.ts` exposes it, and `apps/tradinggoose/hooks/queries/public-billing-catalog.ts`, `apps/tradinggoose/app/(landing)/components/landing-pricing/landing-pricing.tsx`, `apps/tradinggoose/app/(landing)/components/structured-data.tsx`, and `apps/tradinggoose/lib/branding/metadata.ts` all consume that same contract.
- Organization upgrades, system-managed OAuth, and org-scoped SSO are now explicit service contracts instead of incidental side paths. `apps/tradinggoose/lib/subscription/upgrade-target.ts` resolves whether an upgrade should target a user or organization reference, `apps/tradinggoose/lib/system-integrations/catalog.ts` and `apps/tradinggoose/lib/system-integrations/resolver.ts` seed and hydrate enabled integration bundles, `apps/tradinggoose/lib/oauth/system-managed-config.ts` loads per-request system OAuth client credentials for `apps/tradinggoose/app/api/auth/[...all]/route.ts`, and `apps/tradinggoose/app/api/auth/sso/register/route.ts` now rejects manual OIDC endpoint overrides and relies on issuer discovery only.
- Trigger metadata, provider metadata, and workspace navigation ownership were tightened alongside the product changes. `apps/tradinggoose/triggers/types.ts` documents service-oriented `oauthProvider` and `webhookProvider` usage, `apps/tradinggoose/providers/ai/utils.ts` is now the browser-safe provider/model catalog while `apps/tradinggoose/providers/ai/index.ts` remains the runtime execution path, and `apps/tradinggoose/app/workspace/layout.tsx` plus `apps/tradinggoose/global-navbar/use-workspace-switcher.ts` make workspace navigation layout-owned and reset workflow registry state on workspace switches.
- The branch intentionally narrows supported trading and deployment behavior. `apps/tradinggoose/providers/trading/index.ts` now exposes only Alpaca and Tradier after deleting `apps/tradinggoose/providers/trading/robinhood/*`, and `.github/workflows/release.yml` plus `.github/workflows/images.yml` now build release notes from merged PR commits targeting `staging`, add contributors, and publish AMD64 ECR plus multi-arch GHCR images for app, migrations, and realtime services.

### Design Decisions
- System flags and numeric billing settings are split on purpose. `packages/db/schema/system.ts` stores the enable/registration flags separately from `systemBillingSettings`, and runtime code now asks `apps/tradinggoose/lib/billing/settings.ts` whether billing is enabled rather than inferring state from the presence or shape of a default tier.
- Workspace execution billing now resolves from billing owner data instead of from opportunistic workflow defaults. `apps/tradinggoose/lib/workspaces/billing-owner.ts` defines who owns billing, and `apps/tradinggoose/lib/billing/workspace-billing.ts` converts that owner into a concrete subscription scope used by rate limiting, usage metering, logging, and API execution.
- System-managed OAuth credentials are request-scoped rather than globally mutated. `apps/tradinggoose/lib/oauth/system-managed-config.ts` uses `AsyncLocalStorage` to bind resolved client credentials to a request, and `apps/tradinggoose/app/api/auth/[...all]/route.ts` hydrates those credentials only for the Better Auth callback/link/sign-in paths that need them.
- Public-facing registration and billing copy were centralized because those strings now drive pages, APIs, and metadata at the same time. `apps/tradinggoose/lib/registration/shared.ts` and `apps/tradinggoose/lib/billing/public-catalog.ts` are the intended single sources for those decisions.
- Several prior convenience layers were removed to keep ownership explicit. The branch prefers direct imports of specific subscription helpers, org-scoped route contracts, and catalog-backed OAuth/billing helpers over barrels, client-side wrapper hooks, or workflow-local auto-selection logic.

### Shared Contracts and Helpers to Reuse
- Reuse `apps/tradinggoose/lib/registration/shared.ts` for registration-mode literals, primary CTA labels, and auth-route decisions. Do not duplicate those strings in auth or landing components.
- Reuse `apps/tradinggoose/lib/registration/service.ts` for eligibility checks, waitlist mutations, invitation bypass, and waitlist email delivery. Public and admin routes should stay thin wrappers over this service.
- Reuse `apps/tradinggoose/lib/admin/access.ts` for any new admin page or admin API authorization. The bootstrap-admin path and permission model already live there.
- Reuse `apps/tradinggoose/lib/admin/billing/snapshot.ts`, `apps/tradinggoose/lib/admin/billing/tier-mutations.ts`, and `apps/tradinggoose/app/api/admin/billing/*` for system billing admin features. The tier validation and snapshot shape are already canonical.
- Reuse `apps/tradinggoose/lib/workspaces/billing-owner.ts` whenever workspace or organization code needs to read, serialize, or mutate billing ownership. The API shape should stay on the serialized `billingOwner` object returned by `toWorkspaceApiRecord()`.
- Reuse `apps/tradinggoose/lib/billing/settings.ts`, `apps/tradinggoose/lib/billing/core/subscription.ts`, `apps/tradinggoose/lib/billing/workspace-billing.ts`, and `apps/tradinggoose/lib/billing/subscriptions/utils.ts` for billing enablement, default-user provisioning, workspace billing context, and entitled-subscription status rules.
- Reuse `apps/tradinggoose/lib/billing/tier-summary.ts` and `apps/tradinggoose/lib/billing/tiers.ts` for tier access flags, empty-tier summaries, scope resolution, pricing, rate limits, and tier lookups instead of re-encoding those rules in route or component code.
- Reuse `apps/tradinggoose/lib/billing/catalog.ts`, `apps/tradinggoose/lib/billing/public-catalog.ts`, and `apps/tradinggoose/hooks/queries/public-billing-catalog.ts` for hosted pricing UI, metadata, and docs content. This branch made the public billing catalog the canonical hosted pricing contract.
- Reuse `apps/tradinggoose/lib/system-integrations/catalog.ts`, `apps/tradinggoose/lib/system-integrations/resolver.ts`, and `apps/tradinggoose/lib/oauth/system-managed-config.ts` for system-managed OAuth work. Provider bundles, enabled services, and decrypted secrets now belong to that catalog/resolver pipeline.
- Reuse `apps/tradinggoose/app/api/auth/sso/register/route.ts` for org-scoped SSO registration. The supported OIDC contract is issuer-based discovery only.
- Reuse `apps/tradinggoose/lib/subscription/upgrade-target.ts` and `apps/tradinggoose/lib/subscription/upgrade.ts` for upgrade flows that can land on either a user or an organization billing account.
- Reuse `apps/tradinggoose/providers/ai/utils.ts` for browser-safe provider catalogs, model lists, pricing/capability lookup, and cost helpers. Keep runtime request execution in `apps/tradinggoose/providers/ai/index.ts`.
- Reuse `apps/tradinggoose/global-navbar/use-workspace-switcher.ts` whenever workspace switching should also reset workflow-registry state or preserve the current route shape during a workspace change.

### Removed or Replaced Items
- `apps/tradinggoose/lib/workflows/credential-resolver.ts` was removed. Workflow-local OAuth credential auto-selection was replaced by `apps/tradinggoose/lib/system-integrations/{catalog,resolver}.ts` and `apps/tradinggoose/lib/oauth/system-managed-config.ts`.
- `apps/tradinggoose/hooks/use-subscription-state.ts` was removed. Subscription UI should compose Better Auth subscription data with the billing/organization query hooks and the pure helpers in `apps/tradinggoose/lib/subscription/helpers.ts` instead of restoring a monolithic wrapper hook.
- `apps/tradinggoose/lib/subscription/index.ts` was removed. Import `helpers.ts`, `types.ts`, `upgrade-target.ts`, and `upgrade.ts` directly so subscription ownership stays explicit.
- `apps/tradinggoose/hooks/queries/sso.ts` was removed. Org-scoped SSO registration now lives on the app routes and the surrounding organization/admin query invalidation paths.
- `packages/db/scripts/register-sso-provider.ts` and `packages/db/scripts/deregister-sso-provider.ts` were removed. SSO registration is now an application concern handled by `apps/tradinggoose/app/api/auth/sso/register/route.ts` and Better Auth integration, not by direct database scripts.
- `apps/tradinggoose/app/api/users/me/subscription/[id]/transfer/route.ts` was removed. Subscription targeting is now resolved up front by `apps/tradinggoose/lib/subscription/upgrade-target.ts` and `apps/tradinggoose/lib/subscription/upgrade.ts` instead of exposing a transfer endpoint after checkout.
- `apps/tradinggoose/lib/billing/constants.ts` and the old automatic-disable helper in `apps/tradinggoose/lib/billing/settings.ts` were removed. Use `apps/tradinggoose/lib/billing/{tier-summary,tiers,public-catalog,settings}.ts` plus the admin toggle instead of reviving hardcoded billing constants or automatic backfill/backstop behavior.
- `apps/tradinggoose/providers/trading/robinhood/*` was removed, and `apps/tradinggoose/providers/trading/index.ts` now registers only Alpaca and Tradier. There is no replacement Robinhood adapter in this branch.

### Future Branch Guardrails
- Do not reintroduce workflow-local OAuth credential inference. Extend the system integration catalog and request-scoped system OAuth helpers instead.
- Do not reintroduce manual OIDC endpoint override fields into `apps/tradinggoose/app/api/auth/sso/register/route.ts`. Issuer discovery is the only supported OIDC registration path.
- Do not bring back automatic billing-disable logic such as `disableBillingIfConfigurationInvalid()`. Billing enablement is an explicit admin/system-settings decision, and runtime code should check `isBillingEnabledForRuntime()`.
- Do not hardcode `active` as the only entitled subscription state. Use `BILLING_ENTITLED_SUBSCRIPTION_STATUSES` from `apps/tradinggoose/lib/billing/subscriptions/utils.ts`.
- Do not recreate a subscription barrel or a client-side "all billing state" wrapper hook. Import the exact helper or query you need.
- Do not recreate the removed SSO database scripts or the deleted `hooks/queries/sso.ts` layer. Extend the org-scoped app routes and existing organization/admin query contracts instead.
- Do not add another workspace navbar wrapper inside workspace pages. `apps/tradinggoose/app/workspace/layout.tsx` and `apps/tradinggoose/global-navbar/use-workspace-switcher.ts` now own that behavior.
- Do not duplicate pricing copy in landing components, docs, or metadata. Derive it from `apps/tradinggoose/lib/billing/public-catalog.ts` or the server catalog builder.
- Do not reintroduce Robinhood into `apps/tradinggoose/providers/trading/index.ts` without a deliberate product decision and a full replacement provider contract.
- When future code touches workspace billing, keep the serialized `billingOwner` API shape and the workspace-owner/admin invariants enforced by `apps/tradinggoose/lib/workspaces/billing-owner.ts`.
