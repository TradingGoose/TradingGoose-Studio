Das Workflow-Konsolen-Widget bietet eine Echtzeit-Ansicht der Workflow-Ausführung und zeigt Protokolle, Debug-Informationen und Streaming-Ausgaben während der Ausführung Ihres Workflows an.

## Übersicht

| Eigenschaft | Wert |
|----------|-------|
| **Schlüssel** | `workflow_console` |
| **Kategorie** | Dienstprogramm |
| **Symbol** | Aktivität |

## Funktionen

- **Echtzeit-Protokolle**: Verfolgen Sie Ausführungsprotokolle in Echtzeit
- **Streaming-Ausgabe**: Sehen Sie Block-Ausgaben in Echtzeit
- **Ausführungsstatus**: Überwachen Sie den aktuellen Status der Workflow-Ausführung
- **Fehleranzeige**: Identifizieren und beheben Sie Fehler schnell
- **Protokollfilterung**: Filtern Sie Protokolle nach Typ, Block oder Schweregrad
- **Konsole leeren**: Setzen Sie die Konsolenausgabe zurück

## Verwendung

Die Workflow-Konsole ist unerlässlich für:

- Debugging der Workflow-Ausführung
- Überwachung lang laufender Workflows
- Anzeige von Zwischenblock-Ausgaben
- Identifizierung von Fehlern und Engpässen

### Protokolltypen

| Typ | Beschreibung |
|------|-------------|
| **Info** | Allgemeine Ausführungsinformationen |
| **Debug** | Detaillierte Debugging-Informationen |
| **Warnung** | Nicht kritische Probleme |
| **Fehler** | Ausführungsfehler und -ausfälle |
| **Ausgabe** | Block-Ausgabedaten |

### Anzeige des Ausführungsablaufs

Die Konsole zeigt den Ausführungspfad durch Ihren Workflow:

1. **Start**: Workflow-Ausführung beginnt
2. **Block-Eintritt**: Jeder Block beginnt mit der Verarbeitung
3. **Block-Ausgabe**: Ergebnisse von jedem Block
4. **Block-Austritt**: Block wird abgeschlossen
5. **Ende**: Workflow-Ausführung wird abgeschlossen

<Callout>
Dieses Widget unterstützt den **pairColor**-Mechanismus. Verbinden Sie es mit einem Workflow-Editor, um Protokolle für den Workflow zu sehen, den Sie bearbeiten.
</Callout>
