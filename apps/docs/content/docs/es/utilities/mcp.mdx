El [Model Context Protocol](https://modelcontextprotocol.com/) (MCP) es un estándar abierto que permite a los agentes de IA conectarse de forma segura a herramientas externas y fuentes de datos. TradingGoose implementa un cliente MCP que utiliza JSON-RPC 2.0 y admite negociación de versión de protocolo en múltiples revisiones de especificación (`2025-06-18`, `2025-03-26`, `2024-11-05`).

## Transportes Compatibles

Cada servidor MCP que registres utiliza uno de tres modos de transporte basados en HTTP:

| Transporte | Descripción |
|-----------|-------------|
| **HTTP** | HTTP POST estándar con respuestas JSON. Adecuado para herramientas simples de solicitud/respuesta. |
| **SSE** | Server-Sent Events. El servidor puede responder con `text/event-stream` para resultados en streaming. |
| **Streamable HTTP** | El transporte MCP más reciente (especificación 2025-03-26). Admite gestión de sesiones mediante encabezados `Mcp-Session-Id` y negociación de tipo de contenido entre JSON y SSE. |

Los tres transportes son gestionados por el mismo cliente subyacente -- el campo de transporte controla cómo se etiqueta la conexión, pero el cliente negociará automáticamente el formato de respuesta basándose en el encabezado `Content-Type` del servidor.

## Agregar un Servidor MCP

<Steps>
<Step>
### Abrir el widget MCP Editor

Agrega el widget **MCP Editor** a tu dashboard, o navega a **Configuración del Espacio de Trabajo > Servidores MCP**. Consulta la [documentación del widget MCP Editor](/widgets/editor-mcp) para más detalles sobre la interfaz del widget.
</Step>
<Step>
### Proporcionar la configuración del servidor

Cada servidor requiere:

- **Nombre** -- Una etiqueta legible por humanos.
- **Transporte** -- Uno de `http`, `sse` o `streamable-http`.
- **URL** -- El endpoint del servidor (p. ej. `https://mcp.example.com/v1`).
- **Encabezados** (opcional) -- Encabezados personalizados como `Authorization`. Admite interpolación de variables de entorno (ver más abajo).
- **Tiempo de espera** (opcional) -- Tiempo de espera de solicitud en milisegundos. Por defecto 30 000 ms.
- **Reintentos** (opcional) -- Número de intentos de reintento. Por defecto 3.
</Step>
<Step>
### Guardar y probar

Haz clic en **Guardar** y luego usa el botón **Probar conexión** en la barra de herramientas del Editor MCP. El cliente ejecutará el protocolo de inicialización completo -- negociación de versión, intercambio de capacidades y una notificación `initialized` -- para confirmar que el servidor es accesible.
</Step>
</Steps>

## Autenticación y Variables de Entorno

Puedes referenciar variables de entorno del espacio de trabajo o personales en la URL del servidor y en los encabezados utilizando la sintaxis de doble llave:

```
Authorization: Bearer {{MY_API_KEY}}
```

En tiempo de ejecución, el servicio resuelve estos marcadores de posición con tu entorno descifrado efectivo. Si una variable referenciada no existe, la solicitud falla con un error descriptivo que enumera los nombres de las variables no resueltas.

<Callout type="warn">
Nunca codifiques secretos directamente en las configuraciones del servidor. Almacena siempre los valores sensibles como variables de entorno y referencíalos con la sintaxis `{{VAR_NAME}}`.
</Callout>

## Descubrimiento de Herramientas

Al conectarte a un servidor, el cliente envía una solicitud JSON-RPC `tools/list`. La respuesta se analiza y se convierte en una lista de herramientas, cada una con:

- **name** -- Identificador único de la herramienta en ese servidor.
- **description** -- Descripción de lo que hace la herramienta.
- **inputSchema** -- Un JSON Schema que describe los argumentos esperados.

Las herramientas descubiertas se almacenan en caché por espacio de trabajo con una política de desalojo LRU (máximo 1000 entradas). La caché se actualiza automáticamente al expirar, o puedes forzar una actualización desde la barra de herramientas del Editor MCP.

## Uso de Herramientas MCP en Bloques de Agente

Una vez que un servidor está registrado y habilitado, sus herramientas aparecen junto con las herramientas integradas en el selector de herramientas de cualquier bloque **Agente**.

<Steps>
<Step>
### Seleccionar herramientas MCP

Abre un bloque de Agente y expande la sección **Herramientas**. Las herramientas MCP aparecen listadas bajo el nombre de su servidor. Selecciona las que necesites.
</Step>
<Step>
### Ejecutar el agente

Durante la ejecución, el agente llama a `tools/call` en el servidor MCP correspondiente, pasando el nombre de la herramienta y los argumentos. El resultado se devuelve como contenido estructurado (texto, imagen o recurso) y se incorpora al contexto del agente.
</Step>
</Steps>

Cada ejecución de herramienta pasa por un marco de consentimiento y auditoría. El cliente verifica el servidor contra una política de seguridad (orígenes bloqueados, límites de velocidad) y registra un registro de auditoría antes de despachar la llamada.

## Seguridad

El cliente MCP aplica varias medidas de seguridad de forma predeterminada:

- **Control de consentimiento** -- Cada ejecución de herramienta se verifica contra la política de seguridad activa antes del despacho.
- **Lista de orígenes permitidos** -- El origen de la URL del servidor se agrega automáticamente a la lista de orígenes permitidos.
- **Limitación de velocidad** -- Límite configurable de ejecuciones de herramientas por hora (predeterminado: 1000).
- **Registro de auditoría** -- Cada decisión de consentimiento se etiqueta con un ID de auditoría único para su trazabilidad.
- **Orígenes bloqueados** -- Los servidores en la lista de bloqueo son rechazados de inmediato.

## Solución de problemas

<Cards>
<Card title="Servidor sin conexión">
Verifica que la URL sea correcta y que el servidor esté en ejecución. El cliente intenta la conexión con y sin barra diagonal al final, por lo que los problemas de ruta se gestionan automáticamente. Revisa la consola del navegador para obtener detalles sobre los errores de conexión.
</Card>
<Card title="Herramientas no visibles">
Confirma que el servidor esté **habilitado** y no eliminado de forma temporal. Usa el botón **Actualizar herramientas** en el Editor MCP para forzar la limpieza de caché y el redescubrimiento.
</Card>
<Card title="Errores de variables de entorno faltantes">
El mensaje de error indica exactamente qué variables no están resueltas. Agrégalas en la configuración de entorno de tu espacio de trabajo o personal y vuelve a intentarlo.
</Card>
<Card title="Errores de tiempo de espera">
Aumenta el valor de tiempo de espera del servidor en el Editor MCP. El valor predeterminado es 30 segundos. Los servidores lentos o las respuestas de herramientas de gran tamaño pueden requerir más tiempo.
</Card>
</Cards>
