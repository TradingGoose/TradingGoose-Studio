Las Herramientas personalizadas te permiten definir tus propias funciones invocables que los bloques de Agente pueden llamar en tiempo de ejecución. Cada herramienta es un par: una **configuración JSON** que declara el nombre, la descripción y los parámetros de la herramienta, más un **cuerpo de función JavaScript** que se ejecuta cuando se invoca la herramienta.

## Esquema de herramienta

Cada herramienta personalizada sigue el formato JSON de llamada a funciones de OpenAI. El editor valida el esquema en cada cambio y requiere todos los campos que se muestran a continuación.

```json
{
  "type": "function",
  "function": {
    "name": "addItemToOrder",
    "description": "Add one quantity of a food item to the order.",
    "parameters": {
      "type": "object",
      "properties": {
        "itemName": {
          "type": "string",
          "description": "The name of the food item to add to order"
        }
      },
      "required": ["itemName"]
    }
  }
}
```

- `type` debe ser `"function"`.
- `function.name` es el identificador que el agente ve y llama. Debe ser único dentro del espacio de trabajo.
- `function.description` le indica al agente cuándo usar la herramienta.
- `function.parameters` es un objeto JSON Schema estándar que describe las entradas.

## Código de herramienta

La pestaña de código contiene el cuerpo de una `async function(params, environmentVariables)`. Dentro del editor hay disponibles dos convenciones de referencia:

- **Parámetros** — usa el nombre del parámetro directamente (p. ej., `itemName`). El editor ofrece autocompletado basado en el esquema.
- **Variables de entorno** — usa la sintaxis de dobles llaves: `{{API_KEY}}`.

La función debe `return` un valor cuando se espera que la herramienta produzca una salida. Las importaciones externas no están soportadas; usa `fetch` o las APIs integradas.

## Creación y edición de herramientas

<Steps>
  <Step>
    Abre el widget [Editor de herramientas personalizadas](/widgets/editor-custom-tool) en el panel de tu espacio de trabajo.
  </Step>
  <Step>
    Selecciona una herramienta existente del menú desplegable o crea una nueva desde el widget de Lista de herramientas personalizadas.
  </Step>
  <Step>
    Alterna entre las pestañas **Config** y **Code** para editar el esquema JSON y el cuerpo JavaScript respectivamente. Ambas pestañas incluyen un asistente de IA que puede generar contenido a partir de un prompt en lenguaje natural.
  </Step>
  <Step>
    Haz clic en el botón **Guardar** en el encabezado del widget. El editor valida el esquema antes de guardar y rechaza nombres de herramientas duplicados.
  </Step>
</Steps>

## Cómo los agentes invocan herramientas personalizadas

Cuando agregas una herramienta personalizada a la lista de herramientas de un bloque Agente, la herramienta se registra con `type: "custom-tool"`. En tiempo de ejecución, el bloque agente lee `function.name` como identificador de la herramienta, `function.description` como descripción de la herramienta, y `function.parameters` como esquema de entrada. El agente decide si llamar a la herramienta y cuándo hacerlo basándose en estos campos y el contexto de la conversación, y luego ejecuta el cuerpo de la función JavaScript con los parámetros resueltos.

<Callout type="info">
  Los bloques Agente no pueden utilizar bloques core (API, Webhook, Función, Flujo de trabajo, Memoria) como herramientas. En la lista de herramientas solo se permiten integraciones, herramientas MCP y herramientas personalizadas.
</Callout>

## Herramientas personalizadas vs MCP

| Característica | Herramientas personalizadas | MCP |
|---------|-------------|-----|
| **Definición** | Esquema JSON y cuerpo de función JS | Servidor externo que implementa el Model Context Protocol |
| **Alcance** | Una función por herramienta | Un único servidor puede exponer múltiples herramientas |
| **Ejecución de código** | JavaScript en línea en el editor | Se ejecuta en el servidor MCP remoto |
| **Ideal para** | Lógica autocontenida o llamadas a API rápidas | Conexión a servidores de herramientas de terceros estandarizados |

Para integraciones de herramientas externas mediante MCP, consulta la [página de utilidades MCP](/utilities/mcp).

<Callout type="warn">
  Las herramientas personalizadas tienen alcance de workspace. Todos los miembros del workspace pueden verlas, editarlas y eliminarlas. Eliminar una herramienta personalizada suprime permanentemente su código y configuración.
</Callout>
