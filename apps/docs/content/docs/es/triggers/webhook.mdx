Los webhooks permiten que servicios externos activen la ejecución de flujos de trabajo mediante el envío de solicitudes HTTP. TradingGoose admite dos enfoques para los disparadores basados en webhooks.

<Callout type="info">
  Este es un disparador **basado en webhook**. Configura la URL del webhook en tu servicio externo para enviar eventos a TradingGoose.
</Callout>

### Cómo Funciona

1. **Agregar bloque Generic Webhook** — Arrastra el bloque Generic Webhook para iniciar tu flujo de trabajo
2. **Obtener URL del webhook** — Copia el endpoint único generado automáticamente
3. **Integración externa** — Configura tu servicio externo para enviar solicitudes POST a esta URL
4. **Ejecución del flujo de trabajo** — Cada solicitud a la URL del webhook activa el flujo de trabajo

### Características

- **Payload flexible** — Acepta cualquier estructura de payload JSON
- **Análisis automático** — Los datos del webhook se analizan automáticamente y quedan disponibles para los bloques posteriores
- **Autenticación** — Autenticación opcional mediante token bearer o cabecera personalizada
- **Limitación de tasa** — Protección integrada contra el abuso

## Modo Disparador para Bloques de Servicio

Alternativamente, puedes utilizar bloques de servicio específicos (como Slack, GitHub, etc.) en "modo disparador" para crear endpoints de webhook más especializados.

**Usa el Webhook Genérico cuando:**
- Integres con aplicaciones o servicios personalizados
- Necesites máxima flexibilidad en la estructura del payload
- Trabajes con servicios que no tienen bloques dedicados

**Usa el Modo Trigger cuando:**
- Trabajes con servicios compatibles (Slack, GitHub, etc.)
- Quieras filtrado de eventos específico del servicio
- Necesites registro automático de webhooks

<Callout>
El trigger de Webhook Genérico se activa cada vez que la URL del webhook recibe una solicitud, lo que lo hace ideal para integraciones en tiempo real.
</Callout>
