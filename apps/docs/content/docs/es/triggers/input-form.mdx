Utilice un disparador de Formulario de Entrada cuando un flujo de trabajo deba iniciarse desde el editor con campos tipados. El panel de ejecución muestra exactamente los campos que usted define, por lo que el flujo de trabajo siempre recibe datos limpios.

<Callout type="info">
  Este es un disparador **manual** con entrada estructurada. Defina su esquema y ejecútelo desde el editor.
</Callout>

Agregue campos en el constructor de Formato de Entrada. Tipos admitidos:

- **string** — Valores de texto
- **number** — Valores numéricos
- **boolean** — Valores verdadero/falso
- **json** — Objetos JSON

### Esquema de Salida

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `<blockName.field>` | variable | Cada campo definido en el Formato de Entrada |
| `<blockName.input>` | json | Carga útil de entrada estructurada completa |

## Flujos de Trabajo Secundarios

Los disparadores de Formulario de Entrada también alimentan el bloque de Flujo de Trabajo. Al incrustar un flujo de trabajo secundario, el paso de mapeo muestra los campos del formulario secundario para que pueda conectar variables del flujo de trabajo principal.

<Callout>
¿Necesita un botón de ejecución rápida sin campos? Utilice un Disparador Manual en su lugar.
</Callout>
