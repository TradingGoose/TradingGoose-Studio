El disparador de API expone tu flujo de trabajo como un endpoint HTTP seguro. Envía datos JSON al endpoint y tu flujo de trabajo los procesa de inmediato. Las llamadas a la API siempre se ejecutan contra tu último despliegue.

<Callout type="info">
  Este es un disparador **basado en webhook**. Los servicios externos o scripts envían solicitudes HTTP para iniciar el flujo de trabajo.
</Callout>

Añade un campo de **Formato de entrada** para cada parámetro. Tipos admitidos:

- **string** — Valores de texto
- **number** — Valores numéricos
- **boolean** — Valores verdadero/falso
- **json** — Objetos JSON
- **files** — Cargas de archivos (acceso mediante `<api.fieldName[0].url>`, `<api.fieldName[0].name>`, etc.)

### Referencia de salida

| Referencia | Tipo | Descripción |
|-----------|------|-------------|
| `<api.field>` | varía | Campo definido en el Formato de entrada |
| `<api.input>` | json | Cuerpo completo de la solicitud estructurada |

## Ejemplo de solicitud

```bash
curl -X POST \
  https://tradinggoose.ai/api/workflows/WORKFLOW_ID/execute \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: YOUR_KEY' \
  -d '{"userId":"demo-user","maxTokens":1024}'
```

## Respuestas en streaming

Habilita el streaming en tiempo real para recibir la salida del flujo de trabajo a medida que se genera.

### Parámetros de la solicitud

- `stream` — Establece en `true` para habilitar el streaming mediante Server-Sent Events (SSE)
- `selectedOutputs` — Array de salidas de bloques a transmitir en streaming (p. ej., `["agent1.content"]`)

### Ejemplo

```bash
curl -X POST \
  https://tradinggoose.ai/api/workflows/WORKFLOW_ID/execute \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: YOUR_KEY' \
  -d '{"message": "Count to five", "stream": true, "selectedOutputs": ["agent1.content"]}'
```

### Formato de respuesta

```
data: {"blockId":"...","chunk":"One"}
data: {"blockId":"...","chunk":", two"}
data: {"event":"done","success":true,"output":{},"metadata":{"duration":610}}
data: [DONE]
```

### Formato de carga de archivos

```json
{
  "documents": [{
    "type": "file",
    "data": "data:application/pdf;base64,JVBERi0xLjQK...",
    "name": "document.pdf",
    "mime": "application/pdf"
  }]
}
```

<Callout type="warning">
Un flujo de trabajo solo puede contener un API Trigger. Publica un nuevo despliegue después de realizar cambios para que el endpoint se mantenga actualizado.
</Callout>
