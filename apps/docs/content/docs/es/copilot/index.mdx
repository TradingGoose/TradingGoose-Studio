Copilot es un asistente impulsado por IA que opera en todo el espacio de trabajo de TradingGoose, no solo en los flujos de trabajo. Puede gestionar flujos de trabajo, servidores MCP, habilidades, herramientas personalizadas, bases de conocimiento, despliegues, variables y más. Copilot se ejecuta dentro del [widget de Copilot](/widgets/copilot).

## Capacidades

Copilot tiene acceso a un conjunto completo de herramientas que abarcan toda la plataforma:

### Gestión de flujos de trabajo
- **Editar flujos de trabajo** — agregar, eliminar o reconfigurar bloques y conexiones. Las ediciones se transmiten en tiempo real como vistas previas en línea antes de ser confirmadas.
- **Desplegar flujos de trabajo** — verificar el estado del despliegue y activar nuevos despliegues.
- **Ejecutar flujos de trabajo** — ejecutar flujos de trabajo e inspeccionar resultados.
- **Gestionar variables** — obtener y establecer variables globales de flujos de trabajo.
- **Ver consola** — recuperar registros de ejecución y salida en tiempo real.

### Herramientas del espacio de trabajo
- **Gestionar servidores MCP** — crear, actualizar y configurar conexiones de servidores MCP.
- **Gestionar habilidades** — crear, editar y eliminar definiciones de habilidades reutilizables.
- **Gestionar herramientas personalizadas** — crear y editar esquemas y código de herramientas personalizadas.
- **Base de conocimiento** — consultar y referenciar documentos de la base de conocimiento.

### General
- **Explicar** — responder preguntas sobre los conceptos de TradingGoose o cualquier flujo de trabajo en tu espacio de trabajo.
- **Planificar** — para tareas de mayor envergadura, Copilot genera una lista de tareas que rastrea el progreso a medida que avanza en cada paso.
- **Contexto de referencia** — usar el menú de mención `@` para incorporar chats anteriores, otros flujos de trabajo, bloques específicos, documentos de la base de conocimiento, documentación, plantillas o registros de ejecución.
- **Archivos adjuntos** — adjuntar archivos directamente a un mensaje para contexto adicional.

## Niveles de Acceso

El control de nivel de acceso en la barra de entrada determina el grado de autonomía que tiene Copilot.

<Cards>
  <Card title="Limitado (predeterminado)">
    Copilot hace una pausa antes de ejecutar cualquier llamada a herramienta que tenga una marca de interrupción y antes de ejecutar herramientas de integración. Usted revisa y aprueba cada acción.
  </Card>
  <Card title="Completo">
    Copilot ejecuta automáticamente todas las llamadas a herramientas, herramientas de integración y ediciones de flujos de trabajo sin confirmación. Utilice esta opción cuando confíe en la tarea actual y desee una iteración más rápida.
  </Card>
</Cards>

## Modelos Compatibles

Copilot es compatible con modelos de Anthropic y OpenAI. Puede habilitar o deshabilitar modelos individuales en **Configuración > Copilot**. El modelo activo puede cambiarse por mensaje desde la barra de entrada.

**Anthropic** -- claude-4.5-haiku, claude-4-sonnet, claude-4.5-sonnet, claude-4.1-opus

**OpenAI** -- gpt-4o, gpt-4.1, gpt-5-fast, gpt-5, gpt-5-medium, gpt-5-high, o3

Los modelos están agrupados por nivel de capacidad (rápido, estándar, razonamiento). Los modelos habilitados se sincronizan entre el widget y la configuración para que su selección sea coherente en todas partes.

## Configuración

<Steps>
  <Step>
    ### Alojado (tradinggoose.ai)

    Copilot funciona de inmediato. Abra cualquier espacio de trabajo, agregue el widget **Copilot** a un panel y comience a chatear.
  </Step>
  <Step>
    ### Auto-alojado

    Las instancias auto-alojadas necesitan una clave de API de Copilot para conectarse al servicio de inferencia administrado.

    1. Inicie sesión en [tradinggoose.ai](https://tradinggoose.ai) y abra **Configuración > Copilot**.
    2. Haga clic en **Crear** para generar una nueva clave de API. Cópiela de inmediato, ya que se muestra una sola vez.
    3. Establezca la variable de entorno `COPILOT_API_KEY` en su implementación auto-alojada con el valor copiado.
    4. Reinicie la aplicación. El widget de Copilot se conectará a través de la clave.
  </Step>
</Steps>

<Callout type="warn">
  Las claves de API otorgan acceso completo a Copilot. Guárdelas de forma segura y rótalas si se ven comprometidas. Puede eliminar una clave en cualquier momento desde la misma página de configuración; la eliminación revoca el acceso de inmediato.
</Callout>

## Gestión de chats

Cada sesión de Copilot está limitada al contexto del panel activo. El menú desplegable del encabezado muestra todos los chats anteriores agrupados por antigüedad (Hoy, Ayer, Esta semana, La semana pasada, Anteriores). Desde el encabezado puede:

- **Cambiar de chat** -- haga clic en cualquier conversación anterior para reanudarla.
- **Iniciar un nuevo chat** -- haga clic en el botón **+** del encabezado para crear una sesión nueva.
- **Eliminar un chat** -- pase el cursor sobre una entrada de chat y haga clic en el icono de papelera. Un cuadro de diálogo de confirmación evita eliminaciones accidentales.

El estado del chat persiste entre recargas de página y cambios de tamaño del panel. Cuando un widget está vinculado a un panel de editor, el canal de chat queda limitado a ese panel específico, lo que permite mantener conversaciones independientes en paralelo.

## Integración con el editor

Copilot es consciente del contexto — cuando está vinculado a un panel de editor, incluye automáticamente el contexto activo (flujo de trabajo, indicador, skill, servidor MCP o herramienta personalizada) al enviar mensajes, de modo que puede referenciar y modificar los recursos correctos sin necesidad de instrucciones adicionales.

Cuando Copilot transmite una respuesta que incluye llamadas a herramientas (por ejemplo, agregar un bloque, crear un skill o configurar un servidor MCP), cada acción se muestra como una vista previa en línea. En el modo **Limitado**, usted aprueba o rechaza cada cambio; en el modo **Completo**, los cambios se aplican automáticamente.

Si necesita deshacer una acción de Copilot, puede revertir o editar cualquier mensaje anterior directamente en el chat. Los mensajes posteriores al punto editado aparecen atenuados para indicar que serán regenerados.

## Facturación

El uso de Copilot se factura por token del proveedor de LLM subyacente:

- **Tokens de entrada** -- facturados a la tarifa base del proveedor (al costo).
- **Tokens de salida** -- facturados a 1,5x la tarifa base de salida del proveedor.

```
cost = (inputTokens * inputPrice + outputTokens * outputPrice * 1.5) / 1_000_000
```

<Callout type="info">
  Los precios son por millón de tokens. Consulte la página de [Cálculo de Costos](/execution/costs) para ver ejemplos detallados.
</Callout>
