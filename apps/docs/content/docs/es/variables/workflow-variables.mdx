Las variables en TradingGoose funcionan como un almacén global de datos al que cualquier bloque de tu flujo de trabajo puede acceder y modificar, permitiéndote almacenar y compartir datos a lo largo de tu flujo de trabajo mediante variables globales. Proporcionan una forma eficaz de compartir información entre distintas partes del flujo de trabajo, mantener el estado y crear aplicaciones más dinámicas.

</Callout>

La función de Variables actúa como un almacén de datos centralizado para tu flujo de trabajo, permitiéndote:

## Creación de Variables

Puedes crear y gestionar variables desde el panel de Variables en la barra lateral. Cada variable tiene:

- **Nombre**: Un identificador único utilizado para referenciar la variable
- **Valor**: Los datos almacenados en la variable (admite varios tipos de datos)
- **Descripción** (opcional): Una nota que explica el propósito de la variable

## Acceso a las Variables

Se puede acceder a las variables desde cualquier bloque de tu flujo de trabajo mediante el menú desplegable de variables. Simplemente:

1. Escribe `<` en cualquier campo de texto dentro de un bloque
2. Navega por el menú desplegable para seleccionar entre las variables disponibles
3. Selecciona la variable que deseas utilizar

Cuando escribes `<` en un campo de texto, aparece un menú desplegable con todas las variables del flujo de trabajo disponibles. Selecciona la que necesites para insertar una referencia a ella.

<Callout>
  También puedes arrastrar la etiqueta de conexión a un campo para abrir el menú desplegable de variables y acceder
  a las variables disponibles.
</Callout>

## Uso de Variables en Bloques

Cuando accedes a una variable desde un bloque, puedes:

- **Leer su valor**: Utilizar el valor actual de la variable en la lógica de tu bloque
- **Modificarla**: Actualizar el valor de la variable según el procesamiento de tu bloque
- **Usarla en expresiones**: Incluir variables en expresiones y cálculos

## Ámbito de las Variables

Las variables en TradingGoose tienen ámbito global, lo que significa:

- Son accesibles desde cualquier bloque de tu flujo de trabajo
- Los cambios en las variables persisten durante toda la ejecución del flujo de trabajo
- Las variables mantienen sus valores entre ejecuciones, a menos que se reinicien explícitamente

## Buenas Prácticas

- **Usa Nombres Descriptivos**: Elige nombres de variables que indiquen claramente lo que representa la variable. Por ejemplo, usa `userPreferences` en lugar de `up`.
- **Documenta tus Variables**: Añade descripciones a tus variables para ayudar a otros miembros del equipo a comprender su propósito y uso.
- **Ten en Cuenta el Ámbito de las Variables**: Recuerda que las variables son globales y pueden ser modificadas por cualquier bloque. Diseña tu flujo de trabajo teniendo esto en cuenta para evitar comportamientos inesperados.
- **Inicializa las Variables al Principio**: Configura e inicializa tus variables al comienzo de tu flujo de trabajo para asegurarte de que estén disponibles cuando se necesiten.
- **Gestiona las Variables Inexistentes**: Considera siempre el caso en que una variable aún no exista o pueda tener un valor inesperado. Añade la validación adecuada en tus bloques.
- **Limita el Número de Variables**: Mantén el número de variables en un nivel manejable. Demasiadas variables pueden dificultar la comprensión y el mantenimiento de tu flujo de trabajo.
