TradingGoose admite dos sistemas de variables complementarios. Las **variables de entorno** almacenan valores de configuración y secretos (claves API, tokens, URLs de bases de datos) que permanecen fuera de las definiciones de tu flujo de trabajo y se resuelven en tiempo de ejecución. Las **variables de flujo de trabajo** actúan como un almacén de datos global dentro de un único flujo de trabajo, permitiendo que cualquier bloque lea y escriba estado compartido durante una ejecución.

Juntos, cubren tanto la configuración externa como el flujo de datos interno. Las variables de entorno mantienen los secretos seguros y portables entre flujos de trabajo, mientras que las variables de flujo de trabajo permiten que los bloques se pasen datos entre sí sin conexiones directas.

## Tipos de Variables

<Cards>
  <Card title="Variables de Entorno" href="/variables/environment-variables">
    Gestiona claves API, tokens y otros secretos a nivel personal o de espacio de trabajo. Se referencian con sintaxis de doble llave y se resuelven cuando se ejecuta un flujo de trabajo.
  </Card>
  <Card title="Variables de Flujo de Trabajo" href="/variables/workflow-variables">
    Almacén global de clave-valor con alcance a un único flujo de trabajo. Crea, lee y actualiza variables desde cualquier bloque para compartir estado en todo tu flujo de trabajo.
  </Card>
</Cards>

## Comparación Rápida

| | Variables de Entorno | Variables de Flujo de Trabajo |
|---|---|---|
| **Alcance** | Personal o a nivel de espacio de trabajo | Flujo de trabajo único |
| **Duración** | Persistente en todas las ejecuciones | Persiste entre ejecuciones salvo reinicio |
| **Uso típico** | Claves API, tokens, configuración | Pasar datos entre bloques, estado |
| **Sintaxis de acceso** | `{{VARIABLE_NAME}}` | `<variable_name>` mediante menú desplegable |
| **Quién puede editar** | Página de configuración (permisos aplicables) | Panel de variables en el editor |
