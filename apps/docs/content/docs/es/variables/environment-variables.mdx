Las variables de entorno proporcionan una forma segura de gestionar valores de configuración y secretos en sus flujos de trabajo, incluidas las claves de API y otros datos confidenciales a los que sus flujos de trabajo necesitan acceder. Mantienen los secretos fuera de las definiciones de sus flujos de trabajo mientras los ponen a disposición durante la ejecución.

## Tipos de Variables

Las variables de entorno en TradingGoose funcionan en dos niveles:

- **Variables de Entorno Personales**: Privadas para su cuenta, solo usted puede verlas y usarlas
- **Variables de Entorno del Espacio de Trabajo**: Compartidas en todo el espacio de trabajo, disponibles para todos los miembros del equipo

<Callout type="info">
Las variables de entorno del espacio de trabajo tienen precedencia sobre las personales cuando existe un conflicto de nombres.
</Callout>

## Configuración de Variables de Entorno

Navegue a **Configuración** para configurar sus variables de entorno. Desde la configuración de su espacio de trabajo, puede crear y gestionar tanto variables de entorno personales como a nivel de espacio de trabajo. Las variables personales son privadas para su cuenta, mientras que las variables del espacio de trabajo se comparten con todos los miembros del equipo.

### Establecer el Alcance de Variables al Espacio de Trabajo

Use el interruptor de alcance del espacio de trabajo para poner las variables a disposición de todo su equipo. Al habilitar el alcance del espacio de trabajo, la variable queda disponible para todos los miembros del espacio de trabajo y puede utilizarse en cualquier flujo de trabajo dentro de ese espacio de trabajo.

### Vista de Variables del Espacio de Trabajo

Una vez que tenga variables con alcance de espacio de trabajo, aparecerán en su lista de variables de entorno junto con sus variables personales, claramente etiquetadas con su alcance.

## Uso de Variables en Flujos de Trabajo

Para referenciar variables de entorno en sus flujos de trabajo, use la notación `{{}}`. Cuando escriba `{{` en cualquier campo de entrada, aparecerá un menú desplegable que mostrará tanto sus variables de entorno personales como las del espacio de trabajo. Simplemente seleccione la variable que desea utilizar.

## Precedencia de Variables

Cuando tienes variables personales y de espacio de trabajo con el mismo nombre:

1. **Las variables de espacio de trabajo tienen precedencia** sobre las variables personales
2. Esto evita conflictos de nombres y garantiza un comportamiento consistente en los flujos de trabajo del equipo
3. Si existe una variable de espacio de trabajo, la variable personal con el mismo nombre es ignorada

<Callout type="warning">
Elige los nombres de las variables con cuidado para evitar sobreescrituras no deseadas. Considera usar tus iniciales como prefijo en las variables personales o el nombre del proyecto en las variables de espacio de trabajo.
</Callout>

## Mejores Prácticas de Seguridad

### Para Datos Sensibles
- Almacena claves de API, tokens y contraseñas como variables de entorno en lugar de codificarlas directamente
- Usa variables de espacio de trabajo para recursos compartidos que varios miembros del equipo necesiten
- Mantén las credenciales personales en variables personales

### Nomenclatura de Variables
- Usa nombres descriptivos: `DATABASE_URL` en lugar de `DB`
- Sigue convenciones de nomenclatura consistentes en todo el equipo
- Considera usar prefijos para evitar conflictos: `PROD_API_KEY`, `DEV_API_KEY`

### Control de Acceso
- Las variables de entorno del espacio de trabajo respetan los permisos del espacio de trabajo
- Solo los usuarios con acceso de escritura o superior pueden crear/modificar variables de espacio de trabajo
- Las variables personales son siempre privadas para el usuario individual
