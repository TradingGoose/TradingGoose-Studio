El bloque Esperar pausa tu flujo de trabajo durante un tiempo determinado antes de continuar con el siguiente bloque. Úsalo para agregar retrasos entre acciones, respetar los límites de tasa de las API o espaciar operaciones.

## Descripción general

El bloque Esperar te permite:

<Steps>
  <Step>
    <strong>Agregar retrasos de tiempo</strong>: Pausar la ejecución entre los pasos del flujo de trabajo
  </Step>
  <Step>
    <strong>Respetar los límites de tasa</strong>: Espaciar las llamadas a la API para mantenerse dentro de los límites
  </Step>
  <Step>
    <strong>Programar secuencias</strong>: Crear flujos de trabajo temporizados con retrasos entre acciones
  </Step>
</Steps>

## Configuración

### Tiempo de espera

Ingresa la duración para pausar la ejecución:
- **Entrada**: Número positivo
- **Máximo**: 600 segundos (10 minutos) o 10 minutos

### Unidad

Elige la unidad de tiempo:
- **Segundos**: Para retrasos cortos y precisos
- **Minutos**: Para pausas más largas

<Callout type="info">
  Los bloques Esperar pueden cancelarse deteniendo el flujo de trabajo. El tiempo máximo de espera es de 10 minutos.
</Callout>

## Salidas

- **`<wait.waitDuration>`**: La duración de la espera en milisegundos
- **`<wait.status>`**: Estado de la espera ('waiting', 'completed' o 'cancelled')

## Mejores prácticas

- **Mantén las esperas razonables**: Utiliza Wait para retrasos de hasta 10 minutos. Para retrasos más prolongados, considera los flujos de trabajo programados
- **Monitoriza el tiempo de ejecución**: Ten en cuenta que las esperas aumentan la duración total del flujo de trabajo
