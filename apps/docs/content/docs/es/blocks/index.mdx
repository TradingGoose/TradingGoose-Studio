Los bloques son los componentes base que se conectan entre sí para crear flujos de trabajo de IA. Piensa en ellos como módulos especializados que gestionan una tarea específica: desde interactuar con modelos de IA hasta realizar llamadas a APIs o procesar datos.

## Tipos de bloques principales

TradingGoose ofrece tipos de bloques esenciales que gestionan las funciones principales de los flujos de trabajo de IA:

### Bloques de procesamiento
- **[Agent](/blocks/agent)** - Interactúa con modelos de IA (OpenAI, Anthropic, Google, modelos locales)
- **[Function](/blocks/function)** - Ejecuta código JavaScript/TypeScript personalizado
- **[API](/blocks/api)** - Conéctate a servicios externos mediante solicitudes HTTP

### Bloques de lógica
- **[Condition](/blocks/condition)** - Ramifica rutas del flujo de trabajo basándose en expresiones booleanas
- **[Router](/blocks/router)** - Usa IA para enrutar solicitudes de forma inteligente hacia diferentes rutas
- **[Evaluator](/blocks/evaluator)** - Puntúa y evalúa la calidad del contenido mediante IA

### Bloques de control de flujo
- **[Variables](/blocks/variables)** - Define y gestiona variables con ámbito de flujo de trabajo
- **[Wait](/blocks/wait)** - Pausa la ejecución del flujo de trabajo durante un tiempo determinado

### Bloques de salida
- **[Response](/blocks/response)** - Formatea y devuelve los resultados finales de tu flujo de trabajo

## Cómo funcionan los bloques

Cada bloque tiene tres componentes principales:

**Entradas**: Datos que ingresan al bloque desde otros bloques o desde la entrada del usuario
**Configuración**: Ajustes que controlan el comportamiento del bloque
**Salidas**: Datos que el bloque produce para que otros bloques los utilicen

<Steps>
  <Step>
    <strong>Recibir entrada</strong>: El bloque recibe datos de bloques conectados o de la entrada del usuario
  </Step>
  <Step>

  </Step>
  <Step>

  </Step>

## Conectar Bloques

Los flujos de trabajo se crean conectando bloques entre sí. La salida de un bloque se convierte en la entrada de otro:

- **Arrastrar para conectar**: Arrastra desde un puerto de salida hasta un puerto de entrada
- **Conexiones múltiples**: Una salida puede conectarse a múltiples entradas
- **Rutas ramificadas**: Algunos bloques pueden redirigir a diferentes rutas según condiciones

## Patrones Comunes

### Procesamiento Secuencial
Conecta bloques en cadena, donde cada bloque procesa la salida del anterior:

```
User Input → Agent → Function → Response
```

### Ramificación Condicional
Usa bloques de Condición o Enrutador para crear diferentes rutas:

```
User Input → Router → Agent A (for questions)
                   → Agent B (for commands)
```

### Control de Calidad
Usa bloques Evaluador para valorar y filtrar las salidas:

```
Agent → Evaluator → Condition → Response (if good)
                              → Agent (retry if bad)
```

## Configuración de Bloques

Cada tipo de bloque tiene opciones de configuración específicas:

**Todos los Bloques**:
- Conexiones de entrada/salida
- Comportamiento ante errores
- Configuración de tiempo de espera de ejecución

**Bloques de IA** (Agent, Router, Evaluator):
- Selección de modelo (OpenAI, Anthropic, Google, local)
- Claves de API y autenticación
- Temperatura y otros parámetros del modelo
- Prompts de sistema e instrucciones

**Bloques de Lógica** (Condition, Function):
- Expresiones o código personalizado
- Referencias a variables
- Configuración del entorno de ejecución

**Bloques de Integración** (API, Response):
- Configuración de endpoints
- Encabezados y autenticación
- Formato de solicitud/respuesta
