El bloque de Respuesta es el paso final en tu flujo de trabajo que formatea y envía una respuesta estructurada de vuelta a las llamadas a la API. Es como la instrucción "return" de todo tu flujo de trabajo: empaqueta los resultados y los devuelve al llamante.

## Descripción general

El bloque de Respuesta te permite:

<Steps>
  <Step>
    <strong>Formatear Respuestas de API</strong>: Estructura los resultados del flujo de trabajo en respuestas HTTP adecuadas
  </Step>
  <Step>
    <strong>Establecer Códigos de Estado</strong>: Configura los códigos de estado HTTP apropiados según los resultados del flujo de trabajo
  </Step>
  <Step>
    <strong>Controlar Cabeceras</strong>: Añade cabeceras personalizadas para respuestas de API y webhooks
  </Step>
  <Step>
    <strong>Transformar Datos</strong>: Convierte variables del flujo de trabajo en formatos de respuesta compatibles con el cliente
  </Step>
</Steps>

## Cómo Funciona

El bloque de Respuesta finaliza la ejecución del flujo de trabajo:

1. **Recopilar Datos** - Recoge variables y salidas de bloques anteriores
2. **Formatear Respuesta** - Estructura los datos según su configuración
3. **Configurar Detalles HTTP** - Aplica códigos de estado y encabezados
4. **Enviar Respuesta** - Devuelve la respuesta formateada al llamador de la API

## Cuándo Necesita Bloques de Respuesta

- **Endpoints de API**: Cuando su flujo de trabajo es invocado mediante una API, los bloques de Respuesta formatean los datos de retorno
- **Webhooks**: Devuelve confirmación o datos al sistema que realizó la llamada
- **Pruebas**: Visualice resultados formateados al probar su flujo de trabajo

## Dos Formas de Construir Respuestas

### Modo Constructor (Recomendado)
Interfaz visual para construir la estructura de respuesta:
- Arrastrar y soltar campos
- Referenciar variables del flujo de trabajo fácilmente
- Vista previa visual de la estructura de respuesta

### Modo Editor (Avanzado)
Escriba JSON directamente:
- Control total sobre el formato de respuesta
- Soporte para estructuras anidadas complejas
- Use la sintaxis `<variable.name>` para valores dinámicos

## Opciones de Configuración

### Datos de Respuesta

Los datos de respuesta son el contenido principal que se enviará de vuelta al llamador de la API. Deben estar formateados como JSON y pueden incluir:

- Valores estáticos
- Referencias dinámicas a variables del flujo de trabajo usando la sintaxis `<variable.name>`
- Objetos y arrays anidados
- Cualquier estructura JSON válida

### Código de Estado

Configure el código de estado HTTP para la respuesta. Los códigos de estado más comunes incluyen:

### Encabezados de respuesta

Configure encabezados HTTP adicionales para incluir en la respuesta.

Los encabezados se configuran como pares clave-valor:

| Clave | Valor |
|-----|-------|
| Content-Type | application/json |
| Cache-Control | no-cache |
| X-API-Version | 1.0 |

## Referencias de Variables

Utilice la sintaxis `<variable.name>` para insertar dinámicamente variables del flujo de trabajo en su respuesta:

```json
{
  "user": {
    "id": "<variable.userId>",
    "name": "<variable.userName>",
    "email": "<variable.userEmail>"
  },
  "query": "<variable.searchQuery>",
  "results": "<variable.searchResults>",
  "totalFound": "<variable.resultCount>",
  "processingTime": "<variable.executionTime>ms"
}
```

<Callout type="warning">
  Los nombres de variables distinguen entre mayúsculas y minúsculas y deben coincidir exactamente con las variables disponibles en su flujo de trabajo.
</Callout>

## Mejores Prácticas

- **Utilice códigos de estado significativos**: Elija códigos de estado HTTP apropiados que reflejen con precisión el resultado del flujo de trabajo
- **Estructure sus respuestas de forma coherente**: Mantenga una estructura JSON consistente en todos sus endpoints de API para una mejor experiencia del desarrollador
- **Incluya metadatos relevantes**: Agregue marcas de tiempo e información de versión para facilitar la depuración y el monitoreo
- **Gestione los errores con elegancia**: Utilice lógica condicional en su flujo de trabajo para establecer respuestas de error apropiadas con mensajes descriptivos
- **Valide las referencias de variables**: Asegúrese de que todas las variables referenciadas existan y contengan los tipos de datos esperados antes de que se ejecute el bloque de Respuesta
