El bloque Paralelo es un bloque contenedor en TradingGoose que permite ejecutar múltiples instancias de bloques de forma concurrente para un procesamiento de flujos de trabajo más rápido.

El bloque Paralelo admite dos tipos de ejecución concurrente:

<Callout type="info">
  Los bloques Paralelo son nodos contenedores que ejecutan su contenido múltiples veces de forma simultánea, a diferencia de los bucles que se ejecutan secuencialmente.
</Callout>

## Descripción general

El bloque Paralelo te permite:

<Steps>
  <Step>
    <strong>Distribuir el trabajo</strong>: Procesar múltiples elementos de forma concurrente
  </Step>
  <Step>
    <strong>Acelerar la ejecución</strong>: Ejecutar operaciones independientes simultáneamente
  </Step>
  <Step>
    <strong>Gestionar operaciones masivas</strong>: Procesar grandes conjuntos de datos de manera eficiente
  </Step>
  <Step>
    <strong>Agregar resultados</strong>: Recopilar las salidas de todas las ejecuciones paralelas
  </Step>
</Steps>

## Opciones de Configuración

### Tipo de Paralelismo

Elige entre dos tipos de ejecución en paralelo:

<Tabs items={['Basado en cantidad', 'Basado en colección']}>
  <Tab>
    **Paralelismo basado en cantidad** - Ejecuta un número fijo de instancias en paralelo.
    
    Úsalo cuando necesites ejecutar la misma operación varias veces de forma concurrente.
    

    ```
    Example: Run 5 parallel instances
    - Instance 1 ┐
    - Instance 2 ├─ All execute simultaneously
    - Instance 3 │
    - Instance 4 │
    - Instance 5 ┘
    ```

  </Tab>
  <Tab>
    **Paralelismo basado en colección** - Distribuye una colección entre instancias en paralelo.
    
    Cada instancia procesa un elemento de la colección de forma simultánea.
    

    ```
    Example: Process ["task1", "task2", "task3"] in parallel
    - Instance 1: Process "task1" ┐
    - Instance 2: Process "task2" ├─ All execute simultaneously
    - Instance 3: Process "task3" ┘
    ```

  </Tab>
</Tabs>

## Cómo Usar los Bloques en Paralelo

### Creación de un Bloque en Paralelo

1. Arrastra un bloque Parallel desde la barra de herramientas hasta el lienzo
2. Configura el tipo de paralelismo y los parámetros
3. Arrastra un bloque individual dentro del contenedor paralelo
4. Conecta el bloque según sea necesario

### Acceso a los Resultados

Una vez que un bloque en paralelo finaliza, puedes acceder a los resultados agregados:

- **`<parallel.results>`**: Array de resultados de todas las instancias en paralelo

## Funciones Avanzadas

### Agregación de Resultados

Los resultados de todas las instancias en paralelo se recopilan automáticamente:

```javascript
// In a Function block after the parallel
const allResults = input.parallel.results;
// Returns: [result1, result2, result3, ...]
```

### Aislamiento de Instancias

Cada instancia en paralelo se ejecuta de forma independiente:
- Ámbitos de variables separados
- Sin estado compartido entre instancias
- Los fallos en una instancia no afectan a las demás

### Limitaciones

<Callout type="warning">
  Los bloques contenedores (Bucles y Paralelos) no pueden anidarse entre sí. Esto significa:
  - No se puede colocar un bloque Bucle dentro de un bloque Paralelo
  - No se puede colocar otro bloque Paralelo dentro de un bloque Paralelo
  - No se puede colocar ningún bloque contenedor dentro de otro bloque contenedor
</Callout>

<Callout type="warning">
  Los bloques Paralelos solo pueden contener un único bloque. No es posible tener múltiples bloques conectados entre sí dentro de un paralelo; en ese caso, solo se ejecutaría el primer bloque.
</Callout>

<Callout type="info">
  Si bien la ejecución en paralelo es más rápida, tenga en cuenta:
  - Los límites de frecuencia de API al realizar solicitudes concurrentes
  - El uso de memoria con conjuntos de datos grandes
  - Un máximo de 20 instancias concurrentes para evitar el agotamiento de recursos
</Callout>

## Paralelo vs Bucle

Cuándo utilizar cada uno:

| Característica | Paralelo | Bucle |
|----------------|----------|-------|
| Ejecución | Concurrente | Secuencial |
| Velocidad | Más rápido para operaciones independientes | Más lento pero ordenado |
| Orden | Sin orden garantizado | Mantiene el orden |
| Caso de uso | Operaciones independientes | Operaciones dependientes |
| Uso de recursos | Mayor | Menor |

## Mejores prácticas

- **Solo operaciones independientes**: Asegúrese de que las operaciones no dependan entre sí
- **Gestión de límites de tasa**: Añada retrasos o limitación de velocidad para flujos de trabajo con uso intensivo de API
- **Manejo de errores**: Cada instancia debe gestionar sus propios errores de forma adecuada
