El bloque de Conocimiento conecta tu flujo de trabajo a una base de conocimiento, habilitando la generación aumentada por recuperación (RAG) mediante búsquedas de similitud vectorial, carga de nuevos fragmentos y creación de documentos, todo en tiempo de ejecución.

## Descripción general

<Steps>
  <Step>
    <strong>Buscar</strong>: Ejecuta una consulta de similitud semántica contra tu base de conocimiento y recupera los fragmentos más relevantes
  </Step>
  <Step>
    <strong>Cargar fragmento</strong>: Añade un nuevo fragmento de texto a un documento existente para que sea indexable en consultas futuras
  </Step>
  <Step>
    <strong>Crear documento</strong>: Agrega un documento completamente nuevo (con etiquetas opcionales) a la base de conocimiento
  </Step>
</Steps>

## Operaciones

<Tabs items={['Buscar', 'Cargar fragmento', 'Crear documento']}>
  <Tab>
    Realiza una búsqueda de similitud vectorial y devuelve los fragmentos más relevantes.

    | Campo | Requerido | Descripción |
    |---|---|---|
    | **Base de conocimiento** | Sí | La base de conocimiento en la que buscar |
    | **Consulta de búsqueda** | No | Consulta en lenguaje natural. Opcional cuando se usan solo filtros de etiquetas. |
    | **Número de resultados** | No | Cuántos fragmentos devolver (predeterminado: 10) |
    | **Filtros de etiquetas** | No | Restringe los resultados a documentos que coincidan con etiquetas específicas |

    La consulta de búsqueda es opcional -- puede filtrar únicamente por etiquetas para recuperar todos los fragmentos de una categoría determinada.
  </Tab>
  <Tab>
    Agrega un fragmento de texto a un documento existente dentro de la base de conocimiento. El fragmento se incrusta automáticamente y queda disponible para búsquedas futuras.

    | Campo | Requerido | Descripción |
    |---|---|---|
    | **Base de Conocimiento** | Sí | Base de conocimiento de destino |
    | **Documento** | Sí | El documento al que se agregará el fragmento |
    | **Contenido del Fragmento** | Sí | El contenido de texto a cargar |
  </Tab>
  <Tab>
    Crea un nuevo documento en la base de conocimiento. El contenido se divide en fragmentos, se incrusta y se indexa.

    | Campo | Requerido | Descripción |
    |---|---|---|
    | **Base de Conocimiento** | Sí | Base de conocimiento de destino |
    | **Nombre del Documento** | Sí | Nombre para el nuevo documento |
    | **Contenido del Documento** | Sí | Contenido de texto completo |
    | **Etiquetas del Documento** | No | Etiquetas para categorización y filtrado |
  </Tab>
</Tabs>

## Salidas

| Salida | Tipo | Descripción |
|---|---|---|
| `results` | json | Array de fragmentos coincidentes (búsqueda) o datos de confirmación (operaciones de escritura) |
| `query` | string | La consulta que se utilizó |
| `totalResults` | number | Cantidad de resultados devueltos |

## Mejores Prácticas

- **Usa filtros de etiquetas para acotar las búsquedas** -- las etiquetas te permiten particionar una base de conocimiento por tema, cliente o categoría sin necesidad de crear bases separadas.
- **Mantén los fragmentos enfocados** -- los fragmentos más cortos y temáticamente coherentes producen mejores coincidencias de similitud que los textos largos con múltiples temas.
- **Combina con un bloque de Agente** -- conecta los resultados de búsqueda al contexto de un Agente para construir un pipeline RAG.

<Callout type="info">
  El campo de consulta de búsqueda es opcional. Cuando solo necesitas documentos que coincidan con ciertas etiquetas, deja la consulta vacía y usa filtros de etiquetas en su lugar.
</Callout>
