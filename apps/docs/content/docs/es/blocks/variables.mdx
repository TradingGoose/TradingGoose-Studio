El bloque Variables actualiza las variables del flujo de trabajo durante la ejecución. Las variables deben inicializarse primero en la sección Variables de su flujo de trabajo; luego puede usar este bloque para actualizar sus valores mientras el flujo de trabajo se ejecuta.

El bloque Variables le permite:

## Cómo usar Variables

### 1. Inicializar en Variables del flujo de trabajo

Primero, cree sus variables en la sección Variables del flujo de trabajo (accesible desde la configuración del flujo de trabajo):

```
customerEmail = ""
retryCount = 0
currentStatus = "pending"
```

### 2. Actualizar con el bloque Variables

Use el bloque Variables para actualizar estos valores durante la ejecución:

```
customerEmail = <api.email>
retryCount = <variable.retryCount> + 1
currentStatus = "processing"
```

### 3. Acceder desde cualquier lugar

Referencie variables en cualquier bloque:

```
Agent prompt: "Send email to <variable.customerEmail>"
Condition: <variable.retryCount> < 5
API body: {"status": "<variable.currentStatus>"}
```

## Salidas

- **`<variables.assignments>`**: Objeto JSON con todas las asignaciones de variables de este bloque

## Buenas prácticas

- **Inicializar en la configuración del flujo de trabajo**: Crea siempre las variables en la sección Variables del flujo de trabajo antes de utilizarlas
- **Actualizar dinámicamente**: Usa bloques de Variables para actualizar valores según las salidas o cálculos de otros bloques
- **Usar en bucles**: Ideal para rastrear el estado entre iteraciones
- **Nombrar de forma descriptiva**: Usa nombres claros como `currentIndex`, `totalProcessed` o `lastError`
