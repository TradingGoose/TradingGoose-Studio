El bloque Router utiliza IA para decidir de forma inteligente qué camino debe seguir tu flujo de trabajo, enrutando la ejecución según condiciones o lógica específicas. A diferencia de los bloques de Condición, que aplican reglas simples, los bloques Router pueden comprender el contexto y tomar decisiones de enrutamiento inteligentes basadas en el análisis del contenido.

## Descripción general

El bloque Router te permite:

<Steps>
  <Step>
    <strong>Enrutamiento inteligente de contenido</strong>: Usa IA para comprender la intención y el contexto
  </Step>
  <Step>
    <strong>Selección dinámica de rutas</strong>: Enruta flujos de trabajo basándose en el análisis de contenido no estructurado
  </Step>
  <Step>
    <strong>Decisiones contextualmente conscientes</strong>: Toma decisiones de enrutamiento inteligentes más allá de las reglas simples
  </Step>
  <Step>
    <strong>Gestión de múltiples rutas</strong>: Maneja flujos de trabajo complejos con múltiples destinos potenciales
  </Step>
</Steps>

## Router vs Bloques de Condición

<Accordions>
  <Accordion title="Cuándo usar Router">
    - Análisis de contenido con IA requerido
    - Contenido no estructurado o de tipo variable
    - Enrutamiento basado en intención (p. ej., "enrutar tickets de soporte a departamentos")
    - Toma de decisiones contextualmente consciente requerida
  </Accordion>
  <Accordion title="Cuándo usar Condición">
    - Decisiones simples basadas en reglas
    - Datos estructurados o comparaciones numéricas
    - Enrutamiento rápido y determinista requerido
    - Lógica booleana suficiente
  </Accordion>
</Accordions>

## Cómo Funciona

El bloque Router:

<Steps>
  <Step>
    <strong>Analizar el contenido</strong>: Utiliza un LLM para comprender el contenido de entrada y su contexto
  </Step>
  <Step>
    <strong>Evaluar destinos</strong>: Compara el contenido con los bloques de destino disponibles
  </Step>
  <Step>
    <strong>Seleccionar destino</strong>: Identifica la ruta más adecuada según la intención
  </Step>
  <Step>
    <strong>Ejecutar enrutamiento</strong>: Dirige el flujo de trabajo al bloque seleccionado
  </Step>
</Steps>

## Opciones de Configuración

### Contenido/Prompt

El contenido o prompt que el Router analizará para tomar decisiones de enrutamiento. Puede ser:

- Una consulta o entrada directa del usuario
- La salida de un bloque anterior
- Un mensaje generado por el sistema

### Bloques de Destino

Los posibles bloques de destino que el Router puede seleccionar. El Router detectará automáticamente los bloques conectados, pero también puede:

- Personalizar las descripciones de los bloques de destino para mejorar la precisión del enrutamiento
- Especificar criterios de enrutamiento para cada bloque de destino
- Excluir ciertos bloques de ser considerados como destinos de enrutamiento

### Selección de Modelo

Elija un modelo de IA para impulsar la decisión de enrutamiento:

**OpenAI**: GPT-4o, o1, o3, o4-mini, gpt-4.1  \
**Anthropic**: Claude 3.7 Sonnet \
**Google**: Gemini 2.5 Pro, Gemini 2.0 Flash \
**Other Providers**: Groq, Cerebras, xAI, DeepSeek \
**Local Models**: Any model running on Ollama 

**Recomendación**: Utilice modelos con sólidas capacidades de razonamiento, como GPT-4o o Claude 3.7 Sonnet, para obtener decisiones de enrutamiento más precisas.

### Clave API

Su clave API para el proveedor de LLM seleccionado. Se almacena de forma segura y se utiliza para la autenticación.

### Acceso a los Resultados

Después de que un router toma una decisión, puede acceder a sus salidas:

- **`<router.prompt>`**: Resumen del prompt de enrutamiento utilizado
- **`<router.selected_path>`**: Detalles del bloque de destino seleccionado
- **`<router.tokens>`**: Estadísticas de uso de tokens del LLM
- **`<router.cost>`**: Resumen de costes de la llamada de enrutamiento (entrada, salida, total)
- **`<router.model>`**: El modelo utilizado para la toma de decisiones

## Mejores Prácticas

- **Proporcione descripciones claras de los destinos**: Ayude al Router a entender cuándo seleccionar cada destino con descripciones específicas y detalladas
- **Utilice criterios de enrutamiento específicos**: Defina condiciones claras y ejemplos para cada ruta con el fin de mejorar la precisión
- **Implemente rutas de respaldo**: Conecte un destino predeterminado para cuando ninguna ruta específica sea la adecuada
- **Pruebe con entradas diversas**: Asegúrese de que el Router gestione distintos tipos de entrada, casos límite y contenido inesperado
- **Monitoree el rendimiento del enrutamiento**: Revise las decisiones de enrutamiento con regularidad y refine los criterios según los patrones de uso real
- **Elija modelos apropiados**: Utilice modelos con sólidas capacidades de razonamiento para decisiones de enrutamiento complejas
