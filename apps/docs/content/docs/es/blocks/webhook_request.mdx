El bloque de Solicitud Webhook envía una solicitud HTTP POST **saliente** a una URL de webhook externa. Úsalo para enviar los resultados del flujo de trabajo a servicios de terceros como Zapier, Make, n8n, o cualquier endpoint personalizado que acepte payloads de webhook.

<Callout type="warning">
  Este bloque envía llamadas webhook **salientes** (tu flujo de trabajo llama a una URL externa). Es diferente del **Webhook Trigger**, que escucha solicitudes **entrantes** que inician un flujo de trabajo.
</Callout>

## Descripción general

<Steps>
  <Step>
    <strong>Configura la URL de destino</strong>: Apunta a cualquier endpoint que acepte solicitudes HTTP POST
  </Step>
  <Step>
    <strong>Construye el payload JSON</strong>: Incluye datos de bloques anteriores usando referencias de variables como `<agent.content>`
  </Step>
  <Step>
    <strong>Firma la solicitud opcionalmente</strong>: Proporciona un secreto de firma HMAC-SHA256 para que el receptor pueda verificar la autenticidad
  </Step>
  <Step>
    <strong>Agrega encabezados personalizados</strong>: Incluye claves de API, tipos de contenido o cualquier encabezado adicional que requiera el receptor
  </Step>
</Steps>

## Cómo funciona

Cuando el bloque se ejecuta:

1. Resuelve cualquier referencia de variable (`<block.output>`) en la URL, el payload y los encabezados.
2. Serializa el payload como JSON.
3. Si se proporciona un **Secreto de Firma**, calcula una firma HMAC-SHA256 sobre el payload y la adjunta como encabezado webhook para que el servidor receptor pueda verificar que la solicitud no fue alterada.
4. Envía la solicitud POST con `Content-Type: application/json` y cualquier encabezado adicional que hayas configurado.
5. Devuelve los datos de respuesta, el código de estado y los encabezados de respuesta como salidas del bloque.

### Campos

| Campo | Requerido | Descripción |
|---|---|---|
| **URL de Webhook** | Sí | El endpoint de destino. Debe aceptar HTTP POST. |
| **Payload** | No | Cuerpo JSON a enviar. Use `<block.output>` para referenciar datos del paso anterior. |
| **Secreto de Firma** | No | Un secreto compartido utilizado para generar un encabezado de firma HMAC-SHA256 para la verificación del payload. |
| **Encabezados Adicionales** | No | Pares clave-valor añadidos a la solicitud (p. ej., `Authorization: Bearer ...`). |

## Salidas

| Salida | Tipo | Descripción |
|---|---|---|
| `data` | json | El cuerpo de respuesta analizado del endpoint externo |
| `status` | number | Código de estado HTTP devuelto por el endpoint |
| `headers` | json | Encabezados de respuesta |

## Mejores Prácticas

- **Almacene los secretos en variables de entorno** -- referencie su secreto de firma con `{{WEBHOOK_SECRET}}` en lugar de pegarlo directamente.
- **Verifique la salida de estado** -- use un bloque de Condición o Router después del webhook para gestionar correctamente las respuestas que no sean 2xx.
- **Mantenga los payloads pequeños** -- envíe solo los datos que el receptor necesita. Los payloads grandes aumentan la latencia y pueden ser rechazados por algunos endpoints.
