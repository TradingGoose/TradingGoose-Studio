El bloque Condición permite bifurcar la ruta de ejecución del flujo de trabajo basándose en expresiones booleanas, lo que le permite crear flujos de trabajo dinámicos y adaptables con diferentes rutas de ejecución. Evalúa las condiciones y enruta el flujo de trabajo en consecuencia, permitiéndole controlar el flujo de ejecución en función de datos o lógica sin necesidad de un LLM.

## Descripción general

El bloque Condición le permite:

<Steps>
  <Step>
    <strong>Crear lógica de bifurcación</strong>: Enrutar flujos de trabajo basándose en expresiones booleanas
  </Step>
  <Step>
    <strong>Tomar decisiones basadas en datos</strong>: Evaluar condiciones utilizando las salidas de bloques anteriores
  </Step>
  <Step>
    <strong>Gestionar múltiples escenarios</strong>: Definir múltiples condiciones con diferentes rutas
  </Step>
  <Step>
    <strong>Proporcionar enrutamiento determinista</strong>: Tomar decisiones sin necesidad de un LLM
  </Step>
</Steps>

## Cómo funciona

El bloque Condición opera a través de un proceso de evaluación secuencial:

1. **Evaluar expresión** - Procesa la expresión booleana JavaScript/TypeScript utilizando los datos actuales del flujo de trabajo
2. **Determinar resultado** - Devuelve verdadero o falso según la evaluación de la expresión
3. **Enrutar flujo de trabajo** - Dirige la ejecución al bloque de destino apropiado según el resultado
4. **Proporcionar contexto** - Genera metadatos sobre la decisión para depuración y monitoreo

## Opciones de configuración

### Condiciones

Defina una o más condiciones que serán evaluadas. Cada condición incluye:

- **Expresión**: Una expresión de JavaScript/TypeScript que evalúa a verdadero o falso
- **Ruta**: El bloque de destino al que enrutar si la condición es verdadera
- **Descripción**: Explicación opcional de lo que verifica la condición

Puede crear múltiples condiciones que se evalúan en orden, donde la primera condición que coincida determina la ruta de ejecución.

### Formato de Expresión de Condición

Las condiciones utilizan sintaxis de JavaScript y pueden referenciar valores de entrada de bloques anteriores.

### Acceso a Resultados

Una vez que se evalúa una condición, puede acceder a sus salidas:

- **`<condition.result>`**: Resultado booleano de la evaluación de la condición
- **`<condition.matched_condition>`**: ID de la condición que coincidió
- **`<condition.content>`**: Descripción del resultado de la evaluación
- **`<condition.path>`**: Detalles del destino de enrutamiento seleccionado

## Funciones Avanzadas

### Expresiones Complejas

Utilice operadores y funciones de JavaScript en las condiciones:

```javascript
// String operations
<user.email>.endsWith('@company.com')

// Array operations
<api.tags>.includes('urgent')

// Mathematical operations
<agent.confidence> * 100 > 85

// Date comparisons
new Date(<api.created_at>) > new Date('2024-01-01')
```

### Evaluación de Condiciones Múltiples

Las condiciones se evalúan en orden hasta que una coincida:

```javascript
// Condition 1: Check for high priority
<ticket.priority> === 'high'

// Condition 2: Check for urgent keywords
<ticket.subject>.toLowerCase().includes('urgent')

// Condition 3: Default fallback
true
```

### Manejo de Errores

Las condiciones gestionan automáticamente:
- Valores indefinidos o nulos con evaluación segura
- Discrepancias de tipos con alternativas apropiadas
- Expresiones inválidas con registro de errores
- Variables faltantes con valores predeterminados

## Mejores Prácticas

- **Ordena las condiciones correctamente**: Coloca las condiciones más específicas antes que las generales para garantizar que la lógica específica tenga precedencia sobre los valores de reserva
- **Incluye una condición predeterminada**: Agrega una condición comodín (`true`) como última condición para gestionar los casos no coincidentes y evitar que la ejecución del flujo de trabajo quede bloqueada
- **Mantén las expresiones simples**: Utiliza expresiones booleanas claras y directas para mejorar la legibilidad y facilitar la depuración
- **Documenta tus condiciones**: Añade descripciones para explicar el propósito de cada condición y facilitar la colaboración del equipo y el mantenimiento
- **Prueba los casos límite**: Verifica que las condiciones gestionen correctamente los valores límite probando con valores en los extremos de tus rangos de condición
