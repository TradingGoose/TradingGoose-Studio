El bloque Nota es una anotación exclusivamente visual que reside en el lienzo del flujo de trabajo. No se ejecuta, no tiene entradas ni salidas, y el ejecutor lo ignora por completo. Úsalo para documentar decisiones, dejar instrucciones a colaboradores o explicar secciones complejas de un flujo de trabajo.

## Descripción general

Los bloques Nota cumplen una función puramente organizativa:

- **Documentar decisiones de diseño** -- explica por qué se eligió una determinada estrategia de ramificación o modelo.
- **Dejar instrucciones** -- indica a los colaboradores cómo configurar las variables de entorno o qué hace una sección del flujo de trabajo.
- **Insertar referencias** -- pega enlaces de YouTube y se renderizarán como videos incrustados directamente en el lienzo.

<Callout type="info">
  Los bloques Nota no tienen ningún efecto sobre la ejecución del flujo de trabajo. No producen salidas y no pueden conectarse a otros bloques. Se eliminan por completo cuando el flujo de trabajo se ejecuta.
</Callout>

## Configuración

El único campo es un área de texto en Markdown. Escribe texto sin formato, usa el formato Markdown (encabezados, listas, negrita, bloques de código) o pega una URL de YouTube para incrustar un video.

## Mejores prácticas

- **Coloca las notas cerca de los bloques que describen** para que el contexto sea inmediatamente visible.
- **Usa encabezados y listas de Markdown** para mantener las notas más extensas fáciles de escanear.
- **Prefiere las notas sobre los comentarios en los bloques de código** -- las notas son visibles en el nivel del lienzo sin necesidad de abrir la configuración de un bloque.
