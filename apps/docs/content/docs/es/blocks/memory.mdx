El bloque de Memoria almacena y recupera mensajes de conversación que persisten entre ejecuciones de flujos de trabajo. Está diseñado para funcionar junto con los bloques de Agente, proporcionando a sus agentes un contexto conversacional a largo plazo que se mantiene entre ejecuciones.

## Descripción general

<Steps>
  <Step>
    <strong>Agregar memoria</strong>: Almacena un mensaje con un rol (usuario, asistente o sistema) bajo un ID de memoria designado
  </Step>
  <Step>
    <strong>Obtener memoria</strong>: Recupera una entrada de memoria individual por su ID
  </Step>
  <Step>
    <strong>Obtener todas las memorias</strong>: Obtiene todas las memorias almacenadas para el flujo de trabajo actual
  </Step>
  <Step>
    <strong>Eliminar memoria</strong>: Elimina una entrada de memoria por ID
  </Step>
</Steps>

## Cómo Funciona

Las entradas de memoria se almacenan como mensajes en formato de chat con tres campos obligatorios: un **ID** para delimitar el alcance de la memoria, un **Rol** (`user`, `assistant` o `system`) y el texto de **Contenido**. Cuando posteriormente recuperas las memorias y las pasas a la entrada de Memorias de un bloque Agente, el agente las interpreta como turnos de conversación previos, lo que permite el diálogo multi-turno a través de ejecuciones separadas.

Un flujo de trabajo conversacional típico utiliza tres bloques de Memoria alrededor de un Agente:

1. **Memoria (Agregar, rol: usuario)** -- guarda el mensaje entrante del usuario antes de que se ejecute el Agente
2. **Agente** -- procesa el mensaje utilizando las memorias recuperadas como contexto
3. **Memoria (Agregar, rol: asistente)** -- guarda la respuesta del Agente después de su ejecución

<Callout type="info">
  Los IDs de memoria delimitan el alcance de la conversación. Usa el mismo ID en distintas ejecuciones para construir un historial de conversación continuo, o usa IDs diferentes para mantener hilos de conversación separados.
</Callout>

  </Tab>

## Salidas

| Salida | Tipo | Descripción |
|---|---|---|
| `memories` | json | Los datos de memoria recuperados (array de mensajes) |
| `id` | string | El identificador de memoria sobre el que se operó |

## Mejores Prácticas

- **Usa IDs consistentes** -- el ID de memoria es la clave que agrupa los mensajes en una conversación. Utiliza un identificador estable (como un ID de usuario o ID de sesión) para que los recuerdos se acumulen correctamente entre ejecuciones.
- **Almacena ambos lados de la conversación** -- añade un mensaje con rol `user` antes del Agente y un mensaje con rol `assistant` después de él para que el diálogo completo quede preservado.
- **Limpia cuando sea necesario** -- usa la operación Eliminar para borrar conversaciones obsoletas y evitar un crecimiento ilimitado de la memoria.
