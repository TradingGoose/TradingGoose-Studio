El bloque Loop es un contenedor que ejecuta los bloques en su interior de forma repetida. No es un bloque regular con su propio archivo fuente -- es un nodo contenedor especial gestionado por el `LoopBlockHandler` del ejecutor. Arrastra bloques al contenedor del loop y se ejecutarán una vez por iteración.

<Callout type="info">
  Los bloques Loop son nodos contenedores. Arrastra otros bloques dentro de ellos en el lienzo para definir qué se ejecuta en cada iteración.
</Callout>

## Tipos de Loop

<Tabs items={['For', 'ForEach', 'While', 'Do-While']}>
  <Tab>
    **Bucle For** -- se ejecuta un número fijo de veces. El número de iteraciones predeterminado es **5** si no se especifica.

    ```
    Example: Run 3 times
    - Iteration 1 (index 0)
    - Iteration 2 (index 1)
    - Iteration 3 (index 2)
    ```

  </Tab>
  <Tab>
    **Bucle ForEach** -- itera sobre cada elemento de un array u objeto. La colección debe ser no vacía; una colección vacía genera un error.

    ```
    Example: Process ["apple", "banana", "orange"]
    - Iteration 1: currentItem = "apple"
    - Iteration 2: currentItem = "banana"
    - Iteration 3: currentItem = "orange"
    ```

    Para objetos, cada iteración recibe un par `[key, value]`.
  </Tab>
  <Tab>
    **Bucle While** -- evalúa una condición **antes** de cada iteración. Si la condición es falsa en la primera verificación, el cuerpo nunca se ejecuta.

    Los bucles While y Do-While **no tienen límite de iteraciones incorporado**. Dependen de que la condición se vuelva falsa (o del tiempo de espera del flujo de trabajo) para detenerse.
  </Tab>
  <Tab>
    **Bucle Do-While** -- ejecuta el cuerpo al menos una vez y luego evalúa la condición **después** de cada iteración. Útil cuando se necesita al menos un ciclo antes de realizar la verificación.
  </Tab>
</Tabs>

## Cómo Funciona

El `LoopBlockHandler` del ejecutor gestiona el control de iteración:

1. **Inicializar** -- el contador del bucle comienza en 1. Para los bucles `forEach`, la colección se resuelve (JSON, referencias a variables o expresiones).
2. **Verificar continuación** -- `for`/`forEach` comparan el contador con la longitud de la colección o el número de iteraciones. `while`/`doWhile` evalúan la expresión de condición utilizando el mismo motor que el bloque de Condición.
3. **Activar elementos secundarios** -- los bloques conectados al manejador de inicio interno del bucle se colocan en la ruta de ejecución activa.
4. **Recopilar resultados** -- las salidas de cada iteración se agregan en `loop.results`.
5. **Incrementar y repetir** -- el contador avanza y el paso 2 se ejecuta nuevamente. Cuando el bucle finaliza, se activan los bloques conectados al manejador de salida del bucle.

## Variables de Iteración

Dentro del cuerpo del bucle, las siguientes variables están disponibles:

| Variable | Tipo | Descripción |
|---|---|---|
| `loop.index` | number | Índice de base cero de la iteración actual |
| `loop.currentItem` | any | El elemento actual (bucles ForEach) o el contexto del bucle (bucles While) |
| `loop.items` | array/object | La colección completa que se está iterando (solo ForEach) |

Una vez completado el bucle, los bloques posteriores pueden acceder a:

| Variable | Tipo | Descripción |
|---|---|---|
| `loop.results` | array | Array ordenado de todas las salidas de iteración |

## Configuración

| Parámetro | Aplica a | Descripción |
|---|---|---|
| **Tipo de Bucle** | Todos | `for`, `forEach`, `while`, o `doWhile` |
| **Iteraciones** | For | Número de veces a repetir (predeterminado: 5) |
| **Colección** | ForEach | Array u objeto a iterar. Admite literales JSON, referencias `<block.output>` y expresiones. |
| **Condición** | While / Do-While | Una expresión booleana evaluada en cada iteración (p. ej., `<variable.i> < 10`) |

## Casos de Uso de Ejemplo

### Procesamiento de Resultados de API

1. Un bloque de API obtiene una lista de registros de clientes.
2. Un bucle ForEach itera sobre cada cliente.
3. Dentro del bucle, un Agente analiza los datos del cliente y una Función almacena el resultado.
4. Tras el bucle, un bloque de Respuesta devuelve el `loop.results` agregado.

### Refinamiento Iterativo

1. Un bucle For configurado en 5 iteraciones genera variaciones de contenido.
2. Dentro del bucle, un Agente produce un borrador y un Evaluador lo puntúa.
3. Tras el bucle, una Función selecciona la variación con mayor puntuación de `loop.results`.

## Limitaciones

<Callout type="warning">
  Los bloques contenedores (Bucles y Paralelos) **no pueden anidarse** entre sí. No es posible colocar un Bucle dentro de otro Bucle, ni un Paralelo dentro de un Bucle. Si necesita iteración multidimensional, reestructure su flujo de trabajo para usar bucles secuenciales o procesar los datos por etapas.
</Callout>

<Callout type="info">
  Los bucles ejecutan las iteraciones **secuencialmente**. Para la ejecución concurrente de tareas independientes, utilice el bloque Paralelo.
</Callout>

## Buenas Prácticas

- **Establezca conteos de iteración razonables** en los bucles For para evitar flujos de trabajo de larga duración. El valor predeterminado es 5.
- **Use ForEach para colecciones** -- se ajusta automáticamente al tamaño de la colección y le proporciona `currentItem`.
- **Actualice siempre la variable de condición** dentro de los bucles While/Do-While. Olvidar modificar la variable de la que depende la condición crea un bucle infinito (limitado únicamente por el tiempo de espera del flujo de trabajo).
