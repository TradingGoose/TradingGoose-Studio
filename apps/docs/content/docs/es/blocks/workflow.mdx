Añade un bloque de Flujo de Trabajo cuando desees invocar un flujo de trabajo hijo como parte de un flujo mayor. El bloque ejecuta la última versión desplegada de ese flujo de trabajo, espera a que finalice y luego continúa con el flujo principal.

## Configuración

1. **Selecciona un flujo de trabajo** en el menú desplegable (las autorreferencias están bloqueadas para evitar bucles).
2. **Mapea las entradas**: Si el flujo de trabajo hijo tiene un disparador de Formulario de Entrada, verás cada campo y podrás conectar variables del flujo principal. Los valores mapeados son los que recibe el flujo hijo.
3. **Salidas**: Una vez que el flujo hijo finaliza, el bloque expone:
   - `result` – la respuesta final del flujo de trabajo hijo
   - `success` – si se ejecutó sin errores
   - `error` – mensaje cuando la ejecución falla

## Notas de Ejecución

- Los flujos de trabajo hijo se ejecutan en el mismo contexto del espacio de trabajo, por lo que las variables de entorno y las herramientas se transfieren.
- El bloque utiliza versionado de despliegue: cualquier ejecución mediante API, programación, webhook, manual o chat invoca el snapshot desplegado. Vuelve a desplegar el flujo hijo cuando lo modifiques.
- Si el flujo hijo falla, el bloque genera un error a menos que lo gestiones en etapas posteriores.

<Callout>
Mantén los flujos de trabajo secundarios enfocados. Los flujos pequeños y reutilizables facilitan su combinación sin crear anidamientos profundos.
</Callout>
