## Conceptos clave

### Ejecución topológica
Los bloques se ejecutan en orden de dependencia, de forma similar a cómo una hoja de cálculo recalcula las celdas. El motor de ejecución determina automáticamente qué bloques pueden ejecutarse en función de las dependencias completadas.

### Seguimiento de rutas
El motor realiza un seguimiento activo de las rutas de ejecución a través de tu flujo de trabajo. Los bloques de enrutador y condición actualizan dinámicamente estas rutas, garantizando que solo se ejecuten los bloques relevantes.

### Procesamiento por capas
En lugar de ejecutar los bloques uno a uno, el motor identifica capas de bloques que pueden ejecutarse en paralelo, optimizando el rendimiento para flujos de trabajo complejos.

### Contexto de ejecución
Cada flujo de trabajo mantiene un contexto enriquecido durante la ejecución que contiene:
- Salidas y estados de los bloques
- Rutas de ejecución activas
- Seguimiento de iteraciones en bucle y en paralelo
- Variables de entorno
- Decisiones de enrutamiento

## Disparadores de ejecución

Los flujos de trabajo pueden ejecutarse a través de múltiples canales:

- **Manual**: Prueba y depura directamente en el editor
- **Implementar como API**: Crea un endpoint HTTP protegido con claves de API
- **Implementar como Chat**: Crea una interfaz conversacional en un subdominio personalizado
- **Webhooks**: Responde a eventos externos de servicios de terceros
- **Programado**: Ejecuta según un calendario recurrente mediante expresiones cron

### Implementar como API

Cuando implementas un flujo de trabajo como API, TradingGoose:
- Crea un endpoint HTTP único: `https://tradinggoose.ai/api/workflows/{workflowId}/execute`
- Genera una clave de API para autenticación
- Acepta solicitudes POST con cargas JSON
- Devuelve los resultados de ejecución del flujo de trabajo como JSON

Ejemplo de llamada a la API:

```bash
curl -X POST https://tradinggoose.ai/api/workflows/your-workflow-id/execute \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"input": "your data here"}'
```

### Implementar como Chat

La implementación como Chat crea una interfaz conversacional para tu flujo de trabajo:
- Alojada en un subdominio personalizado: `https://your-name.tradinggoose.ai`
- Autenticación opcional (pública, por contraseña o basada en correo electrónico)
- Interfaz de usuario personalizable con tu identidad de marca
- Respuestas en streaming para interacción en tiempo real
- Ideal para asistentes de IA, bots de soporte o herramientas interactivas

Cada método de implementación pasa los datos al bloque inicial de tu flujo de trabajo, iniciando el flujo de ejecución.

## Instantáneas de Implementación

Todos los puntos de entrada públicos —API, Chat, Programado, Webhook y ejecuciones manuales— ejecutan la instantánea de implementación activa del flujo de trabajo. Publica una nueva implementación cada vez que modifiques el canvas para que todos los disparadores utilicen la versión actualizada.

El modal de implementación mantiene un historial completo de versiones en una tabla de instantáneas de implementación. Puedes inspeccionar cualquier instantánea, compararla con tu borrador, y promoverla o revertirla con un solo clic cuando necesites restaurar una versión anterior.

## Ejecución Programática

Ejecuta flujos de trabajo desde tus aplicaciones utilizando nuestros SDKs oficiales:

```bash
# TypeScript/JavaScript
npm install tradinggoose-ts-sdk

# Python
pip install tradinggoose-sdk
```

```typescript
// TypeScript Example
import { TradingGooseClient } from 'tradinggoose-ts-sdk';

const client = new TradingGooseClient({ 
  apiKey: 'your-api-key' 
});

const result = await client.executeWorkflow('workflow-id', {
  input: { message: 'Hello' }
});
```

## Mejores Prácticas

### Diseño para la Confiabilidad
- Maneja los errores de forma controlada con rutas de contingencia apropiadas
- Utiliza variables de entorno para datos sensibles
- Añade registros en los bloques de Función para depuración

### Optimización del Rendimiento
- Minimiza las llamadas a APIs externas cuando sea posible
- Utiliza la ejecución en paralelo para operaciones independientes
- Almacena resultados en caché con bloques de Variables cuando sea conveniente

### Supervisión de Ejecuciones
- Revisa los registros regularmente para comprender los patrones de rendimiento
- Realiza un seguimiento de los costos del uso de modelos de IA
- Utiliza instantáneas del flujo de trabajo para depurar problemas
