TradingGoose proporciona una API externa completa para consultar registros de ejecuciﾃｳn de flujos de trabajo y configurar webhooks para notificaciones en tiempo real cuando los flujos de trabajo se completan.

## Autenticaciﾃｳn

Todas las solicitudes a la API requieren una clave de API enviada en el encabezado `x-api-key`:

```bash
curl -H "x-api-key: YOUR_API_KEY" \
  https://tradinggoose.ai/api/v1/logs?workspaceId=YOUR_WORKSPACE_ID
```

Puedes generar claves de API desde la configuraciﾃｳn de usuario en el panel de TradingGoose.

## API de Registros

Todas las respuestas de la API incluyen informaciﾃｳn sobre los lﾃｭmites de ejecuciﾃｳn de flujos de trabajo y el uso actual:

```json
"limits": {
  "workflowExecutionRateLimit": {
    "sync": {
      "limit": 60,        // Max sync workflow executions per minute
      "remaining": 58,    // Remaining sync workflow executions
      "resetAt": "..."    // When the window resets
    },
    "async": {
      "limit": 60,        // Max async workflow executions per minute
      "remaining": 59,    // Remaining async workflow executions
      "resetAt": "..."    // When the window resets
    }
  },
  "usage": {
    "currentPeriodCost": 1.234,  // Current billing period usage in USD
    "limit": 10,                  // Usage limit in USD
    "plan": "pro",                // Current subscription plan
    "isExceeded": false           // Whether limit is exceeded
  }
}
```

**Nota:** Los lﾃｭmites de frecuencia en el cuerpo de la respuesta corresponden a las ejecuciones de flujos de trabajo. Los lﾃｭmites de frecuencia para llamar a este endpoint de la API se encuentran en los encabezados de respuesta (`X-RateLimit-*`).

## Suscripciones a Webhooks

Recibe notificaciones en tiempo real cuando las ejecuciones de flujos de trabajo finalicen. Los webhooks se configuran a travﾃｩs de la interfaz de TradingGoose en el editor de flujos de trabajo.

### Configuraciﾃｳn

Los webhooks pueden configurarse para cada flujo de trabajo a travﾃｩs de la interfaz del editor. Haz clic en el ﾃｭcono de webhook en la barra de control para abrir el panel de configuraciﾃｳn de webhooks, donde puedes agregar, editar y gestionar tus suscripciones.

**Opciones de Configuraciﾃｳn Disponibles:**
- `url`: URL del endpoint de tu webhook
- `secret`: Secreto opcional para verificaciﾃｳn de firma HMAC
- `includeFinalOutput`: Incluye la salida final del flujo de trabajo en el payload
- `includeTraceSpans`: Incluye trazas de ejecuciﾃｳn detalladas
- `includeRateLimits`: Incluye informaciﾃｳn de lﾃｭmite de tasa del propietario del flujo de trabajo
- `includeUsageData`: Incluye datos de uso y facturaciﾃｳn del propietario del flujo de trabajo
- `levelFilter`: Array de niveles de log a recibir (`info`, `error`)
- `triggerFilter`: Array de tipos de disparador a recibir (`api`, `webhook`, `schedule`, `manual`, `chat`)
- `active`: Habilitar/deshabilitar la suscripciﾃｳn al webhook

### Encabezados del Webhook

Cada solicitud de webhook incluye estos encabezados:

- `tradinggoose-event`: Tipo de evento (siempre `workflow.execution.completed`)
- `tradinggoose-timestamp`: Marca de tiempo Unix en milisegundos
- `tradinggoose-delivery-id`: ID de entrega ﾃｺnico para idempotencia
- `tradinggoose-signature`: Firma HMAC-SHA256 para verificaciﾃｳn (si se configura un secreto)
- `Idempotency-Key`: Igual que el ID de entrega para detecciﾃｳn de duplicados

Si configura un secreto de webhook, verifique la firma para asegurarse de que el webhook proviene de TradingGoose:

    import crypto from 'crypto';

    function verifyWebhookSignature(body, signature, secret) {
      const [timestampPart, signaturePart] = signature.split(',');
      const timestamp = timestampPart.replace('t=', '');
      const expectedSignature = signaturePart.replace('v1=', '');
      
      const signatureBase = `${timestamp}.${body}`;
      const hmac = crypto.createHmac('sha256', secret);

### Polﾃｭtica de Reintentos

Las entregas de webhook fallidas se reintentan con retroceso exponencial y jitter:

- Intentos mﾃ｡ximos: 5
- Demoras de reintento: 5 segundos, 15 segundos, 1 minuto, 3 minutos, 10 minutos
- Jitter: Hasta un 10% de demora adicional para prevenir el efecto thundering herd
- Solo las respuestas HTTP 5xx y 429 desencadenan reintentos
- Las entregas expiran despuﾃｩs de 30 segundos

<Callout type="info">
  Las entregas de webhook se procesan de forma asﾃｭncrona y no afectan el rendimiento de ejecuciﾃｳn del flujo de trabajo.
</Callout>

## Buenas Prﾃ｡cticas

1. **Estrategia de Sondeo**: Al sondear registros, utilice paginaciﾃｳn basada en cursor con `order=asc` y `startDate` para obtener nuevos registros de manera eficiente.

2. **Seguridad de Webhook**: Configure siempre un secreto de webhook y verifique las firmas para garantizar que las solicitudes provienen de TradingGoose.

3. **Idempotencia**: Utilice el encabezado `Idempotency-Key` para detectar y gestionar entregas de webhook duplicadas.

4. **Privacidad**: De forma predeterminada, `finalOutput` y `traceSpans` se excluyen de las respuestas. Habilite estos campos solo si necesita los datos y comprende las implicaciones de privacidad.

5. **Limitaciﾃｳn de Tasa**: Implemente retroceso exponencial cuando reciba respuestas 429. Consulte el encabezado `Retry-After` para conocer el tiempo de espera recomendado.

## Limitaciﾃｳn de Tasa

La API implementa limitaciﾃｳn de tasa para garantizar un uso equitativo:

- **Plan gratuito**: 10 solicitudes por minuto
- **Plan Pro**: 30 solicitudes por minuto
- **Plan Equipo**: 60 solicitudes por minuto
- **Plan Empresarial**: Lﾃｭmites personalizados

La informaciﾃｳn sobre lﾃｭmites de velocidad se incluye en los encabezados de respuesta:
- `X-RateLimit-Limit`: Mﾃ｡ximo de solicitudes por ventana
- `X-RateLimit-Remaining`: Solicitudes restantes en la ventana actual
- `X-RateLimit-Reset`: Marca de tiempo ISO cuando se reinicia la ventana

## Ejemplo: Sondeo de nuevos registros

```javascript
let cursor = null;
const workspaceId = 'YOUR_WORKSPACE_ID';
const startDate = new Date().toISOString();

async function pollLogs() {
  const params = new URLSearchParams({
    workspaceId,
    startDate,
    order: 'asc',
    limit: '100'
  });
  
  if (cursor) {
    params.append('cursor', cursor);
  }
  
  const response = await fetch(
    `https://tradinggoose.ai/api/v1/logs?${params}`,
    {
      headers: {
        'x-api-key': 'YOUR_API_KEY'
      }
    }
  );
  
  if (response.ok) {
    const data = await response.json();
    
    // Process new logs
    for (const log of data.data) {
      console.log(`New execution: ${log.executionId}`);
    }
    
    // Update cursor for next poll
    if (data.nextCursor) {
      cursor = data.nextCursor;
    }
  }
}

// Poll every 30 seconds
setInterval(pollLogs, 30000);
```

## Ejemplo: Procesamiento de webhooks

```javascript
import express from 'express';
import crypto from 'crypto';

const app = express();
app.use(express.json());

app.post('/tradinggoose-webhook', (req, res) => {
  // Verify signature
  const signature = req.headers['tradinggoose-signature'];
  const body = JSON.stringify(req.body);
  
  if (!verifyWebhookSignature(body, signature, process.env.WEBHOOK_SECRET)) {
    return res.status(401).send('Invalid signature');
  }
  
  // Check timestamp to prevent replay attacks
  const timestamp = parseInt(req.headers['tradinggoose-timestamp']);
  const fiveMinutesAgo = Date.now() - (5 * 60 * 1000);
  
  if (timestamp < fiveMinutesAgo) {
    return res.status(401).send('Timestamp too old');
  }
  
  // Process the webhook
  const event = req.body;
  
  switch (event.type) {
    case 'workflow.execution.completed':
      const { workflowId, executionId, status, cost } = event.data;
      
      if (status === 'error') {
        console.error(`Workflow ${workflowId} failed: ${executionId}`);
        // Handle error...
      } else {
        console.log(`Workflow ${workflowId} completed: ${executionId}`);
        console.log(`Cost: $${cost.total}`);
        // Process successful execution...
      }
      break;
  }
  
  // Return 200 to acknowledge receipt
  res.status(200).send('OK');
});

app.listen(3000, () => {
  console.log('Webhook server listening on port 3000');
});
```
