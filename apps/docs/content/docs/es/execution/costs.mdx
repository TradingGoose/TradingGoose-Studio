TradingGoose calcula automáticamente los costos de todas las ejecuciones de flujos de trabajo, ofreciendo una estructura de precios transparente basada en el uso de modelos de IA y los cargos de ejecución. Comprender estos costos le permite optimizar sus flujos de trabajo y gestionar su presupuesto de manera eficiente.

## Cómo Se Calculan los Costos

Cada ejecución de flujo de trabajo incluye dos componentes de costo:

**Cargo Base de Ejecución**: $0.001 por ejecución

**Uso del Modelo de IA**: Costo variable según el consumo de tokens

```javascript
modelCost = (inputTokens × inputPrice + outputTokens × outputPrice) / 1,000,000
totalCost = baseExecutionCharge + modelCost
```

<Callout type="info">
  Los precios de los modelos de IA son por millón de tokens. El cálculo divide entre 1,000,000 para obtener el costo real. Los flujos de trabajo sin bloques de IA solo incurren en el cargo base de ejecución.
</Callout>

## Desglose de Modelos en los Registros

Para los flujos de trabajo que utilizan bloques de IA, puede consultar información detallada de costos en los registros:

El panel lateral de detalles del registro incluye una sección de desglose por modelo que muestra:
- **Uso de Tokens**: Cantidad de tokens de entrada y salida por cada modelo
- **Desglose de Costos**: Costos individuales por modelo y operación
- **Distribución de Modelos**: Qué modelos se utilizaron y cuántas veces
- **Costo Total**: Costo agregado de toda la ejecución del flujo de trabajo

## Monitoreo de Uso

Supervise su uso y facturación en Configuración → Suscripción:

- **Uso Actual**: Uso y costos en tiempo real para el período vigente
- **Límites de Uso**: Límites del plan con indicadores de progreso visuales
- **Detalles de Facturación**: Cargos proyectados y compromisos mínimos
- **Gestión del Plan**: Opciones de actualización e historial de facturación

### Seguimiento de Uso Programático

Puede consultar su uso actual y límites de forma programática mediante la API:

**Endpoint:**

```text
GET /api/users/me/usage-limits
```

**Autenticación:**
- Incluya su clave de API en el encabezado `X-API-Key`

**Ejemplo de Solicitud:**

```bash
curl -X GET -H "X-API-Key: YOUR_API_KEY" -H "Content-Type: application/json" https://tradinggoose.ai/api/users/me/usage-limits
```

**Ejemplo de Respuesta:**

```json
{
  "success": true,
  "rateLimit": {
    "sync": { "isLimited": false, "limit": 10, "remaining": 10, "resetAt": "2025-09-08T22:51:55.999Z" },
    "async": { "isLimited": false, "limit": 50, "remaining": 50, "resetAt": "2025-09-08T22:51:56.155Z" },
    "authType": "api"
  },
  "usage": {
    "currentPeriodCost": 12.34,
    "limit": 100,
    "plan": "pro"
  }
}
```

**Campos de Respuesta:**
- `currentPeriodCost` refleja el uso en el período de facturación actual
- `limit` se deriva de los límites individuales (Free/Pro) o de los límites agrupados de la organización (Team/Enterprise)
- `plan` es el plan activo de mayor prioridad asociado a su usuario

## Modelo de Facturación

TradingGoose utiliza un modelo de facturación de **suscripción base + excedente**:

### Cómo Funciona

**Plan Pro ($20/mes):**
- La suscripción mensual incluye $20 de uso
- Uso inferior a $20 → Sin cargos adicionales
- Uso superior a $20 → Se paga el excedente al final del mes
- Ejemplo: $35 de uso = $20 (suscripción) + $15 (excedente)

**Plan Equipo ($40/puesto/mes):**
- Uso agrupado entre todos los miembros del equipo
- El excedente se calcula sobre el uso total del equipo
- El propietario de la organización recibe una única factura

**Planes Enterprise:**
- Precio mensual fijo, sin excedentes
- Límites de uso personalizados según acuerdo

### Facturación por Umbral

Cuando el excedente no facturado alcanza los $50, TradingGoose factura automáticamente el importe total pendiente.

**Ejemplo:**
- Día 10: $70 de excedente → Facturación inmediata de $70
- Día 15: $35 de uso adicional ($105 en total) → Ya facturado, sin acción requerida
- Día 20: $50 de uso adicional ($155 en total, $85 sin facturar) → Facturación inmediata de $85

Esto distribuye los cargos por excedente elevados a lo largo del mes, en lugar de generar una única factura de gran importe al final del período.

## Buenas Prácticas de Gestión de Costes

1. **Supervisión Regular**: Consulta frecuentemente el panel de uso para evitar sorpresas
2. **Establecer Presupuestos**: Utiliza los límites del plan como guías de control del gasto
3. **Optimizar Flujos de Trabajo**: Revisa las ejecuciones de mayor coste y optimiza los prompts o la selección de modelos
4. **Usar Modelos Adecuados**: Adapta la complejidad del modelo a los requisitos de cada tarea
5. **Agrupar Tareas Similares**: Combina múltiples solicitudes cuando sea posible para reducir la sobrecarga
