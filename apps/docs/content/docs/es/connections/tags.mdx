Las etiquetas de conexión son representaciones visuales de los datos disponibles de los bloques conectados, que proporcionan una forma sencilla de referenciar datos entre bloques y las salidas de bloques anteriores en tu flujo de trabajo.

### ¿Qué son las etiquetas de conexión?

Las etiquetas de conexión son elementos interactivos que aparecen cuando se conectan bloques. Representan los datos que pueden fluir de un bloque a otro y te permiten:

- Visualizar los datos disponibles de los bloques de origen
- Referenciar campos de datos específicos en los bloques de destino
- Crear flujos de datos dinámicos entre bloques

<Callout type="info">
  Las etiquetas de conexión facilitan ver qué datos están disponibles de los bloques anteriores y utilizarlos en tu
  bloque actual sin necesidad de recordar estructuras de datos complejas.
</Callout>

## Sintaxis de etiquetas

Las etiquetas de conexión utilizan una sintaxis sencilla para hacer referencia a los datos:

```
<blockName.path.to.data>
```

Donde:

- `blockName` es el nombre del bloque de origen (o su ID)
- `path.to.data` es la ruta separada por puntos al campo de datos específico

Por ejemplo:

- `<agent1.content>` - Hace referencia al campo de contenido de un bloque llamado "agent1"
- `<api2.data.users[0].name>` - Hace referencia al nombre del primer usuario en el array de usuarios del campo de datos de un bloque llamado "api2"

### Tipos de referencias especiales

Además de las referencias a bloques, el resolver admite varios prefijos especiales:

- **Alias de disparadores**: `<start.input>`, `<api.fieldName>`, `<chat.input>`, `<manual.input>` - referencias abreviadas a las salidas del bloque disparador
- **Variables**: `<variable.myVar>` - referencia variables del flujo de trabajo por nombre
- **Bucles**: `<loop.item>`, `<loop.index>` - referencia el elemento actual o el índice dentro de un bucle
- **Paralelos**: `<parallel.item>`, `<parallel.index>` - referencia el elemento actual o el índice dentro de una ejecución en paralelo

## Referencias de etiquetas dinámicas

Las etiquetas de conexión se evalúan en tiempo de ejecución, lo que significa que:

1. Siempre hacen referencia a los datos más actuales del contexto de ejecución
2. Pueden utilizarse en expresiones y combinarse con texto estático
3. Pueden anidarse dentro de otras estructuras de datos
4. La indexación de arrays se admite mediante notación de corchetes (p. ej., `items[0]`)

### Ejemplos

```javascript
// Reference in text
"The user's name is <userBlock.name>"

// Reference in JSON
{
  "userName": "<userBlock.name>",
  "orderTotal": <apiBlock.data.total>
}

// Using trigger aliases
"User said: <start.input>"

// Using loop references inside a loop
"Processing item <loop.index>: <loop.item.name>"
```

<Callout type="warning">
  Al utilizar etiquetas de conexión en contextos numéricos, asegúrese de que los datos referenciados sean efectivamente un número
  para evitar problemas de conversión de tipos.
</Callout>
