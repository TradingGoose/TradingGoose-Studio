## Cómo Funcionan las Conexiones

Las conexiones son los canales que permiten el flujo de datos entre los bloques de tu flujo de trabajo. En TradingGoose, las conexiones definen cómo se transmite la información de un bloque a otro, habilitando el flujo de datos a lo largo de tu flujo de trabajo.

<Callout type="info">
  Cada conexión representa una relación dirigida donde los datos fluyen desde la salida de un bloque fuente
  hasta la entrada de un bloque destino.
</Callout>

### Creación de Conexiones

<Steps>
  <Step>
    <strong>Seleccionar Bloque Fuente</strong>: Haz clic en el puerto de salida del bloque desde el que deseas conectar
  </Step>
  <Step>
    <strong>Trazar Conexión</strong>: Arrastra hasta el puerto de entrada del bloque destino
  </Step>
  <Step>
    <strong>Confirmar Conexión</strong>: Suelta para crear la conexión
  </Step>
</Steps>

### Flujo de Conexión

El flujo de datos a través de las conexiones sigue estos principios:

1. **Flujo Direccional**: Los datos siempre fluyen desde las salidas hacia las entradas
2. **Orden de Ejecución**: Los bloques se ejecutan en orden según sus conexiones
3. **Transformación de Datos**: Los datos pueden transformarse al pasar entre bloques
4. **Rutas Condicionales**: Algunos bloques (como Router y Condition) pueden dirigir el flujo hacia distintas rutas

<Callout type="warning">
  Eliminar una conexión detendrá inmediatamente el flujo de datos entre los bloques. Asegúrate de que
  esto es intencional antes de eliminar conexiones.
</Callout>
