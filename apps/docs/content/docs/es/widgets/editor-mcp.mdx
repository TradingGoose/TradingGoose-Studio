## Descripción general

| Propiedad | Valor |
|----------|-------|
| **Clave** | `editor_mcp` |
| **Categoría** | Editor |

El Editor MCP te permite inspeccionar, configurar, probar y actualizar un servidor de Protocolo de Contexto de Modelo (Model Context Protocol) conectado a tu espacio de trabajo. Muestra el estado de conexión del servidor, la configuración de transporte, las herramientas descubiertas y cualquier error reciente. Utiliza el widget [MCP List](/utilities/mcp) para agregar o eliminar servidores; el editor es la interfaz principal para gestionar la configuración de un servidor individual.

## Funcionalidades

- **Encabezado de información del servidor** muestra el nombre del servidor, un distintivo de estado de conexión codificado por color (Conectado / Error / Desconectado) y el tipo de transporte (p. ej., HTTP, SSE).
- **Marcas de tiempo** muestran tiempos relativos para "última actualización", "herramientas actualizadas" y "última conexión", para que puedas ver de un vistazo qué tan recientes son los datos.
- **Formulario del servidor** permite editar el tipo de transporte (streamable-http por defecto), la URL del endpoint, el tiempo de espera de solicitud y las cabeceras HTTP personalizadas.
- **Botón de prueba de conexión** envía una solicitud de prueba al servidor con los valores actuales del formulario y muestra el resultado en línea.
- **Botón de actualizar herramientas** desencadena una actualización en el lado del servidor que redescubre todas las herramientas expuestas por el servidor MCP y actualiza la lista de herramientas local.
- **Botón de restablecer formulario** revierte los campos del formulario a la última configuración guardada del servidor.
- **Botón de guardar** persiste la configuración actualizada del servidor. Si aún no se ha ejecutado ninguna prueba, guardar activa automáticamente una prueba primero y luego guarda y actualiza las herramientas.
- **Botón de limpiar selección** deselecciona el servidor actual.
- **Lista de herramientas** enumera todas las herramientas descubiertas en el servidor, mostrando el nombre y la descripción de cada una.
- **Panel de errores** aparece en la parte inferior cuando el servidor tiene un `lastError`, mostrando el mensaje de error.
- **Selector de servidor** en la barra de encabezado permite cambiar entre servidores MCP en el espacio de trabajo. Cuando se vincula mediante un enlace de color, se sincroniza con el widget MCP List.

## Uso

1. Añade el widget a un espacio de trabajo desde el selector de widgets.
2. Selecciona un servidor MCP desde el menú desplegable del encabezado, o añade uno desde el widget [MCP List](/utilities/mcp).
3. Revisa el estado de la conexión y las marcas de tiempo en la parte superior.
4. Edita la **URL**, el **Tiempo de espera** o los **Encabezados** si es necesario.
5. Haz clic en **Probar conexión** para verificar que el servidor es accesible con la configuración actual.
6. Haz clic en **Actualizar herramientas** para redescubrir las herramientas que expone el servidor.
7. Haz clic en **Guardar** para persistir los cambios. El widget probará la conexión si aún no lo has hecho, luego guardará y actualizará las herramientas.

### Acciones del Encabezado

La barra de herramientas del encabezado proporciona cinco botones de acción (de izquierda a derecha):

| Botón | Ícono | Acción |
|--------|------|--------|
| Actualizar herramientas | Rotar | Redescubre las herramientas del servidor |
| Probar conexión | Play | Envía una solicitud de prueba con los valores actuales del formulario |
| Restablecer formulario | Deshacer | Revierte el formulario a los últimos valores guardados |
| Guardar servidor | Guardar | Persiste la configuración, auto-prueba si es necesario, actualiza herramientas |
| Borrar selección | X | Deselecciona el servidor |

<Callout>
Este widget admite el mecanismo **pairColor**. Asigna el mismo color a un MCP Editor y a un widget [MCP List](/utilities/mcp) para que al seleccionar un servidor en la lista se abra automáticamente en el editor.
</Callout>

## Configuración

El servidor seleccionado se almacena en los parámetros del widget como `mcpServerId` y se sincroniza a través del almacén de color de par cuando está vinculado. Los campos del formulario (transporte, URL, tiempo de espera, encabezados) son estado local derivado del servidor seleccionado y no se persisten hasta que hagas clic en Guardar.
