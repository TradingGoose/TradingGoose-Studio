ワークフローコンソールウィジェットは、ワークフロー実行のリアルタイムビューを提供し、ワークフローの実行中にログ、デバッグ情報、ストリーミング出力を表示します。

## 概要

| プロパティ | 値 |
|----------|-------|
| **キー** | `workflow_console` |
| **カテゴリ** | ユーティリティ |
| **アイコン** | アクティビティ |

## 機能

- **リアルタイムログ**: 実行ログをリアルタイムで表示
- **ストリーミング出力**: ブロック出力をリアルタイムで確認
- **実行ステータス**: ワークフロー実行の現在の状態を監視
- **エラー表示**: エラーを素早く特定してデバッグ
- **ログフィルタリング**: タイプ、ブロック、重要度でログをフィルタリング
- **コンソールクリア**: コンソール出力をリセット

## 使用方法

ワークフローコンソールは以下の用途に不可欠です：

- ワークフロー実行のデバッグ
- 長時間実行されるワークフローの監視
- 中間ブロック出力の表示
- エラーとボトルネックの特定

### ログタイプ

| タイプ | 説明 |
|------|-------------|
| **情報** | 一般的な実行情報 |
| **デバッグ** | 詳細なデバッグ情報 |
| **警告** | 重大ではない問題 |
| **エラー** | 実行エラーと失敗 |
| **出力** | ブロック出力データ |

### 実行フローの表示

コンソールはワークフローを通じた実行パスを表示します：

1. **開始**: ワークフロー実行が始まる
2. **ブロック入力**: 各ブロックが処理を開始
3. **ブロック出力**: 各ブロックからの結果
4. **ブロック終了**: ブロックが完了
5. **終了**: ワークフロー実行が完了

<Callout>
このウィジェットは**pairColor**メカニズムをサポートしています。ワークフローエディタとペアにすると、編集中のワークフローのログを確認できます。
</Callout>
