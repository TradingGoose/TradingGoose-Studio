路由器（Router）模块利用 AI 智能判断工作流的下一步路径，根据特定条件或逻辑对工作流执行进行路由。与使用简单规则的条件（Condition）模块不同，路由器模块能够理解上下文，并基于内容分析做出智能路由决策。

## 概览

路由器模块支持以下功能：

<Steps>
  <Step>
    <strong>智能内容路由</strong>：利用 AI 理解意图与上下文
  </Step>
  <Step>
    <strong>动态路径选择</strong>：基于非结构化内容分析对工作流进行路由
  </Step>
  <Step>
    <strong>上下文感知决策</strong>：突破简单规则限制，做出智能路由选择
  </Step>
  <Step>
    <strong>多路径管理</strong>：处理具有多个潜在目标的复杂工作流
  </Step>
</Steps>

## 路由器模块与条件模块的对比

<Accordions>
  <Accordion title="何时使用路由器">
    - 需要 AI 驱动的内容分析
    - 非结构化或多变的内容类型
    - 基于意图的路由（例如："将支持工单路由至各部门"）
    - 需要上下文感知决策
  </Accordion>
  <Accordion title="何时使用条件">
    - 简单的基于规则的决策
    - 结构化数据或数值比较
    - 需要快速、确定性的路由
    - 布尔逻辑已足够满足需求
  </Accordion>
</Accordions>

## 工作原理

路由器模块：

<Steps>
  <Step>
    <strong>分析内容</strong>：使用 LLM 理解输入内容及上下文
  </Step>
  <Step>
    <strong>评估目标</strong>：将内容与可用的目标模块进行比较
  </Step>
  <Step>
    <strong>选择目标</strong>：根据意图识别最合适的路径
  </Step>
  <Step>
    <strong>执行路由</strong>：将工作流引导至所选模块
  </Step>
</Steps>

## 配置选项

### 内容/提示词

路由器将分析以下内容或提示词来做出路由决策，可以是：

- 用户的直接查询或输入
- 上一个模块的输出结果
- 系统生成的消息

### 目标模块

路由器可从中选择的目标模块。路由器将自动检测已连接的模块，此外您还可以：

- 自定义目标模块的描述以提高路由准确性
- 为每个目标模块指定路由条件
- 将某些模块排除在路由目标之外

### 模型选择

选择用于驱动路由决策的 AI 模型：

**OpenAI**: GPT-4o, o1, o3, o4-mini, gpt-4.1  \
**Anthropic**: Claude 3.7 Sonnet \
**Google**: Gemini 2.5 Pro, Gemini 2.0 Flash \
**Other Providers**: Groq, Cerebras, xAI, DeepSeek \
**Local Models**: Any model running on Ollama 

**建议**：使用具备强大推理能力的模型（如 GPT-4o 或 Claude 3.7 Sonnet）以获得更精准的路由决策。

### API 密钥

所选 LLM 提供商的 API 密钥，将被安全存储并用于身份验证。

### 访问结果

路由器做出决策后，您可以访问其输出内容：

- **`<router.prompt>`**：所用路由提示词的摘要
- **`<router.selected_path>`**：所选目标模块的详细信息
- **`<router.tokens>`**：LLM 的 Token 使用统计
- **`<router.cost>`**：路由调用的费用摘要（输入、输出、合计）
- **`<router.model>`**：用于决策的模型名称

## 最佳实践

- **提供清晰的目标描述**：帮助路由器通过具体、详细的描述来判断何时选择每个目标节点
- **使用明确的路由条件**：为每条路径定义清晰的条件和示例，以提高准确性
- **设置回退路径**：连接一个默认目标节点，用于处理没有匹配路径的情况
- **使用多样化输入进行测试**：确保路由器能够处理各种输入类型、边界情况及意外内容
- **监控路由性能**：定期审查路由决策，并根据实际使用模式优化路由条件
- **选择合适的模型**：针对复杂的路由决策，优先选用具备强大推理能力的模型
