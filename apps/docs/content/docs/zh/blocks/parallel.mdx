并行块是 TradingGoose 中的一种容器块，允许您同时执行多个块实例，从而加快工作流处理速度。

并行块支持两种并发执行类型：

<Callout type="info">
  并行块是容器节点，可同时多次执行其内容，与顺序执行的循环不同。
</Callout>

## 概述

并行块使您能够：

  <Step>
    <strong>分配任务</strong>：并发处理多个项目
  </Step>

## 配置选项

### 并行类型

在两种并行执行类型之间选择：

<Tabs items={['基于数量', '基于集合']}>
  <Tab>
    **基于数量的并行** - 执行固定数量的并行实例。
    
    当您需要同时多次运行相同操作时，请使用此方式。
    

    ```
    Example: Run 5 parallel instances
    - Instance 1 ┐
    - Instance 2 ├─ All execute simultaneously
    - Instance 3 │
    - Instance 4 │
    - Instance 5 ┘
    ```

  </Tab>
  <Tab>
    **基于集合的并行** - 将集合分发到并行实例中。
    
    每个实例同时处理集合中的一个条目。
    

    ```
    Example: Process ["task1", "task2", "task3"] in parallel
    - Instance 1: Process "task1" ┐
    - Instance 2: Process "task2" ├─ All execute simultaneously
    - Instance 3: Process "task3" ┘
    ```

  </Tab>
</Tabs>

## 如何使用并行块

### 创建并行块

1. 从工具栏将并行块拖拽到画布上
2. 配置并行类型和参数
3. 将单个块拖入并行容器内
4. 根据需要连接该块

### 访问结果

并行块执行完成后，您可以访问聚合结果：

- **`<parallel.results>`**：来自所有并行实例的结果数组

## 高级功能

### 结果聚合

所有并行实例的结果将自动收集：

```javascript
// In a Function block after the parallel
const allResults = input.parallel.results;
// Returns: [result1, result2, result3, ...]
```

### 实例隔离

每个并行实例独立运行：
- 独立的变量作用域
- 实例间无共享状态
- 单个实例的失败不影响其他实例

### 限制

<Callout type="warning">
  容器块（循环块和并行块）不能相互嵌套。这意味着：
  - 不能将循环块放置在并行块内
  - 不能将另一个并行块放置在并行块内
  - 不能将任何容器块放置在另一个容器块内
</Callout>

<Callout type="warning">
  并行块只能包含单个块。在并行块内部，不能将多个块相互连接——在这种情况下，只有第一个块会执行。
</Callout>

<Callout type="info">
  并行执行速度更快，但需注意以下几点：
  - 并发请求时的 API 速率限制
  - 大数据集的内存占用
  - 最多 20 个并发实例，以防止资源耗尽
</Callout>

## 并行 vs 循环

了解何时使用各自的方式：

| 特性 | 并行 | 循环 |
|---------|----------|------|
| 执行方式 | 并发 | 顺序 |
| 速度 | 独立操作更快 | 较慢但有序 |
| 顺序 | 不保证顺序 | 保持顺序 |
| 适用场景 | 独立操作 | 依赖操作 |
| 资源占用 | 较高 | 较低 |

## 最佳实践

- **仅限独立操作**：确保各操作之间互不依赖
- **处理速率限制**：为 API 密集型工作流添加延迟或限流机制
- **错误处理**：每个实例应能优雅地处理自身的错误
