Agent 模块使您能够：

## 配置选项

### 系统提示词

系统提示词用于设定 Agent 的运行参数与行为约束。该配置定义了 Agent 的角色、响应方法论，以及对所有传入请求的处理边界。

```markdown
You are a helpful assistant that specializes in financial analysis.
Always provide clear explanations and cite sources when possible.
When responding to questions about investments, include risk disclaimers.
```

### 用户提示词

用户提示词是推理处理的主要输入数据。该参数接受自然语言文本或结构化数据，Agent 将对其进行分析并作出响应。输入来源包括：

- **静态配置**：在模块配置中直接指定的文本输入
- **动态输入**：通过连接接口从上游模块传递的数据
- **运行时生成**：在工作流执行过程中以编程方式生成的内容

### 模型选择

Agent 模块通过统一的推理接口支持多种 LLM 提供商。可用模型包括：

**OpenAI 模型**：GPT-5、GPT-4o、o1、o3、o4-mini、gpt-4.1（基于 API 的推理）
**Anthropic 模型**：Claude 3.7 Sonnet（基于 API 的推理）
**Google 模型**：Gemini 2.5 Pro、Gemini 2.0 Flash（基于 API 的推理）
**其他提供商**：Groq、Cerebras、xAI、DeepSeek（基于 API 的推理）
**本地部署**：兼容 Ollama 的模型（自托管推理）

### 温度参数

控制响应的创造性与随机性：

### API 密钥

所选 LLM 提供商的 API 密钥。该密钥经过安全存储，用于身份验证。

### 工具

工具通过外部 API 集成和服务连接扩展智能体的能力。工具系统支持函数调用，使智能体能够执行文本生成之外的操作。

**工具集成流程**：
1. 在 Agent 模块中访问工具配置区域
2. 从 60 余个预构建集成中选择，或自定义函数
3. 配置身份验证参数和操作约束

**可用工具类别**：
- **通信**：Gmail、Slack、Telegram、WhatsApp、Microsoft Teams
- **数据源**：Notion、Google Sheets、Airtable、Supabase、Pinecone
- **Web 服务**：Firecrawl、Google Search、Exa AI、浏览器自动化
- **开发**：GitHub、Jira、Linear 仓库与问题管理
- **AI 服务**：OpenAI、Perplexity、Hugging Face、ElevenLabs

**工具执行控制**：
- **自动（Auto）**：模型根据上下文和需要自行决定是否调用工具
- **必需（Required）**：每次推理请求时必须调用该工具
- **禁用（None）**：工具定义可用，但不纳入模型上下文

### 响应格式

响应格式参数通过 JSON Schema 验证强制执行结构化输出生成。这确保响应一致、可机器读取，并符合预定义的数据结构：

```json
{
  "name": "user_analysis",
  "schema": {
    "type": "object",
    "properties": {
      "sentiment": {
        "type": "string",
        "enum": ["positive", "negative", "neutral"]
      },
      "confidence": {
        "type": "number",
        "minimum": 0,
        "maximum": 1
      }
    },
    "required": ["sentiment", "confidence"]
  }
}
```

此配置约束模型的输出以符合指定的 Schema，防止生成自由文本响应，确保结构化数据的生成。

### 访问结果

智能体执行完成后，您可以访问其输出：

- **`<agent.content>`**：智能体的响应文本或结构化数据
- **`<agent.tokens>`**：Token 使用统计信息（提示、补全、合计）
- **`<agent.tool_calls>`**：智能体在执行过程中所用工具的详细信息
- **`<agent.cost>`**：API 调用的预估费用（如可用）

## 高级功能

### 记忆 + Agent：对话历史

使用具有一致 `id`（例如 `chat`）的 `Memory` 块，可在多次运行之间持久化消息，并将该历史记录包含在 Agent 的提示词中。

- 在 Agent 运行前添加用户消息
- 读取对话历史以获取上下文
- 在 Agent 运行后追加其回复

详情请参阅 [`Memory`](/tools/memory) 块参考文档。

## 最佳实践

- **在系统提示中明确说明**：清晰定义智能体的角色、语气与限制范围。指令越具体，智能体越能有效实现其预期目标。
- **选择合适的温度参数**：当准确性至关重要时，使用较低的温度设置（0–0.3）；若需要更具创意或多样化的回复，则提高温度（0.7–2.0）。
- **有效利用工具**：集成与智能体目标相辅相成、能增强其能力的工具。在工具选择上应有所取舍，避免工具过多导致智能体负担过重。对于任务重叠度较低的场景，建议使用独立的 Agent 块以获得最佳效果。
