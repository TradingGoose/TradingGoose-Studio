Webhook 请求块向外部 Webhook URL 发送**出站** HTTP POST 请求。可用于将工作流结果推送至第三方服务，例如 Zapier、Make、n8n，或任何接受 Webhook 数据负载的自定义端点。

<Callout type="warning">
  此模块发送**出站** Webhook 调用（您的工作流调用外部 URL）。它与 **Webhook 触发器**不同，后者监听用于启动工作流的**入站**请求。
</Callout>

## 概述

<Steps>
  <Step>
    <strong>设置目标 URL</strong>：指向任何接受 HTTP POST 请求的端点
  </Step>
  <Step>
    <strong>构建 JSON 负载</strong>：使用变量引用（如 `<agent.content>`）包含来自上游模块的数据
  </Step>
  <Step>
    <strong>可选：对请求签名</strong>：提供 HMAC-SHA256 签名密钥，以便接收方验证请求的真实性
  </Step>
  <Step>
    <strong>添加自定义请求头</strong>：包含接收方所需的 API 密钥、内容类型或其他额外请求头
  </Step>
</Steps>

## 工作原理

当模块执行时，它将：

1. 解析 URL、负载和请求头中的所有变量引用（`<block.output>`）。
2. 将负载序列化为 JSON 格式。
3. 若提供了**签名密钥**，则对负载计算 HMAC-SHA256 签名，并将其作为 Webhook 请求头附加，以便接收服务器验证请求未被篡改。
4. 使用 `Content-Type: application/json` 及您配置的所有附加请求头发送 POST 请求。
5. 将响应数据、状态码和响应头作为模块输出返回。

### 字段

| 字段 | 必填 | 描述 |
|---|---|---|
| **Webhook URL** | 是 | 目标端点，必须接受 HTTP POST 请求。 |
| **Payload** | 否 | 要发送的 JSON 请求体。使用 `<block.output>` 引用上游数据。 |
| **Signing Secret** | 否 | 用于生成 HMAC-SHA256 签名请求头以验证负载的共享密钥。 |
| **Additional Headers** | 否 | 附加到请求的键值对（例如，`Authorization: Bearer ...`）。 |

## 输出

| 输出 | 类型 | 描述 |
|---|---|---|
| `data` | json | 来自外部端点的已解析响应体 |
| `status` | number | 端点返回的 HTTP 状态码 |
| `headers` | json | 响应头 |

## 最佳实践

- **将密钥存储在环境变量中** —— 使用 `{{WEBHOOK_SECRET}}` 引用您的签名密钥，而非直接粘贴。
- **检查状态输出** —— 在 Webhook 之后使用条件块或路由块，以优雅地处理非 2xx 响应。
- **保持负载精简** —— 仅发送接收方所需的数据。过大的负载会增加延迟，且可能被某些端点拒绝。
