## 概述

响应块使您能够：

<Steps>
  <Step>
    <strong>格式化 API 响应</strong>：将工作流结果构建为标准的 HTTP 响应
  </Step>
  <Step>
    <strong>设置状态码</strong>：根据工作流执行结果配置适当的 HTTP 状态码
  </Step>
  <Step>
    <strong>控制响应头</strong>：为 API 响应和 Webhook 添加自定义头信息
  </Step>
  <Step>
    <strong>转换数据</strong>：将工作流变量转换为客户端友好的响应格式
  </Step>
</Steps>

## 工作原理

响应块负责完成工作流的执行：

1. **收集数据** - 从前置块中收集变量和输出结果
2. **格式化响应** - 根据配置对数据进行结构化处理
3. **设置 HTTP 详情** - 应用状态码和请求头
4. **发送响应** - 将格式化后的响应返回给 API 调用方

## 何时需要响应块

- **API 端点**：当工作流通过 API 调用时，响应块负责格式化返回数据
- **Webhooks**：向调用系统返回确认信息或数据
- **测试**：在测试工作流时查看格式化后的结果

## 两种构建响应的方式

### 构建器模式（推荐）
可视化界面，用于构建响应结构：
- 拖放字段
- 轻松引用工作流变量
- 响应结构可视化预览

### 编辑器模式（高级）
直接编写 JSON：
- 完全掌控响应格式
- 支持复杂的嵌套结构
- 使用 `<variable.name>` 语法处理动态值

## 配置选项

### 响应数据

响应数据是将发送回 API 调用方的主要内容。该内容应格式化为 JSON，可包含以下内容：

- 静态值
- 使用 `<variable.name>` 语法对工作流变量的动态引用
- 嵌套对象和数组
- 任何合法的 JSON 结构

### 状态码

为响应设置 HTTP 状态码。常见状态码包括：

### 响应头

配置要包含在响应中的附加 HTTP 头。

头部以键值对的形式配置：

| 键 | 值 |
|-----|-------|
| Content-Type | application/json |
| Cache-Control | no-cache |
| X-API-Version | 1.0 |

## 变量引用

使用 `<variable.name>` 语法将工作流变量动态插入到您的响应中：

```json
{
  "user": {
    "id": "<variable.userId>",
    "name": "<variable.userName>",
    "email": "<variable.userEmail>"
  },
  "query": "<variable.searchQuery>",
  "results": "<variable.searchResults>",
  "totalFound": "<variable.resultCount>",
  "processingTime": "<variable.executionTime>ms"
}
```

<Callout type="warning">
  变量名称区分大小写，必须与工作流中可用的变量完全匹配。
</Callout>

## 最佳实践

- **使用有意义的状态码**：选择能准确反映工作流执行结果的适当 HTTP 状态码
- **保持响应结构一致性**：在所有 API 端点中维护统一的 JSON 结构，以提升开发者体验
- **包含相关元数据**：添加时间戳和版本信息，以便于调试和监控
- **优雅处理错误**：在工作流中使用条件逻辑，设置包含描述性消息的适当错误响应
- **验证变量引用**：在 Response 块执行前，确保所有引用的变量均存在且包含预期的数据类型
