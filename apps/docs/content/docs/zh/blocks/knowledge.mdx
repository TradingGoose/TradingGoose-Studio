知识库模块将您的工作流与知识库相连接，通过执行向量相似性搜索、上传新文本块及创建文档，在运行时实现检索增强生成（RAG）。

## 概述

<Steps>
  <Step>
    <strong>搜索</strong>：对知识库执行语义相似性查询，检索最相关的文本块
  </Step>
  <Step>
    <strong>上传文本块</strong>：将新文本块追加至现有文档，使其在后续查询中可被检索
  </Step>
  <Step>
    <strong>创建文档</strong>：向知识库添加全新文档（支持可选标签）
  </Step>
</Steps>

## 操作

<Tabs items={['Search', 'Upload Chunk', 'Create Document']}>
  <Tab>
    执行向量相似性搜索并返回最匹配的文本块。

    | 字段 | 必填 | 描述 |
    |---|---|---|
    | **知识库** | 是 | 要搜索的知识库 |
    | **搜索查询** | 否 | 自然语言查询。单独使用标签过滤器时可选填。 |
    | **结果数量** | 否 | 返回的文本块数量（默认：10） |
    | **标签过滤器** | 否 | 将结果范围缩小至匹配特定标签的文档 |

    搜索查询是可选的——您可以纯粹通过标签进行过滤，以检索某一类别下的所有数据块。
  </Tab>
  <Tab>
    将文本块追加到知识库中的现有文档。该块会自动嵌入向量并可用于后续搜索。

    | 字段 | 必填 | 描述 |
    |---|---|---|
    | **知识库** | 是 | 目标知识库 |
    | **文档** | 是 | 要添加数据块的文档 |
    | **块内容** | 是 | 要上传的文本内容 |
  </Tab>
  <Tab>
    在知识库中创建新文档。内容将被分块、嵌入向量并建立索引。

    | 字段 | 必填 | 描述 |
    |---|---|---|
    | **知识库** | 是 | 目标知识库 |
    | **文档名称** | 是 | 新文档的名称 |
    | **文档内容** | 是 | 完整文本内容 |
    | **文档标签** | 否 | 用于分类和过滤的标签 |
  </Tab>
</Tabs>

## 输出

| 输出 | 类型 | 描述 |
|---|---|---|
| `results` | json | 匹配块的数组（搜索）或确认数据（写入操作） |
| `query` | string | 所使用的查询语句 |
| `totalResults` | number | 返回的结果数量 |

## 最佳实践

- **使用标签过滤器限定搜索范围** —— 标签可让您按主题、客户或类别对知识库进行分区，而无需创建独立的知识库。
- **保持块内容聚焦** —— 简短且主题连贯的块比冗长的多主题文本能产生更好的相似性匹配效果。
- **与 Agent 块配合使用** —— 将搜索结果传入 Agent 的上下文，以构建 RAG 流水线。

<Callout type="info">
  搜索查询字段为可选项。当您只需要匹配特定标签的文档时，可将查询留空，改用标签过滤器。
</Callout>
