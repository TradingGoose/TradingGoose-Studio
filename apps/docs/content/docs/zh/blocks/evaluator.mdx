## 工作原理

评估器模块通过 AI 驱动的评估流程对内容进行处理：

1. **接收内容** - 从工作流中的前置模块获取输入内容
2. **应用指标** - 根据您定义的自定义指标对内容进行评估
3. **生成评分** - AI 模型为每个指标分配数值评分
4. **提供摘要** - 返回包含评分和说明的详细评估结果

## 配置选项

### 评估指标

定义用于评估内容的自定义指标。每个指标包含：

- **名称**：指标的简短标识符
- **描述**：对该指标所衡量内容的详细说明
- **范围**：评分的数值区间（例如 1-5、0-10）

示例指标：

```
Accuracy (1-5): How factually accurate is the content?
Clarity (1-5): How clear and understandable is the content?
Relevance (1-5): How relevant is the content to the original query?
```

### 内容

待评估的内容，可以是：

- 直接在模块配置中提供
- 从另一个模块的输出连接（通常为 Agent 模块）
- 在工作流执行过程中动态生成

### 模型选择

选择用于执行评估的 AI 模型：

**OpenAI**：GPT-4o、o1、o3、o4-mini、gpt-4.1
**Anthropic**：Claude 3.7 Sonnet
**Google**：Gemini 2.5 Pro、Gemini 2.0 Flash
**其他提供商**：Groq、Cerebras、xAI、DeepSeek
**本地模型**：任何运行在 Ollama 上的模型

**建议**：使用具备强大推理能力的模型（如 GPT-4o 或 Claude 3.7 Sonnet）以获得更准确的评估结果。

### API 密钥

所选 LLM 提供商的 API 密钥，经安全存储并用于身份验证。

## 工作原理

1. 评估器模块获取提供的内容及您的自定义指标
2. 生成专用提示词，指示 LLM 对内容进行评估
3. 提示词包含每个指标的明确评分准则
4. LLM 评估内容并为每个指标返回数值评分
5. 评估器模块将这些评分格式化为结构化输出，以供工作流后续使用

## 最佳实践

- **使用具体的指标描述**：明确定义每个指标的衡量内容，以获得更准确的评估
- **选择适当的范围**：选择能够提供足够细粒度而又不过于复杂的评分范围
- **与 Agent 模块连接**：使用评估器模块来评估 Agent 模块的输出并创建反馈循环
- **使用一致的指标**：在进行比较分析时，在类似评估中保持指标的一致性
- **结合多个指标**：使用多个指标以获得全面的评估
