## 概述

API 模块使您能够：

<Steps>
  <Step>
    <strong>连接外部服务</strong>：向 REST API 和 Web 服务发起 HTTP 请求
  </Step>
  <Step>

  </Step>

## 工作原理

API 模块通过结构化方式处理 HTTP 请求：

1. **配置请求** - 设置 URL、方法、请求头和请求体参数
2. **执行请求** - 向指定端点发送 HTTP 请求
3. **处理响应** - 处理响应数据、状态码和请求头
4. **错误处理** - 管理超时、重试和错误条件

## 配置选项

### URL

API 请求的端点 URL，可以是：

- 直接在模块中输入的静态 URL
- 从其他模块输出连接的动态 URL
- 包含路径参数的 URL

### 方法

选择请求所使用的 HTTP 方法：

- **GET**：从服务器检索数据
- **POST**：向服务器发送数据以创建资源
- **PUT**：更新服务器上的现有资源
- **DELETE**：从服务器删除资源
- **PATCH**：部分更新现有资源

### 查询参数

定义将作为查询参数附加到 URL 的键值对。例如：

```
Key: apiKey
Value: your_api_key_here

Key: limit
Value: 10
```

这些参数将以 `?apiKey=your_api_key_here&limit=10` 的形式附加到 URL。

### 请求头

为请求配置 HTTP 请求头。常见的请求头包括：

```
Key: Content-Type
Value: application/json

Key: Authorization
Value: Bearer your_token_here
```

### 请求体

对于支持请求体的方法（POST、PUT、PATCH），您可以定义要发送的数据。请求体可以是：

- 直接在模块中输入的 JSON 数据
- 从其他模块输出连接的数据
- 在工作流执行期间动态生成的数据

### 访问结果

API 请求完成后，您可以访问其输出：

- **`<api.data>`**：来自 API 的响应体数据
- **`<api.status>`**：HTTP 状态码（200、404、500 等）
- **`<api.headers>`**：来自服务器的响应头
- **`<api.error>`**：请求失败时的错误详情

## 高级功能

### 动态URL构建

使用前置块中的变量动态构建URL：

```javascript
// In a Function block before the API
const userId = <start.userId>;
const apiUrl = `https://api.example.com/users/${userId}/profile`;
```

### 请求重试

API块自动处理以下情况：
- 网络超时（指数退避策略）
- 速率限制响应（429状态码）
- 服务器错误（5xx状态码）及重试逻辑
- 连接失败及重连尝试

### 响应验证

在处理前验证API响应：

```javascript
// In a Function block after the API
if (<api.status> === 200) {
  const data = <api.data>;
  // Process successful response
} else {
  // Handle error response
  console.error(`API Error: ${<api.status>}`);
}
```

## 最佳实践

- **对敏感数据使用环境变量**：请勿硬编码 API 密钥或凭据
- **优雅处理错误**：为失败的请求连接错误处理逻辑
- **验证响应**：在处理数据前检查状态码和响应格式
- **遵守速率限制**：注意 API 速率限制并实施适当的限流措施
