循环块是一个容器，用于重复执行其内部的块。它不是一个拥有独立源文件的普通块——它是一个由执行器的 `LoopBlockHandler` 管理的特殊包装节点。将块拖入循环容器后，它们将在每次迭代时执行一次。

<Callout type="info">
  循环块是容器节点。在画布上将其他块拖入其中，以定义每次迭代时运行的内容。
</Callout>

## 循环类型

<Tabs items={['For', 'ForEach', 'While', 'Do-While']}>
  <Tab>
    **For 循环** —— 运行固定次数。若未指定，默认迭代次数为 **5**。

    ```
    Example: Run 3 times
    - Iteration 1 (index 0)
    - Iteration 2 (index 1)
    - Iteration 3 (index 2)
    ```

  </Tab>
  <Tab>
    **ForEach 循环** —— 遍历数组或对象中的每个元素。集合必须非空；空集合将抛出错误。

    ```
    Example: Process ["apple", "banana", "orange"]
    - Iteration 1: currentItem = "apple"
    - Iteration 2: currentItem = "banana"
    - Iteration 3: currentItem = "orange"
    ```

    对于对象，每次迭代将接收一个 `[key, value]` 键值对。
  </Tab>
  <Tab>
    **While 循环** —— 在每次迭代**前**对条件进行求值。若首次检查时条件为假，则循环体永不执行。

    While 和 Do-While 循环**没有内置的迭代上限**。它们依赖条件变为假（或工作流超时）来终止。
  </Tab>
  <Tab>
    **Do-While 循环** —— 至少执行一次循环体，然后在每次迭代**后**对条件进行求值。适用于在检查条件前需要至少执行一次的场景。
  </Tab>
</Tabs>

## 工作原理

执行器的 `LoopBlockHandler` 负责管理迭代控制：

1. **初始化** -- 循环计数器从 1 开始。对于 `forEach` 循环，集合会被解析（JSON、变量引用或表达式）。
2. **检查继续条件** -- `for`/`forEach` 将计数器与集合长度或迭代次数进行比较。`while`/`doWhile` 使用与条件块相同的引擎对条件表达式进行求值。
3. **激活子节点** -- 连接到循环内部起始句柄的块被置于活动执行路径上。
4. **收集结果** -- 每次迭代的输出被聚合至 `loop.results`。
5. **递增并重复** -- 计数器递增，步骤 2 再次执行。当循环结束时，连接到循环退出句柄的块将被激活。

## 迭代变量

在循环体内部，以下变量可供使用：

| 变量 | 类型 | 描述 |
|---|---|---|
| `loop.index` | number | 当前迭代的从零开始的索引 |
| `loop.currentItem` | any | 当前元素（ForEach 循环）或循环上下文（While 循环） |
| `loop.items` | array/object | 正在迭代的完整集合（仅限 ForEach） |

循环完成后，下游块可访问：

| 变量 | 类型 | 描述 |
|---|---|---|
| `loop.results` | array | 所有迭代输出的有序数组 |

## 配置

| 参数 | 适用范围 | 描述 |
|---|---|---|
| **循环类型** | 所有 | `for`、`forEach`、`while` 或 `doWhile` |
| **迭代次数** | For | 重复执行的次数（默认值：5） |
| **集合** | ForEach | 要迭代的数组或对象。支持 JSON 字面量、`<block.output>` 引用及表达式。 |
| **条件** | While / Do-While | 每次迭代时求值的布尔表达式（例如：`<variable.i> < 10`） |

## 示例用例

### 处理 API 结果

1. 一个 API 块获取客户记录列表。
2. ForEach 循环遍历每个客户。
3. 在循环内部，Agent 分析客户数据，Function 存储结果。
4. 循环结束后，Response 块返回聚合的 `loop.results`。

### 迭代优化

1. 设置为 5 次迭代的 For 循环生成内容变体。
2. 在循环内部，Agent 生成草稿，Evaluator 对其评分。
3. 循环结束后，Function 从 `loop.results` 中选出得分最高的变体。

## 限制

<Callout type="warning">
  容器块（循环与并行）**不能相互嵌套**。不能将 Loop 放置在另一个 Loop 内，也不能将 Parallel 放置在 Loop 内。如需多维迭代，请重构工作流，使用顺序循环或分阶段处理数据。
</Callout>

<Callout type="info">
  循环以**顺序**方式执行各迭代。如需并发执行独立任务，请改用 Parallel 块。
</Callout>

## 最佳实践

- **为 For 循环设置合理的迭代次数**，以避免工作流长时间运行。默认值为 5。
- **对集合使用 ForEach** —— 它会自动适配集合长度，并为您提供 `currentItem`。
- **始终在 While/Do-While 循环内更新条件变量**。若忘记修改条件所依赖的变量，将导致无限循环（仅受工作流超时时间限制）。
