## 概述

条件块支持您：

  </Step>

## 工作原理

条件块通过顺序求值流程运行：

1. **求值表达式** - 使用当前工作流数据处理 JavaScript/TypeScript 布尔表达式
2. **确定结果** - 根据表达式求值返回 true 或 false
3. **路由工作流** - 根据结果将执行定向至相应的目标块
4. **提供上下文** - 生成关于决策的元数据，用于调试和监控

## 配置选项

### 条件

定义一个或多个待求值的条件。每个条件包含：

- **表达式**：一个 JavaScript/TypeScript 表达式，求值结果为 true 或 false
- **路径**：条件为 true 时路由到的目标块
- **描述**：对条件检查内容的可选说明

您可以创建多个按顺序求值的条件，第一个匹配的条件将决定执行路径。

### 条件表达式格式

条件使用 JavaScript 语法，可引用前置块的输入值。

### 访问结果

条件求值后，您可以访问其输出：

- **`<condition.result>`**：条件求值的布尔结果
- **`<condition.matched_condition>`**：已匹配条件的 ID
- **`<condition.content>`**：求值结果的描述
- **`<condition.path>`**：所选路由目标的详细信息

## 高级功能

### 复杂表达式

在条件中使用 JavaScript 运算符和函数：

```javascript
// String operations
<user.email>.endsWith('@company.com')

// Array operations
<api.tags>.includes('urgent')

// Mathematical operations
<agent.confidence> * 100 > 85

// Date comparisons
new Date(<api.created_at>) > new Date('2024-01-01')
```

### 多条件求值

条件按顺序依次求值，直到某一条件匹配为止：

```javascript
// Condition 1: Check for high priority
<ticket.priority> === 'high'

// Condition 2: Check for urgent keywords
<ticket.subject>.toLowerCase().includes('urgent')

// Condition 3: Default fallback
true
```

### 错误处理

条件自动处理以下情况：
- 使用安全求值处理 undefined 或 null 值
- 使用适当的回退值处理类型不匹配
- 使用错误日志记录处理无效表达式
- 使用默认值处理缺失变量

## 最佳实践

- **正确排列条件顺序**：将更具体的条件置于通用条件之前，确保特定逻辑优先于兜底逻辑
- **包含默认条件**：将捕获所有情况的条件（`true`）作为最后一个条件，以处理未匹配的情况并防止工作流执行卡住
- **保持表达式简洁**：使用清晰直观的布尔表达式，提升可读性并便于调试
- **为条件添加说明**：添加描述以解释每个条件的用途，促进团队协作与维护
- **测试边界情况**：通过在条件范围边界处使用测试值，验证条件能否正确处理边界值
