模块是您连接在一起以创建AI工作流的构建组件。将它们视为专用模块，每个模块负责处理特定任务——从与AI模型对话到发起API调用或处理数据。

## 核心模块类型

TradingGoose提供处理AI工作流核心功能的基础模块类型：

### 处理模块
- **[Agent](/blocks/agent)** - 与AI模型对话（OpenAI、Anthropic、Google、本地模型）
- **[Function](/blocks/function)** - 运行自定义JavaScript/TypeScript代码
- **[API](/blocks/api)** - 通过HTTP请求连接外部服务

### 逻辑模块
- **[Condition](/blocks/condition)** - 基于布尔表达式分支工作流路径
- **[Router](/blocks/router)** - 使用AI智能地将请求路由至不同路径
- **[Evaluator](/blocks/evaluator)** - 使用AI对内容质量进行评分和评估

### 控制流模块
- **[Variables](/blocks/variables)** - 设置和管理工作流作用域变量
- **[Wait](/blocks/wait)** - 按指定时间延迟暂停工作流执行

### 输出模块
- **[Response](/blocks/response)** - 格式化并返回工作流的最终结果

## 模块工作原理

每个模块包含三个主要组成部分：

**输入**：从其他模块或用户输入传入该模块的数据
**配置**：控制模块行为的设置项
**输出**：该模块为其他模块生成的数据

<Steps>
  <Step>
    <strong>接收输入</strong>：模块从已连接的模块或用户输入接收数据
  </Step>
  <Step>
    <strong>处理</strong>：模块根据其配置对输入进行处理
  </Step>
  <Step>

  </Step>
</Steps>

## 连接模块

您可以通过将模块连接在一起来创建工作流。一个模块的输出将成为另一个模块的输入：

- **拖拽连接**：从输出端口拖拽至输入端口
- **多重连接**：一个输出可以连接到多个输入
- **分支路径**：某些模块可以根据条件路由到不同的路径

## 常用模式

### 顺序处理
将模块链式连接，每个模块处理前一个模块的输出：

```
User Input → Agent → Function → Response
```

### 条件分支
使用条件块或路由块创建不同的执行路径：

```
User Input → Router → Agent A (for questions)
                   → Agent B (for commands)
```

### 质量控制
使用评估块对输出进行评估和过滤：

```
Agent → Evaluator → Condition → Response (if good)
                              → Agent (retry if bad)
```

## 模块配置

每种模块类型都有特定的配置选项：

**所有模块**：
- 输入/输出连接
- 错误处理行为
- 执行超时设置

**AI 模块**（Agent、Router、Evaluator）：
- 模型选择（OpenAI、Anthropic、Google、本地）
- API 密钥与身份验证
- 温度参数及其他模型参数
- 系统提示词与指令

**逻辑模块**（Condition、Function）：
- 自定义表达式或代码
- 变量引用
- 执行环境设置

**集成模块**（API、Response）：
- 端点配置
- 请求头与身份验证
- 请求/响应格式化
