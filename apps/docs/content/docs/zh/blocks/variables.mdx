## 概述

变量块支持以下操作：

  <Step>

  </Step>
  <Step>
    <strong>存储动态数据</strong>：将块输出捕获至变量中
  </Step>
  <Step>
    <strong>维护状态</strong>：跟踪计数器、标志位及中间结果

## 如何使用变量

### 1. 在工作流变量中初始化

首先，在工作流的"变量"部分创建变量（可通过工作流设置访问）：

```
customerEmail = ""
retryCount = 0
currentStatus = "pending"
```

### 2. 使用变量块进行更新

使用变量块在执行期间更新这些值：

```
customerEmail = <api.email>
retryCount = <variable.retryCount> + 1
currentStatus = "processing"
```

### 3. 在任意位置访问

在任意块中引用变量：

```
Agent prompt: "Send email to <variable.customerEmail>"
Condition: <variable.retryCount> < 5
API body: {"status": "<variable.currentStatus>"}
```

## 输出

- **`<variables.assignments>`**：包含该模块所有变量赋值的 JSON 对象

## 最佳实践

- **在工作流设置中初始化**：使用变量前，始终在工作流的变量部分中预先创建变量
- **动态更新**：使用变量模块根据模块输出或计算结果更新变量值
- **在循环中使用**：非常适合跨迭代追踪状态
- **使用描述性命名**：采用清晰的变量名，例如 `currentIndex`、`totalProcessed` 或 `lastError`
