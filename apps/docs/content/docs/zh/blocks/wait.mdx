等待块会在工作流中暂停指定的时间，然后继续执行下一个块。可用于在操作之间添加延迟、遵守 API 速率限制，或将操作分散执行。

## 概述

等待块支持以下功能：

<Steps>
  <Step>
    <strong>添加时间延迟</strong>：在工作流步骤之间暂停执行
  </Step>
  <Step>
    <strong>遵守速率限制</strong>：分散 API 调用以保持在限制范围内
  </Step>
  <Step>
    <strong>调度执行序列</strong>：创建在操作之间具有延迟的定时工作流
  </Step>
</Steps>

## 配置

### 等待时长

输入暂停执行的持续时间：
- **输入**：正整数
- **最大值**：600 秒（10 分钟）或 10 分钟

### 单位

选择时间单位：
- **秒**：用于短暂、精确的延迟
- **分钟**：用于较长时间的暂停

<Callout type="info">
  等待块可通过停止工作流来取消。最大等待时间为 10 分钟。
</Callout>

## 输出

- **`<wait.waitDuration>`**：等待持续时间（毫秒）
- **`<wait.status>`**：等待状态（'waiting'、'completed' 或 'cancelled'）

## 最佳实践

- **保持合理的等待时长**：对于不超过 10 分钟的延迟，使用等待块；若需更长延迟，建议使用计划工作流
- **监控执行时间**：请注意，等待会延长工作流的总执行时长
