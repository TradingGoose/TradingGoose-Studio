## 工具

### `trading_order_detail`

通过 Trading Goose 订单 ID 检索先前记录的交易订单的所有提供商可用详情。

#### 输入

| 参数 | 类型 | 必填 | 描述 |
| --------- | ---- | -------- | ----------- |
| `orderId` | string | 是 | 提交订单时创建的 Trading Goose 订单 ID \(orderHistoryTable.id\)。 |
| `provider` | string | 否 | 该订单的预期提供商，用于凭证选择和不匹配验证。 |
| `environment` | string | 否 | 可选的环境覆盖 \(模拟盘或实盘\)，用于获取提供商订单详情。 |
| `credential` | string | 否 | 所选券商的 OAuth 凭证 ID \(从所选账户填充\)。 |
| `tradierCredential` | string | 否 | Tradier OAuth 凭证 ID。 |
| `robinhoodCredential` | string | 否 | Robinhood OAuth 凭证 ID。 |
| `alpacaCredential` | string | 否 | Alpaca OAuth 凭证 ID。 |
| `apiKey` | string | 否 | Alpaca API 密钥 ID \(使用 OAuth 时可选\)。 |
| `apiSecret` | string | 否 | Alpaca API 密钥 \(使用 OAuth 时可选\)。 |
| `accountId` | string | 否 | 若存储的订单元数据中未包含 Tradier 账户 ID，可通过此字段覆盖。 |
| `accountUrl` | string | 否 | 如有需要，可覆盖 Robinhood 账户 URL。 |

#### 输出

| 参数 | 类型 | 描述 |
| --------- | ---- | ----------- |
| `summary` | string | 订单详情检索的状态消息。 |
| `provider` | string | 订单的券商/提供商。 |
| `appOrderId` | string | Trading Goose 订单 ID \(orderHistoryTable.id\)。 |
| `providerOrderId` | string | 用于 API 查询的提供商原生订单 ID。 |

## 使用说明

通过订单ID查询Trading Goose订单历史记录，解析提供商订单ID，并获取最新的提供商订单详情。
