工具是 TradingGoose 中的强大组件，使您的工作流能够与外部服务交互、处理数据并执行专项任务。通过提供对各类 API 和服务的访问能力，工具极大地扩展了智能体与工作流的功能边界。

## 什么是工具？

工具是一种专用组件，提供特定功能或与外部服务的集成能力。工具可用于网络搜索、数据库交互、图像处理、文本或图像生成、消息平台通信等多种场景。

## 在工作流中使用工具

在 TradingGoose 工作流中使用工具主要有两种方式：

<Steps>
  <Step>
    <strong>作为独立模块</strong>：当您需要确定性地直接调用工具功能时，可将工具作为独立模块添加到画布中。这使您能够精确控制工具的调用时机与调用方式。
  </Step>
  <Step>
    <strong>作为智能体工具</strong>：可通过点击"添加工具"并配置所需参数，将工具添加至智能体模块。这使智能体能够根据任务的上下文与需求，动态选择合适的工具。
  </Step>
</Steps>

## 工具配置

每个工具都需要特定的配置才能正常运行。常见的配置项包括：

- **API 密钥**：许多工具需要通过 API 密钥进行身份验证
- **连接参数**：端点、数据库标识符等
- **输入格式化**：数据传入工具时的结构规范
- **输出处理**：如何处理工具返回的结果

## 可用工具

TradingGoose 提供多种工具，适用于各类用途，包括：

- **AI 与语言处理**：OpenAI、ElevenLabs、翻译服务
- **搜索与研究**：Google Search、Tavily、Exa、Perplexity
- **文档处理**：Google Docs、Google Sheets、Notion、Confluence
- **媒体处理**：视觉识别、图像生成
- **通信**：Slack、WhatsApp、Twilio SMS、Gmail
- **数据存储**：Pinecone、Supabase、Airtable
- **开发**：GitHub

每个工具均有专属的文档页面，提供配置与使用的详细说明。

## 工具输出

工具通常返回结构化数据，可由工作流中后续模块进行处理。数据格式因工具和操作而异，但通常包含：

- 主要内容或结果
- 操作元数据
- 状态信息

请参阅各工具的专属文档，以了解其确切的输出格式。
