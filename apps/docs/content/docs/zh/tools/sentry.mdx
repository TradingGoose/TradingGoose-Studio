为特定环境中的 Sentry 版本创建部署记录。部署记录用于追踪版本的部署时间和部署位置。返回已创建的部署详情。

<div className="text-sm font-medium text-fd-muted-foreground mt-4 mb-2">输入</div>

| 参数 | 类型 | 必填 | 描述 |
| --------- | ---- | -------- | ----------- |
| `apiKey` | string | 是 | Sentry API 身份验证令牌 |
| `organizationSlug` | string | 是 | 组织的 slug |
| `version` | string | 是 | 待部署版本的版本标识符 |
| `environment` | string | 是 | 版本部署的目标环境名称 \(例如："production"、"staging"\) |
| `name` | string | 否 | 本次部署的可选名称 \(例如："Deploy v2.0 to Production"\) |
| `url` | string | 否 | 指向部署的 URL \(例如：CI/CD 流水线 URL\) |
| `dateStarted` | string | 否 | 部署开始时间的 ISO 8601 时间戳 \(默认为当前时间\) |
| `dateFinished` | string | 否 | 部署完成时间的 ISO 8601 时间戳 |

<div className="text-sm font-medium text-fd-muted-foreground mt-4 mb-2">输出</div>

| 参数 | 类型 | 描述 |
| --------- | ---- | ----------- |
| `id` | string | 唯一部署 ID |
| `environment` | string | 版本部署的目标环境名称 |
| `name` | string | 部署名称 |
| `url` | string | 指向部署的 URL |
| `dateStarted` | string | 部署开始时间 \(ISO 时间戳\) |

</div>
