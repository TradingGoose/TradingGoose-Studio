技能是定义 Agent 模块专属行为的可复用指令集。创建一次技能后，即可将其附加到同一工作区内任意工作流中的任意 Agent。

## 技能定义

每个技能均以 `SkillDefinition` 形式存储，包含四个面向用户的字段：

| 字段 | 规则 | 用途 |
|---|---|---|
| **name** | 短横线命名格式，1-64 个字符（例如 `market-analyst`） | 工作区内的唯一标识符 |
| **description** | 1-1024 个字符 | 向 Agent 及团队成员说明技能的功能 |
| **content** | 1-50,000 个字符 | 完整提示词：包含指令、上下文、规则及示例 |
| **id** | 自动生成 | 供 API 和存储层使用的不可变引用 |

每个技能还包含 `workspaceId`、`userId`、`createdAt`，以及可选的 `updatedAt` 时间戳。

## 创建与编辑技能

<Steps>
  <Step>
    在工作区中打开 [技能编辑器](/widgets/editor-skill) 组件。
  </Step>
  <Step>
    输入 **name**（短横线命名格式）、简短的 **description**，以及完整的 **content**——即 Agent 应遵循的指令。
  </Step>
  <Step>
    **保存**技能。API 将对所有字段进行验证，并拒绝同一工作区内的重复项。
  </Step>
</Steps>

如需更新现有技能，请在技能编辑器中选中该技能，修改任意字段后保存。所有引用该技能的工作流将在下次运行时获取最新内容。

如需删除技能，请使用技能编辑器中的删除操作。删除操作需要工作区的 `admin` 或 `write` 权限。

## 在 Agent 模块中使用技能

Agent 模块提供了一个 **技能** 输入项（`skill-input` 类型）。在配置阶段，您可从工作区中选择一个或多个技能。在运行时，Agent 将接收每个已选技能的内容，作为附加指令与系统提示和用户提示一并传入。

<Callout type="info">
  技能的作用域为工作区级别。所有拥有工作区访问权限的团队成员均可浏览和使用技能，具备写入权限的成员还可对技能进行编辑或删除。
</Callout>

## 技能在运行时的工作原理

1. 工作流执行器读取 Agent 的 `skills` 输入——即已选技能引用的数组。
2. 技能存储（`useSkillsStore`）通过 `getSkill(id, workspaceId)` 将每个引用解析至当前活动工作区。
3. 解析后的技能内容在 LLM 调用前被合并到 Agent 的提示上下文中。

技能存储按工作区（`skillsByWorkspace`）建立索引，切换工作区后将即时加载对应的技能集合。

## 权限说明

| 操作 | 所需权限 |
|---|---|
| 列出 / 读取技能 | 任意工作区访问权限 |
| 创建 / 更新 / 删除 | `admin` 或 `write` |

## 最佳实践

- **每个角色对应一个技能** —— 保持每项技能专注于单一角色或任务类型。
- **在内容中包含示例** 以固定 Agent 的行为模式。
- **就地迭代** —— 编辑技能后，所有引用该技能的工作流将自动更新，无需重新配置。
- **使用描述性名称** —— 短横线命名格式（如 `code-reviewer`、`financial-analyst`）同时作为界面中人类可读的标识符使用。
