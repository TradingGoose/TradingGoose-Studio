自定义工具允许您定义可供 Agent 块在运行时调用的自定义函数。每个工具由一对组件构成：**JSON 配置**（声明工具的名称、描述和参数）以及**JavaScript 函数体**（工具被调用时执行的逻辑）。

## 工具模式

每个自定义工具均遵循 OpenAI 函数调用 JSON 格式。编辑器会在每次更改时验证模式，并要求填写以下所有字段。

```json
{
  "type": "function",
  "function": {
    "name": "addItemToOrder",
    "description": "Add one quantity of a food item to the order.",
    "parameters": {
      "type": "object",
      "properties": {
        "itemName": {
          "type": "string",
          "description": "The name of the food item to add to order"
        }
      },
      "required": ["itemName"]
    }
  }
}
```

- `type` 必须为 `"function"`。
- `function.name` 是 Agent 可见并调用的标识符，在工作区内必须唯一。
- `function.description` 告知 Agent 何时使用该工具。
- `function.parameters` 是描述输入参数的标准 JSON Schema 对象。

## 工具代码

代码选项卡包含 `async function(params, environmentVariables)` 的函数体。编辑器内支持两种引用约定：

- **参数** — 直接使用参数名称（例如 `itemName`）。编辑器会根据模式提供自动补全。
- **环境变量** — 使用双花括号语法：`{{API_KEY}}`。

当工具需要产生输出时，函数应 `return` 一个值。不支持外部导入，请使用 `fetch` 或内置 API。

## 创建与编辑工具

<Steps>
  <Step>
    在工作区仪表板中打开 [自定义工具编辑器](/widgets/editor-custom-tool) 组件。
  </Step>
  <Step>
    从下拉列表中选择现有工具，或通过自定义工具列表组件新建一个工具。
  </Step>
  <Step>
    切换 **配置** 和 **代码** 选项卡，分别编辑 JSON 模式和 JavaScript 函数体。两个选项卡均包含 AI 魔棒功能，可根据自然语言提示生成内容。
  </Step>
  <Step>
    点击组件标题栏中的 **保存** 按钮。编辑器会在保存前验证模式，并拒绝重复的工具名称。
  </Step>
</Steps>

## 代理如何调用自定义工具

当您将自定义工具添加到 Agent 块的工具列表时，该工具将注册到 `type: "custom-tool"`。在运行时，Agent 块读取 `function.name` 作为工具标识符，`function.description` 作为工具描述，`function.parameters` 作为输入模式。代理根据这些字段和对话上下文决定是否以及何时调用该工具，然后使用解析后的参数执行 JavaScript 函数体。

<Callout type="info">
  Agent 块不能将核心块（API、Webhook、函数、工作流、内存）用作工具。工具列表中仅允许使用集成、MCP 工具和自定义工具。
</Callout>

## 自定义工具与 MCP 的对比

| 特性 | 自定义工具 | MCP |
|---------|-------------|-----|
| **定义** | JSON 模式与 JS 函数体 | 实现模型上下文协议的外部服务器 |
| **范围** | 每个工具对应一个函数 | 单个服务器可暴露多个工具 |
| **代码执行** | 编辑器中的内联 JavaScript | 在远程 MCP 服务器上运行 |
| **最适用于** | 快速、独立的逻辑或 API 调用 | 连接标准化的第三方工具服务器 |

有关使用 MCP 的外部工具集成，请参阅 [MCP 工具页面](/utilities/mcp)。

<Callout type="warn">
  自定义工具的作用域为工作区级别。所有工作区成员均可查看、编辑和删除它们。删除自定义工具将永久移除其代码和配置。
</Callout>
