TradingGoose 中的变量作为全局数据存储，可被工作流中的任意模块访问和修改，允许您通过全局变量在整个工作流中存储和共享数据。变量提供了一种强大的方式，用于在工作流的不同部分之间共享信息、维护状态，并构建更具动态性的应用程序。

<Callout type="info">
  变量允许您在整个工作流中存储和共享数据，从而轻松维护状态并构建复杂的互联系统。
</Callout>

## 概述

变量功能作为工作流的中央数据存储，使您能够：

<Steps>
  <Step>
    <strong>存储全局数据</strong>：创建在整个工作流执行过程中持久存在的变量
  </Step>
  <Step>
    <strong>在模块间共享信息</strong>：从工作流中的任意模块访问相同的数据
  </Step>
  <Step>
    <strong>维护工作流状态</strong>：在工作流运行过程中跟踪重要数值
  </Step>
  <Step>
    <strong>创建动态工作流</strong>：构建能够根据存储值自适应的灵活系统
  </Step>
</Steps>

## 创建变量

您可以通过侧边栏中的变量面板创建和管理变量。每个变量包含：

- **名称**：用于引用该变量的唯一标识符
- **值**：存储在变量中的数据（支持多种数据类型）
- **描述**（可选）：说明变量用途的备注

## 访问变量

工作流中的任意模块均可通过变量下拉菜单访问变量。操作步骤如下：

1. 在模块内的任意文本字段中输入 `<`
2. 浏览下拉菜单以从可用变量中进行选择
3. 选择您要使用的变量

当您在文本字段中输入 `<` 时，将出现一个下拉菜单，列出所有可用的工作流变量。选择所需变量以插入对其的引用。

<Callout>
  您也可以将连接标签拖入字段中，以打开变量下拉菜单并访问可用变量。
</Callout>

## 在模块中使用变量

当您从模块访问变量时，可以执行以下操作：

- **读取其值**：在模块逻辑中使用变量的当前值
- **修改变量**：根据模块的处理结果更新变量的值
- **用于表达式**：在表达式和计算中引用变量

## 变量作用域

TradingGoose 中的变量具有全局作用域，这意味着：

- 可从工作流中的任意模块访问
- 对变量的修改在整个工作流执行期间持续有效
- 变量在多次运行之间保持其值，除非被显式重置

## 最佳实践

- **使用描述性名称**：选择能清晰表明变量用途的命名。例如，使用 `userPreferences` 而非 `up`。
- **为变量添加说明**：为变量添加描述，帮助团队成员了解其用途和使用方式。
- **注意变量作用域**：请记住，变量是全局的，可被任意模块修改。在设计工作流时应充分考虑这一点，以防止出现意外行为。
- **提前初始化变量**：在工作流开始时设置并初始化变量，确保在需要时变量已可用。
- **处理缺失变量**：始终考虑变量可能尚不存在或具有意外值的情况，并在模块中添加适当的验证逻辑。
- **控制变量数量**：保持变量数量在可管理的范围内。过多的变量会使工作流难以理解和维护。
