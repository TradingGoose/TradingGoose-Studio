## 连接的工作原理

连接是允许数据在工作流中的各模块之间流动的通道。在 TradingGoose 中，连接定义了信息如何从一个模块传递到另一个模块，从而实现整个工作流的数据流转。

<Callout type="info">
  每个连接代表一种有向关系，数据从源模块的输出端流向目标模块的输入端。
</Callout>

### 创建连接

<Steps>
  <Step>
    <strong>选择源模块</strong>：点击您要连接的模块的输出端口
  </Step>
  <Step>
    <strong>绘制连接</strong>：拖拽至目标模块的输入端口
  </Step>
  <Step>
    <strong>确认连接</strong>：松开鼠标以创建连接
  </Step>
</Steps>

### 连接流向

数据通过连接传输时遵循以下原则：

1. **单向流动**：数据始终从输出端流向输入端
2. **执行顺序**：模块按照其连接关系依次执行
3. **数据转换**：数据在模块间传递时可能会发生转换
4. **条件路径**：某些模块（如路由器和条件模块）可将数据流引导至不同路径

<Callout type="warning">
  删除连接将立即中断模块之间的数据流。在移除连接之前，请确认此操作符合预期。
</Callout>
