当您连接模块时，了解不同模块输出的数据结构非常重要，因为源模块的输出数据结构决定了目标模块中可用的数据值。每种模块类型都会产生特定的输出结构，您可以在下游模块中引用这些结构。

<Callout type="info">
  了解这些数据结构对于有效使用连接标签以及在工作流中访问正确数据至关重要。
</Callout>

## 模块输出结构

不同类型的模块会产生不同的输出结构。以下是各类模块的输出说明：

<Tabs items={['Agent Output', 'API Output', 'Function Output', 'Evaluator Output', 'Condition Output', 'Router Output']}>
  <Tab>

    ```json
    {
      "content": "The generated text response",
      "model": "gpt-4o",
      "tokens": {
        "prompt": 120,
        "completion": 85,
        "total": 205
      },
      "toolCalls": [...],
      "cost": [...],
      "usage": [...]
    }
    ```

    ### Agent 模块输出字段

    - **content**：由 Agent 生成的主要文本响应
    - **model**：所使用的 AI 模型（例如 "gpt-4o"、"claude-3-opus"）
    - **tokens**：Token 使用统计
      - **prompt**：提示词中的 Token 数量
      - **completion**：补全内容中的 Token 数量
      - **total**：已使用的 Token 总数
    - **toolCalls**：Agent 执行的工具调用数组（如有）
    - **cost**：每次工具调用的费用对象数组（如有）
    - **usage**：整个响应的 Token 使用统计

  </Tab>
  <Tab>

    ```json
    {
      "data": "Response data",
      "status": 200,
      "headers": {
        "content-type": "application/json",
        "cache-control": "no-cache"
      }
    }
    ```

    ### API 模块输出字段

    - **data**：来自 API 的响应数据（可以是任意类型）
    - **status**：响应的 HTTP 状态码
    - **headers**：API 返回的 HTTP 头信息

  </Tab>
  <Tab>

    ```json
    {
      "result": "Function return value",
      "stdout": "Console output",
    }
    ```

    ### Function 模块输出字段

    - **result**：函数的返回值（可以是任意类型）
    - **stdout**：函数执行期间捕获的控制台输出

  </Tab>
  <Tab>

    ```json
    {
      "content": "Evaluation summary",
      "model": "gpt-5",
      "tokens": {
        "prompt": 120,
        "completion": 85,
        "total": 205
      },
      "metric1": 8.5,
      "metric2": 7.2,
      "metric3": 9.0
    }
    ```

    ### 评估器模块输出字段

    - **content**：评估摘要
    - **model**：用于评估的 AI 模型
    - **tokens**：Token 使用统计
    - **[metricName]**：评估器中每个指标的得分（动态字段）

  </Tab>
  <Tab>

    ```json
    {
      "content": "Original content passed through",
      "conditionResult": true,
      "selectedPath": {
        "blockId": "2acd9007-27e8-4510-a487-73d3b825e7c1",
        "blockType": "agent",
        "blockTitle": "Follow-up Agent"
      },
      "selectedConditionId": "condition-1"
    }
    ```

    ### 条件模块输出字段

    - **content**：透传的原始内容
    - **conditionResult**：条件评估的布尔结果
    - **selectedPath**：所选路径的信息
      - **blockId**：所选路径中下一个模块的 ID
      - **blockType**：下一个模块的类型
      - **blockTitle**：下一个模块的标题
    - **selectedConditionId**：所选条件的 ID

  </Tab>
  <Tab>

    ```json
    {
      "content": "Routing decision",
      "model": "gpt-4o",
      "tokens": {
        "prompt": 120,
        "completion": 85,
        "total": 205
      },
      "selectedPath": {
        "blockId": "2acd9007-27e8-4510-a487-73d3b825e7c1",
        "blockType": "agent",
        "blockTitle": "Customer Service Agent"
      }
    }
    ```

    ### 路由模块输出字段

    - **content**：路由决策文本
    - **model**：用于路由的 AI 模型
    - **tokens**：Token 使用统计
    - **selectedPath**：所选路径的信息
      - **blockId**：所选目标模块的 ID
      - **blockType**：所选模块的类型
      - **blockTitle**：所选模块的标题

  </Tab>
</Tabs>

## 自定义输出结构

部分模块可能会根据其配置生成自定义输出结构：

1. **带响应格式的 Agent 模块**：在 Agent 模块中使用响应格式时，输出结构将遵循所定义的 Schema，而非标准结构。

2. **函数模块**：`result` 字段可包含函数代码返回的任意数据结构。

3. **API 模块**：`data` 字段将包含 API 返回的任意内容，可以是任何合法的 JSON 结构。

<Callout type="warning">
  开发过程中请务必检查模块的实际输出结构，以确保在连接中正确引用相应字段。
</Callout>

## 嵌套数据结构

许多模块输出包含嵌套数据结构。您可以在连接标签中使用点表示法访问这些数据：

```
<blockName.path.to.nested.data>
```

例如：

- `<agent1.tokens.total>` - 访问 Agent 模块的总 token 数
- `<api1.data.results[0].id>` - 访问 API 响应中第一个结果的 ID
- `<function1.result.calculations.total>` - 访问 Function 模块结果中的嵌套字段
