连接标签是对已连接模块可用数据的可视化呈现，提供了一种在工作流中跨模块引用数据及引用前序模块输出的便捷方式。

### 什么是连接标签？

连接标签是模块连接后显示的交互元素，代表可在模块之间传递的数据，允许您：

- 可视化源模块的可用数据
- 在目标模块中引用特定数据字段
- 在模块之间创建动态数据流

<Callout type="info">
  连接标签让您轻松查看前序模块的可用数据，并在当前模块中直接使用，无需记忆复杂的数据结构。
</Callout>

## 标签语法

连接标签使用简单的语法来引用数据：

```
<blockName.path.to.data>
```

其中：

- `blockName` 是源代码块的名称（或其 ID）
- `path.to.data` 是指向特定数据字段的点分隔路径

例如：

- `<agent1.content>` - 引用名为 "agent1" 的代码块中的 content 字段
- `<api2.data.users[0].name>` - 引用名为 "api2" 的代码块的 data 字段中 users 数组第一个用户的 name

### 特殊引用类型

除块引用外，解析器还支持几种特殊前缀：

- **触发器别名**：`<start.input>`、`<api.fieldName>`、`<chat.input>`、`<manual.input>` - 触发器块输出的简写引用
- **变量**：`<variable.myVar>` - 按名称引用工作流变量
- **循环**：`<loop.item>`、`<loop.index>` - 引用循环中的当前项或索引
- **并行执行**：`<parallel.item>`、`<parallel.index>` - 引用并行执行中的当前项或索引

## 动态标签引用

连接标签在运行时求值，这意味着：

1. 它们始终引用执行上下文中的最新数据
2. 它们可以在表达式中使用，并与静态文本组合
3. 它们可以嵌套在其他数据结构中
4. 支持使用括号表示法进行数组索引（例如，`items[0]`）

### 示例

```javascript
// Reference in text
"The user's name is <userBlock.name>"

// Reference in JSON
{
  "userName": "<userBlock.name>",
  "orderTotal": <apiBlock.data.total>
}

// Using trigger aliases
"User said: <start.input>"

// Using loop references inside a loop
"Processing item <loop.index>: <loop.item.name>"
```

<Callout type="warning">
  在数值上下文中使用连接标签时，请确保所引用的数据确实为数字类型，
  以避免类型转换问题。
</Callout>
