## 概览

| 属性 | 值 |
|----------|-------|
| **Key** | `editor_workflow` |
| **Category** | Editor |

工作流编辑器是构建 AI 工作流的主画布。它渲染一个完整的 React Flow 图形，您可以在其中放置模块、绘制模块之间的连接，并配置自动化流水线的每个步骤。该组件在独立的 Socket 连接和工作流存储中运行，因此同一工作区中可以有多个编辑器独立运行。

## 功能特性

- **React Flow 画布**，内置平移、缩放和节点选择功能
- **头部区域的模块工具栏**，带有可搜索的下拉菜单，按类别分组列出模块、工具和触发器；将模块从工具栏拖拽到画布即可添加
- **循环和并行模块**，作为控制流逻辑的特殊工具栏项提供
- **浮动控件**，用于快速执行画布操作（在组件配置中默认启用）
- **训练控件**切换开关，可按组件开启或关闭训练模式覆盖层
- **差异控件**，用于对比当前工作流与历史版本之间的变更
- **触发器列表**，显示附加到工作流的所有触发器
- **视口感知定位** —— 组件跟踪自身的边界矩形，确保新模块即使在嵌入小面板时也能落在画布的可视中心
- **嵌套子流验证**，当子流引用无效时实时高亮显示错误
- **触发器警告对话框**，在添加重复触发器或触发其他触发器约束时弹出

## 使用方法

### 添加小组件

在任意工作区面板中打开小组件选择器，选择 **工作流编辑器**。该小组件将加载当前工作区的活动工作流。若尚未创建任何工作流，则显示占位提示信息。

### 标题栏布局

标题栏分为三个区域：

- **左侧**：模块工具栏 —— 一个下拉菜单，用于浏览和搜索可用的模块、工具及触发器，并将其添加至画布
- **中间**：工作流选择器下拉菜单，用于在工作区内切换不同工作流
- **右侧**：控制栏，包含执行、保存及工作流级别的操作

### 主要交互操作

1. **添加模块** —— 打开工具栏下拉菜单，搜索或浏览后点击模块，将其放置于视口中心
2. **连接模块** —— 从输出节点拖拽至输入节点以创建连线
3. **选择与配置** —— 点击节点以选中它；其设置将显示在配对的 Copilot 或变量小组件中
4. **删除** —— 选中节点或连线后按 Delete 键
5. **撤销 / 重做** —— 使用标准键盘快捷键

### 配对颜色同步

当分配了配对颜色后，编辑器会将其活动工作流与共享相同颜色的其他小组件同步。在标题栏选择器中更改工作流时，所选内容将广播至所有已配对的小组件。当配对颜色为 `gray`（未关联）时，小组件将通过小组件参数独立管理自身的选择。

<Callout>
使用匹配的配对颜色将此小组件与 **Console**、**Chat**、**Variables** 或 **Copilot** 配对，以构建完整的工作流开发工作区。
</Callout>

## 配置

画布 UI 由 `WorkflowCanvasUIConfig` 控制：

| 选项 | 默认值 | 描述 |
|--------|---------|-------------|
| `floatingControls` | `true` | 在画布上显示浮动操作按钮 |
| `trainingControls` | `true` | 启用训练模式控制覆盖层 |
| `forceTrainingControls` | `false` | 无论用户是否切换，始终显示训练控件 |
| `diffControls` | `true` | 显示差异审查控件 |
| `triggerList` | `true` | 显示触发器列表面板 |
| `controlBar` | `false` | 显示旧版控制栏（组件中默认关闭） |

这些默认值在组件层级设置，并与布局层级的覆盖配置合并。
