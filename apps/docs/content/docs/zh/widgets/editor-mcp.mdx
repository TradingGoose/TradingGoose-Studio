## 概览

| 属性 | 值 |
|----------|-------|
| **键** | `editor_mcp` |
| **类别** | 编辑器 |

MCP 编辑器允许您检查、配置、测试并刷新附加到工作区的模型上下文协议（Model Context Protocol）服务器。它显示服务器的连接状态、传输设置、已发现的工具以及最近发生的错误。使用 [MCP 列表](/utilities/mcp) 组件添加或移除服务器；编辑器是管理单个服务器配置的主要界面。

## 功能特性

- **服务器信息标题栏** 显示服务器名称、带颜色编码的连接状态徽章（已连接 / 错误 / 已断开）以及传输类型（如 HTTP、SSE）。
- **时间戳** 以相对时间显示"最后更新"、"工具刷新时间"和"最后连接时间"，便于快速了解数据的时效性。
- **服务器表单** 允许编辑传输类型（默认为 streamable-http）、端点 URL、请求超时时间及自定义 HTTP 请求头。
- **测试连接按钮** 使用当前表单值向服务器发送测试请求，并将结果内联显示。
- **刷新工具按钮** 触发服务器端刷新，重新发现 MCP 服务器暴露的所有工具并更新本地工具列表。
- **重置表单按钮** 将表单字段恢复至最后保存的服务器配置。
- **保存按钮** 持久化更新后的服务器设置。若尚未执行测试，保存操作将自动先执行测试，随后保存并刷新工具。
- **清除选择按钮** 取消选中当前服务器。
- **工具列表** 枚举从服务器发现的所有工具，显示每个工具的名称和描述。
- **错误面板** 在服务器存在 `lastError` 时出现在底部，显示错误信息。
- **服务器选择器** 位于顶部标题栏，用于在工作区中的 MCP 服务器之间切换。通过颜色链接配对后，可与 MCP 列表组件同步。

## 使用方法

1. 从小部件选择器将该小部件添加到工作区。
2. 从顶部下拉菜单选择一个 MCP 服务器，或通过 [MCP 列表](/utilities/mcp) 小部件添加一个。
3. 查看顶部的连接状态和时间戳。
4. 如有需要，编辑 **URL**、**超时时间** 或 **请求头**。
5. 点击 **测试连接** 以验证服务器在当前设置下是否可达。
6. 点击 **刷新工具** 以重新发现服务器暴露的工具。
7. 点击 **保存** 以持久化更改。如果尚未测试连接，小部件将先进行测试，然后保存并刷新工具。

### 顶栏操作

顶栏工具栏提供五个操作按钮（从左到右）：

| 按钮 | 图标 | 操作 |
|--------|------|--------|
| 刷新工具 | 旋转 | 重新发现服务器的工具 |
| 测试连接 | 播放 | 使用当前表单值发送测试请求 |
| 重置表单 | 撤销 | 将表单恢复为上次保存的值 |
| 保存服务器 | 保存 | 持久化配置，按需自动测试，刷新工具 |
| 清除选择 | X | 取消选中服务器 |

<Callout>
此小部件支持 **pairColor** 机制。将相同颜色分配给 MCP 编辑器和 [MCP 列表](/utilities/mcp) 小部件，这样在列表中选择服务器时将自动在编辑器中打开该服务器。
</Callout>

## 配置

所选服务器以 `mcpServerId` 的形式存储在小部件参数中，并在关联时通过配对颜色存储进行同步。表单字段（传输协议、URL、超时时间、请求头）均为本地状态，派生自所选服务器，在点击保存之前不会被持久化。
