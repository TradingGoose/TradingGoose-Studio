Webhook 允许外部服务通过发送 HTTP 请求来触发工作流执行。TradingGoose 支持两种基于 Webhook 的触发方式。

<Callout type="info">
  这是一个基于 **Webhook** 的触发器。请在您的外部服务中配置 Webhook URL，以将事件发送至 TradingGoose。
</Callout>

### 工作原理

1. **添加通用 Webhook 模块** — 将通用 Webhook 模块拖入以启动您的工作流
2. **获取 Webhook URL** — 复制自动生成的唯一端点地址
3. **外部集成** — 配置您的外部服务向此 URL 发送 POST 请求
4. **工作流执行** — 每次向 Webhook URL 发送请求均会触发工作流

### 功能特性

- **灵活的载荷** — 接受任意 JSON 载荷结构
- **自动解析** — Webhook 数据将被自动解析，并可供后续模块使用
- **身份验证** — 支持可选的 Bearer Token 或自定义请求头认证
- **速率限制** — 内置防滥用保护机制

## 服务模块的触发模式

此外，您也可以将特定服务模块（如 Slack、GitHub 等）设置为「触发模式」，以创建更专用的 Webhook 端点。

**在以下情况下使用通用 Webhook：**
- 与自定义应用程序或服务集成
- 需要对载荷结构有最大灵活性
- 与没有专用模块的服务配合使用

**在以下情况下使用触发器模式：**
- 与受支持的服务（Slack、GitHub 等）配合使用
- 需要针对特定服务进行事件过滤
- 需要自动注册 Webhook

<Callout>
通用 Webhook 触发器在每次 Webhook URL 收到请求时触发，非常适合实时集成场景。
</Callout>
