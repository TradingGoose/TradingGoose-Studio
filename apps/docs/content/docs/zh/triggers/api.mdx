API 触发器将您的工作流暴露为安全的 HTTP 端点。向该端点发送 JSON 数据，工作流将立即对其进行处理。API 调用始终针对您的最新部署执行。

<Callout type="info">
  这是一个**基于 Webhook** 的触发器。外部服务或脚本通过发送 HTTP 请求来启动工作流。
</Callout>

为每个参数添加一个**输入格式**字段。支持的类型：

- **string** — 文本值
- **number** — 数值
- **boolean** — 布尔值（真/假）
- **json** — JSON 对象
- **files** — 文件上传（通过 `<api.fieldName[0].url>`、`<api.fieldName[0].name>` 等访问）

### 输出参考

| 引用 | 类型 | 描述 |
|-----------|------|-------------|
| `<api.field>` | 可变 | 在输入格式中定义的字段 |
| `<api.input>` | json | 完整的结构化请求体 |

## 请求示例

```bash
curl -X POST \
  https://tradinggoose.ai/api/workflows/WORKFLOW_ID/execute \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: YOUR_KEY' \
  -d '{"userId":"demo-user","maxTokens":1024}'
```

## 流式响应

启用实时流式传输，以在工作流输出生成时即时接收。

### 请求参数

- `stream` — 设置为 `true` 以启用服务器发送事件（SSE）流式传输
- `selectedOutputs` — 需要流式传输的块输出数组（例如：`["agent1.content"]`）

### 示例

```bash
curl -X POST \
  https://tradinggoose.ai/api/workflows/WORKFLOW_ID/execute \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: YOUR_KEY' \
  -d '{"message": "Count to five", "stream": true, "selectedOutputs": ["agent1.content"]}'
```

### 响应格式

```
data: {"blockId":"...","chunk":"One"}
data: {"blockId":"...","chunk":", two"}
data: {"event":"done","success":true,"output":{},"metadata":{"duration":610}}
data: [DONE]
```

### 文件上传格式

```json
{
  "documents": [{
    "type": "file",
    "data": "data:application/pdf;base64,JVBERi0xLjQK...",
    "name": "document.pdf",
    "mime": "application/pdf"
  }]
}
```

<Callout type="warning">
工作流只能包含一个 API 触发器。更改后请发布新部署，以确保端点保持最新状态。
</Callout>
