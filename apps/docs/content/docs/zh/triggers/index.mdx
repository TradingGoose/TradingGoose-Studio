## 核心触发器

每个工作流选择一个触发器以定义其启动方式：

<Cards>
  <Card title="API" href="/triggers/api">
    将 JSON 请求体映射为工作流输入的 HTTP 端点
  </Card>
  <Card title="Chat" href="/triggers/chat">
    支持流式响应的已部署对话界面
  </Card>
  <Card title="Input Form" href="/triggers/input-form">
    用于编辑器运行和子工作流的类型化手动输入
  </Card>
  <Card title="Manual" href="/triggers/manual">
    无需额外数据的按需运行
  </Card>
  <Card title="Schedule" href="/triggers/schedule">
    基于 Cron 或时间间隔的定时执行
  </Card>
  <Card title="Webhook" href="/triggers/webhook">
    接收外部 Webhook 载荷
  </Card>
</Cards>

## 快速对比

| 触发器 | 启动条件 |
|---------|-----------------|
| **API** | 已认证的 HTTP POST 请求 |
| **Chat** | 对话部署消息 |
| **Input Form** | 在编辑器或父工作流中手动提交 |
| **Manual** | 编辑器中的运行按钮 |
| **Schedule** | 在计划模态框中管理的定时器 |
| **Webhook** | 收到入站 HTTP 请求时 |

## 使用触发器

1. 将触发器模块拖放至起始插槽。
2. 配置所需的 Schema 或身份验证。
3. 将模块连接至工作流的其余部分。

> 每个触发器均由部署驱动。更新工作流并重新部署后，所有触发器入口点将自动加载最新快照。详情请参阅 [执行 → 部署快照](/execution)。

旧版 Starter 模块在现有流程中仍可使用，但不再出现在新构建中。
