## 核心概念

### 拓扑执行
模块按依赖顺序执行，类似于电子表格重新计算单元格的方式。执行引擎根据已完成的依赖项自动判断哪些模块可以运行。

### 路径追踪
引擎主动追踪工作流中的执行路径。路由模块和条件模块动态更新这些路径，确保仅执行相关模块。

### 基于层次的处理
执行引擎并非逐一执行模块，而是识别可并行运行的模块层，从而优化复杂工作流的性能。

### 执行上下文
每个工作流在执行过程中维护丰富的上下文信息，包含：
- 模块输出与状态
- 活跃执行路径
- 循环与并行迭代追踪
- 环境变量
- 路由决策

## 执行触发器

工作流可通过多种渠道执行：

- **手动**：直接在编辑器中测试和调试
- **部署为 API**：创建以 API 密钥保护的 HTTP 端点
- **部署为聊天**：在自定义子域名上创建对话界面
- **Webhooks**：响应来自第三方服务的外部事件
- **定时执行**：使用 Cron 表达式按周期性计划运行

### 部署为 API

将工作流部署为 API 时，TradingGoose 将：
- 创建唯一的 HTTP 端点：`https://tradinggoose.ai/api/workflows/{workflowId}/execute`
- 生成用于身份验证的 API 密钥
- 接受携带 JSON 负载的 POST 请求
- 以 JSON 格式返回工作流执行结果

API 调用示例：

```bash
curl -X POST https://tradinggoose.ai/api/workflows/your-workflow-id/execute \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"input": "your data here"}'
```

### 部署为聊天

聊天部署为您的工作流创建一个对话界面：
- 托管于自定义子域名：`https://your-name.tradinggoose.ai`
- 可选身份验证（公开、密码或基于邮箱）
- 可自定义 UI 以匹配您的品牌风格
- 支持流式响应以实现实时交互
- 非常适合 AI 助手、客服机器人或交互式工具

每种部署方式均将数据传递至工作流的起始块，从而启动执行流程。

## 部署快照

所有公共入口点——API、聊天、定时任务、Webhook 及手动运行——均执行工作流的当前活跃部署快照。每次修改画布后请发布新的部署，以确保所有触发器均使用最新版本。

部署弹窗以部署快照表格的形式保存完整的版本历史记录。您可以查看任意快照，将其与草稿进行对比，并在需要恢复先前版本时一键升级或回滚。

## 程序化执行

通过我们的官方 SDK 在您的应用程序中执行工作流：

```bash
# TypeScript/JavaScript
npm install tradinggoose-ts-sdk

# Python
pip install tradinggoose-sdk
```

```typescript
// TypeScript Example
import { TradingGooseClient } from 'tradinggoose-ts-sdk';

const client = new TradingGooseClient({ 
  apiKey: 'your-api-key' 
});

const result = await client.executeWorkflow('workflow-id', {
  input: { message: 'Hello' }
});
```

## 最佳实践

### 设计可靠性
- 通过适当的回退路径优雅地处理错误
- 对敏感数据使用环境变量
- 为 Function 块添加日志记录以便调试

### 优化性能
- 尽可能减少外部 API 调用
- 对独立操作使用并行执行
- 在适当情况下使用 Variables 块缓存结果

### 监控执行
- 定期查看日志以了解性能模式
- 跟踪 AI 模型使用的成本
- 使用工作流快照调试问题
