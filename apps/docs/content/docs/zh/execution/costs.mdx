TradingGoose 自动计算所有工作流执行的费用，根据 AI 模型使用量和执行收费提供透明定价。了解这些费用有助于您优化工作流并有效管理预算。

## 费用计算方式

每次工作流执行包含两项费用组成部分：

**基础执行费用**：每次执行 $0.001

**AI 模型使用费**：根据 Token 消耗量浮动计费

```javascript
modelCost = (inputTokens × inputPrice + outputTokens × outputPrice) / 1,000,000
totalCost = baseExecutionCharge + modelCost
```

<Callout type="info">
  AI 模型价格按每百万 Token 计算。计算时除以 1,000,000 得出实际费用。不含 AI 模块的工作流仅收取基础执行费用。
</Callout>

## 日志中的模型费用明细

对于使用 AI 模块的工作流，您可以在日志中查看详细费用信息：

日志详情侧边栏包含模型费用明细部分，显示以下内容：
- **Token 使用量**：每个模型的输入和输出 Token 数量
- **费用明细**：每个模型和操作的单项费用
- **模型分布**：使用了哪些模型及其调用次数
- **总费用**：整个工作流执行的汇总费用

## 用量监控

在「设置 → 订阅」中监控您的使用量与账单信息：

- **当前用量**：当前计费周期内的实时用量与费用
- **用量限额**：含可视化进度指示器的套餐限额
- **账单详情**：预计费用与最低消费承诺
- **套餐管理**：升级选项与账单历史记录

### 通过编程方式追踪用量

您可通过 API 以编程方式查询当前用量与限额：

**接口端点：**

```text
GET /api/users/me/usage-limits
```

**身份验证：**
- 请在 `X-API-Key` 请求头中包含您的 API 密钥

**请求示例：**

```bash
curl -X GET -H "X-API-Key: YOUR_API_KEY" -H "Content-Type: application/json" https://tradinggoose.ai/api/users/me/usage-limits
```

**响应示例：**

```json
{
  "success": true,
  "rateLimit": {
    "sync": { "isLimited": false, "limit": 10, "remaining": 10, "resetAt": "2025-09-08T22:51:55.999Z" },
    "async": { "isLimited": false, "limit": 50, "remaining": 50, "resetAt": "2025-09-08T22:51:56.155Z" },
    "authType": "api"
  },
  "usage": {
    "currentPeriodCost": 12.34,
    "limit": 100,
    "plan": "pro"
  }
}
```

**响应字段说明：**
- `currentPeriodCost` 反映当前计费周期内的用量
- `limit` 来源于各套餐的独立限额（免费版/专业版），或组织的共享限额池（团队版/企业版）
- `plan` 为与您账户关联的最高优先级有效套餐

## 计费模式

TradingGoose 采用**基础订阅 + 超额用量**计费模式：

### 运作方式

**专业版（$20/月）：**
- 月度订阅包含 $20 的用量额度
- 用量低于 $20 → 无额外费用
- 用量超过 $20 → 月末支付超额部分
- 示例：用量 $35 = $20（订阅费）+ $15（超额费）

**团队版（$40/席位/月）：**
- 所有团队成员共享用量池
- 超额费用基于团队总用量计算
- 账单统一发送至组织所有者

**企业版：**
- 固定月费，无超额收费
- 按协议定制用量上限

### 阈值计费

当未结算的超额费用达到 $50 时，TradingGoose 将自动对全部未结算金额进行扣款。

**示例：**
- 第10天：超额 $70 → 立即扣款 $70
- 第15天：额外用量 $35（累计 $105）→ 已结算，无需操作
- 第20天：再次用量 $50（累计 $155，未结算 $85）→ 立即扣款 $85

此机制将大额超额费用分散至月内多次扣款，避免月末产生单笔大额账单。

## 成本管理最佳实践

1. **定期监控**：频繁查看用量仪表板，避免费用超出预期
2. **设置预算**：以套餐限额作为支出的控制基准
3. **优化工作流**：审查高成本执行任务，优化提示词或模型选择
4. **选用合适的模型**：根据任务需求匹配相应的模型复杂度
5. **批量处理相似任务**：尽量合并多个请求以降低额外开销
