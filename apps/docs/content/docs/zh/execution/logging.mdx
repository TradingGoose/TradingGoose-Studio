TradingGoose 为所有工作流执行提供全面的日志记录，让您完整掌握工作流的运行状况、数据流向以及潜在问题所在。

## 日志系统

TradingGoose 提供两种互补的日志界面，以适配不同的工作流场景和使用需求：

### 实时控制台

在手动或聊天工作流执行期间，日志将实时显示在工作流编辑器右侧的控制台面板中：

控制台面板位于工作流编辑器右侧，显示以下内容：
- 块执行进度（高亮显示当前活动块）
- 块完成时的实时输出
- 每个块的执行耗时
- 成功/错误状态指示

### 日志页面

所有工作流执行记录——无论是手动触发、通过 API、聊天、定时任务还是 Webhook 触发——均记录至专用日志页面：

日志页面提供：
- 按时间范围、状态、触发类型、文件夹及工作流进行综合筛选
- 跨所有日志的搜索功能
- 实时更新的直播模式
- 7 天日志保留期（可升级以延长保留时长）

## 日志详情侧边栏

点击任意日志条目，将打开详细侧边栏视图，显示执行时间线、块级详情及费用摘要。

### 块输入/输出

通过选项卡查看每个块的完整数据流，可在以下视图间切换：

<Tabs items={['Output', 'Input']}>
  <Tab>
    **输出选项卡** 显示块的执行结果：
    - 带 JSON 格式化的结构化数据
    - AI 生成内容的 Markdown 渲染
    - 便于数据提取的复制按钮
  </Tab>
  
  <Tab>
    **输入选项卡** 显示传入块的内容：
    - 已解析的变量值
    - 其他块的引用输出
    - 所用环境变量
    - API 密钥自动脱敏以保障安全
  </Tab>
</Tabs>

### 执行时间线

对于工作流级别的日志，查看详细的执行指标：
- 开始和结束时间戳
- 工作流总耗时
- 各模块执行时间
- 性能瓶颈识别

## 工作流快照

对于任何已记录的执行，点击"查看快照"以查看执行时的精确工作流状态。快照提供：
- 展示工作流结构的冻结画布
- 执行期间各模块的状态和连接关系
- 点击任意模块查看其输入和输出
- 适用于调试已被修改的工作流

<Callout type="info">
  工作流快照仅适用于增强日志系统引入后的执行记录。较旧的迁移日志会显示"未找到已记录状态"的提示信息。
</Callout>

## 日志保留

- **免费计划**：日志保留 7 天
- **专业计划**：日志保留 30 天
- **团队计划**：日志保留 90 天
- **企业计划**：提供自定义保留周期

## 最佳实践

### 开发阶段
- 在测试期间使用实时控制台获取即时反馈
- 检查模块的输入和输出以验证数据流
- 使用工作流快照对比正常版本与异常版本

### 生产环境
- 定期监控日志页面，排查错误或性能问题
- 设置过滤器以聚焦特定工作流或时间段
- 在关键部署期间启用实时模式，实时监控执行情况

### 调试阶段
- 始终检查执行时间线以识别慢速模块
- 对比正常执行与失败执行之间的输入差异
- 使用工作流快照查看问题发生时的精确状态
