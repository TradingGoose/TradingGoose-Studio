了解工作流在 TradingGoose 中的执行方式，是构建高效、可靠自动化流程的关键。执行引擎自动处理依赖关系、并发控制和数据流，确保您的工作流稳定、可预期地运行。

## 工作流的执行方式

TradingGoose 的执行引擎通过分析依赖关系并以最优顺序运行各模块，实现对工作流的智能处理。

### 默认并发执行

当多个模块之间不存在依赖关系时，它们将并发运行。这种并行执行方式无需手动配置，即可显著提升性能。

例如，若一个 Start 模块同时连接到 Customer Support 智能体和 Deep Researcher 智能体，两个智能体模块将同时执行，从而最大化执行效率。

### 自动合并输出

当某个模块存在多个依赖项时，执行引擎会自动等待所有依赖项完成，再将其合并后的输出提供给下一个模块，无需手动合并。

例如，一个连接了两个上游智能体模块的 Function 模块，将等待两者均执行完毕后，再接收其合并输出进行处理。

### 智能路由

工作流可通过路由模块向多个方向分支。执行引擎同时支持确定性路由（通过 Condition 模块）和 AI 驱动路由（通过 Router 模块）。

例如，一个工作流在初始模块之后可能分支为一个 Condition 模块（用于评估逻辑表达式）和一个 Router 模块（由 AI 决定执行路径）。每条分支均独立执行。

## 模块类型

TradingGoose 提供不同类型的模块，在工作流中各司其职：

<Cards>
  <Card title="触发器" href="/triggers">

  </Card>
  
  <Card title="处理块" href="/blocks">
    **Agent 块**与 AI 模型交互，**函数块**运行自定义代码，**API 块**连接外部服务。这些块负责转换和处理您的数据。
  </Card>
  
  <Card title="控制流" href="/blocks">

  </Card>
  
  <Card title="实用块" href="/blocks">

  </Card>

所有块均根据其依赖关系自动执行——您无需手动管理执行顺序或时序。

## 执行触发器

工作流可以通过多种方式触发，具体取决于您的使用场景：

### 手动测试
在工作流编辑器中点击「运行」，以在开发过程中测试您的工作流。非常适合调试和验证。

### 定时执行
使用 cron 表达式设置周期性执行。非常适合定期数据处理、报告生成或维护任务。

### API 部署
将工作流部署为 HTTP 端点，可从您的应用程序以编程方式调用。

### Webhook 集成
实时响应来自 GitHub、Stripe 或自定义系统等外部服务的事件。

### 聊天界面
在自定义子域上托管对话界面，用于面向用户的 AI 应用程序。

## 执行监控

当工作流运行时，TradingGoose 提供对执行过程的实时可见性：

- **实时区块状态**：查看当前正在执行、已完成或失败的区块
- **执行日志**：实时显示详细日志，包括输入、输出及任何错误信息
- **性能指标**：追踪每个区块的执行时间和成本
- **路径可视化**：了解工作流中所经过的执行路径

<Callout type="info">
  所有执行详情均已记录，即使工作流完成后仍可供查阅，有助于调试与优化。
</Callout>

## 核心执行原则

理解以下核心原则将有助于您构建更优质的工作流：

1. **基于依赖的执行**：区块仅在其所有依赖项完成后才会运行
2. **自动并行化**：独立区块无需配置即可并发执行
3. **智能数据流**：输出自动流转至已连接的区块
4. **错误处理**：失败的区块将中止其执行路径，但不影响独立路径
5. **状态持久化**：所有区块输出及执行详情均被保留，以供调试使用
