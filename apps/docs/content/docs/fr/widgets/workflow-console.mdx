Le widget Console de flux de travail offre une vue en temps réel de l'exécution du flux de travail, affichant les journaux, les informations de débogage et la sortie en streaming pendant l'exécution de votre flux de travail.

## Aperçu

| Propriété | Valeur |
|----------|-------|
| **Clé** | `workflow_console` |
| **Catégorie** | Utilitaire |
| **Icône** | Activité |

## Fonctionnalités

- **Journaux en temps réel** : Visualisez les journaux d'exécution en direct
- **Sortie en streaming** : Observez les sorties des blocs en temps réel
- **État d'exécution** : Surveillez l'état actuel de l'exécution du flux de travail
- **Affichage des erreurs** : Identifiez et déboguez rapidement les erreurs
- **Filtrage des journaux** : Filtrez les journaux par type, bloc ou gravité
- **Effacement de la console** : Réinitialisez la sortie de la console

## Utilisation

La Console de flux de travail est essentielle pour :

- Déboguer l'exécution du flux de travail
- Surveiller les flux de travail de longue durée
- Visualiser les sorties intermédiaires des blocs
- Identifier les erreurs et les goulots d'étranglement

### Types de journaux

| Type | Description |
|------|-------------|
| **Info** | Informations générales d'exécution |
| **Debug** | Informations détaillées de débogage |
| **Warning** | Problèmes non critiques |
| **Error** | Erreurs et échecs d'exécution |
| **Output** | Données de sortie des blocs |

### Visualisation du flux d'exécution

La console montre le chemin d'exécution à travers votre flux de travail :

1. **Démarrage** : L'exécution du flux de travail commence
2. **Entrée de bloc** : Chaque bloc commence le traitement
3. **Sortie de bloc** : Résultats de chaque bloc
4. **Sortie de bloc** : Le bloc se termine
5. **Fin** : L'exécution du flux de travail se termine

<Callout>
Ce widget prend en charge le mécanisme **pairColor**. Associez-le à un Éditeur de flux de travail pour voir les journaux du flux de travail que vous éditez.
</Callout>
